Скриптування та Безпека¶

Можливість включення скриптів Python у blend-файли є цінною для просунутих завдань таких, як оснащення, автоматизація та використання ігрового рушія Game Engine. Однак це створює ризик для безпеки, оскільки Python не обмежує те, що може робити скрипт.

Тому, вам слід виконувати скрипти лише джерел, які ви знаєте та яким довіряєте.

Автоматичне виконання стандартно вимкнене, однак, деякі з blend-файлів потребують його для правильного функціонуванн.

При спробі blend-файлом виконати скрипт і при відсутності дозволу на це у заголовку вікна з'являється повідомлення про це з кнопками «Перевантажити Довірене» Reload Trusted та «Ігнорувати» Ignore.

Попередження про Авто-виконання у заголовку редактора Інфо.

Скрипти у Blend-Файлах¶

Авто Виконання¶

Ось різні способи, якими blend-файли можуть автоматично запускати скрипти.

Зареєстровані текстові блоки
Текстовий блок може мати увімкнену опцію реєстрації Register, що означає, що він буде завантажуватися при запуску програми.
Драйвери Анімації
Вирази на Python можуть бути використані для значень Drive та часто використовуються у більш просунутих оснастках та анімаціях.
Авто Старт Ігрового Рушія
Скрипти часто використовуються для ігрової логіки, а blend-файли з ними можуть мати увімкнену опцію авто-старту Auto Start для запуску гри при запуску програми.

Виконання Вручну¶

Існують і інші способи, якими можна виконувати скрипти у blend-файлах, що вимагають взаємодії користувача (а тому ці скрипти будуть виконуватися, коли опція авто виконання вимкнута), але ви повинні знати, що так буває, і це не є обов'язково очевидним.

  • Виконання скрипту у редакторі Тексту.
  • Рендеринг рушієм FreeStyle, оскільки цей рушій використовує скрипти для керування стилями ліній.
  • Виконання Ігровим Рушієм.

Керування Виконанням Скрипту¶

Blender передбачає кілька способів контролювання, чи скриптам з blend-файлу дозволено автоматично виконуватися.

Перш за все, Оглядач Файлів -- File Browser має опцію «Довірене Джерело» -- Trusted Source, яка може використовуватися у кожному конкретному випадку для керування авто виконанням.

Однак, ви можете забути про цю установку або відкривати файл не через Оглядач Файлів -- а тому існує можливість зміни стандартної поведінки опції авто-виконання (описана нижче).

Установлення Стандартів¶

У вкладці «Файл»File вікна Уподобання Користувача є стяг-перемикач «Авто Виконувати Скрипти Python» Auto Run Python Scripts.

При увімкненні це означатиме, що опція «Довірене Джерело» Trusted Source в Оглядачі Файлів також буде стандартно увімкнена і скрипти зможуть виконуватися при завантаженні blend-файлів у Blender без потреби використання Оглядача Файлів.

Після увімкнення цієї опції в Уподобаннях Користувача ви можете задати винятки її застосування для файлів із зазначених тек, інакше типова конфігурація буде довіряти всім шляхам, окрім типової теки для звантажень.

Стяг Авто Виконання Скриптів Python.

Командний Рядок¶

Ви можете забажати здійснювати пакетне зображування або деякі інші завдання з командного рядка -- виконуючи Blender без інтерфейс.

У такому випадку, Уподобання Користувача (User Preferences) все одно будуть використовуватися і вам слід змінювати стан цієї опції через командний рядок:

  • Умикання здійснюється через додання до команди виконання Blender'а аргументу -y або --enable-autoexec
  • Вимикання здійснюється через додання до команди виконання Blender'а аргументу -Y or --disable-autoexec

Приклад¶

Зображування анімації у фоновому режими з дозволом виконання драйверів та інших скриптів:

blender --background --enable-autoexec my_movie.blend --render-anim

Ghi chú

Ці аргументи командного рядка можуть використовуватися для запуску звичайного примірника Blender'а та заміщувати Уподобання Користувача.
