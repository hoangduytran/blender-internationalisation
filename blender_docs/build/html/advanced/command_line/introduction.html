Вступ¶

Вікно консолі Console Window, також відоме як «Термінал» -- Terminal -- це текстове вікно операційної системи, що показує повідомлення про операції, стан та внутрішні помилки Blender'а.

Випадки Використання:

  • Для автоматизації та пакетної обробки, які вимагають запуску Blender'а з різними аргументами.
  • Для розробки на Python, щоб бачити вивід команди print().
  • Якщо Blender закривається несподівано, ці повідомлення можуть вказувати на причину або помилку цього.
  • При усуненні несправностей, щоб бачити повідомлення виводу --debug.

Залежні від Платформи Інструкції¶

Linux¶

Вікно консолі Console Windows у Linux типово видиме на екрані тільки, якщо Blender вручну запущено з термінала, Blender виводить у це вікно консолі Console Window, з якого він запускається.

Залежно від укладу середовища вашої стільниці, після установлення вами Blender'а для його запуску його іконка може з'явитися на вашій стільниці або у ваше меню додасться пункт для цього. Часто ви можете редагувати таку іконку та увімкнути для неї опції "Run in terminal" -- «Виконати у терміналі».

Конфігурування меню в KDE для іконки для запуску Blender'а з терміналу.

Цей екранознімок показує запущений Blender з Терміналу Linux та результатний виведений текст у ньому:

Запуск Blender з вікна терміналу Linux.

Apple macOS¶

macOS використовує «файли» з розширенням .app, що називаються applications -- «застосунки». Ці файли -- це фактично теки, що показуються як файли у пошуковці Finder. Щоб виконати Blender, ви повинні будете визначити шлях до виконуваного файлу Blender'а всередині цієї теки, для отримання усього виводу у терміналі. Ви можете запустити термінал з Applications ‣ Utilities. Шляхом до виконуваного файлу у такій теці .app є ./blender.app/Contents/MacOS/blender.

Якщо ви маєте установленим Blender у теці Applications, то може використовуватися наступна команда:

/Applications/blender-2.79/blender.app/Contents/MacOS/blender

Запуск Blender'а з вікна консолі macOS.

Microsoft Windows¶

При запуску Blender'а на операційній системі Microsoft Windows вікно консолі Console Window (зване також як «Командний Рядок» -- Command Prompt) спершу створюється як окреме вікно на стільниці. Далі з'являється головне вікно Blender'а, а вікно консолі Console Window зникає. Для показу цієї консолі знову скористайтеся меню «Вікно > Перемкнути Системну Консоль» -- Window ‣ Toggle System Console.

Для запуску Blender'а з командного рядка вам необхідно відкрити примірник Command Prompt. Для цього натисніть WinKey-R, далі уведіть cmd; це вікдриє вікно командного рядка Command Prompt. Далі вам необхідно знайти шлях до виконуваного файла Blender'а. Якщо ви установили Blender за допомогою установника, то він може бути знайдений тут:

C:\Program Files\Blender Foundation\Blender\blender.exe

Вікно Консолі Blender'а на Microsoft Windows.

Цей екранознімок показує вікно консолі Blender'а Console Window на Microsoft Windows прямо після запуску Blender'а і далі за короткий час після відкриття файлу відповідні повідомлення.

Закриття вікна консолі Console Window Blender'а

Закриття вікна консолі Console Window також закриває Blender із втратою будь-якої незбереженої роботи.

Для вимкнення консолі без закриття Blender'а слід знову запустити команду «Перемкнути Системну Консоль» -- Toggle System Console з меню (як згадувалося вище).

Повідомлення Стану та Помилок у Вікні Консолі¶

The Blender Console Window can display many different types of status and error messages. Some messages simply inform the user what Blender is doing, but have no real impact on Blender's ability to function. Other messages can indicate serious errors that will most likely prevent Blender carrying out a particular task and may even make Blender non-responsive or shut down completely. The Blender Console Window messages can also originate internally from within the Blender code or from external sources such as Python scripts.

Загальні Повідомлення¶

  • found bundled python: (FOLDER)

    Це повідомлення вказує, що Blender зміг знайти бібліотеку :ref:`Python <scripting-index>`для інтерпретатора Python, вбудованого у пакеті Blender'а. Якщо така тека відсутня або не може бути знайдена, то цілком ймовірно станеться помилка і таке повідомлення не буде показуватися.

  • malloc returns nil()

    При виконанні Blender'ом операцій, що вимагають додаткової пам'яті (RAM), то викликається функція, що зветься «malloc» (скорочено від «memory allocate», «розподіл пам'яті»), яка намагається перерозподілити пам'ять потрібного обсягу для Blender'а. Якщо це не можливо зробити, то malloc повертає значення nil/null/0, яке вказує, стався збій виконання запиту. Якщо це трапляється, то Blender не може виконати операцію, запитану користувачем. Це здебільшого призведе до уповільнення роботи або закриття Blender'а. Якщо ви хочете уникнути браку пам'яті, ви можете установити більше пам'яті у вашій системі, скоротити кількість деталей ваших моделей або закрити інші програми і служби, які займають пам'ять, яку зміг би використати Blender.
