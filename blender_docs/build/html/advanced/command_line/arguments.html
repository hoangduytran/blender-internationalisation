Аргументи Командного Рядка¶

Blender 2.79 Використання: blender [args ...] [file] [args ...]

Опції Рендерингу¶

-b, --background
Запуск у фоні (часто використовується для зображування без інтерфейсу користувача).
-a, --render-anim
Запустити зображування кадрів від початку до кінця (включно).
-S, --scene <name>
Задати ім'я <name> активної сцени для зображування.
-f, --render-frame <frame>

Рендерити, зобразити кадр <frame> та зберегти його.

  • +<frame> початковий кадр відносно, -<frame> кінцевий кадр відносно.
  • Може також використовуватися розділений комами список кадрів (без пробілів).
  • Діапазон кадрів може бути виражений за допомогою роздільника .. між першим та останнім кадрами (включно).
-s, --frame-start <frame>
Задати початок діапазону кадрів <frame>, підтримує +/- для відносно кадрів також.
-e, --frame-end <frame>
Задати кінець діапазону кадрів <frame>, підтримує +/- для відносно кадрів також.
-j, --frame-jump <frames>
Задати кількість кадрів, які перестрибувати після кожного зображеного кадру.
-o, --render-output <path>

Задати ім'я файлу та шлях для результатів зображування. Використовуйте // на початку відносного шляху для зображування blend-файлу.

Символи # замінюються номерами кадрів та використовуються для визначення заповнення нулями розрядів номерів.

  • ani_##_test.png стає ani_01_test.png
  • test-######.png стає test-000001.png

Коли ім'я файлу не містить жодних #, то стандартно додасться суфікс #### у це ім'я файлу.

Такий номер кадру буде додано у кінець такого імені файлі, наприклад:

blender -b foobar.blend -o //render_ -F PNG -x 1 -a

//render_ стане //render_####, а кадри запишуться як //render_0001.png

-E, --engine <engine>
Визначити рушій зображування (render engine). Використовуйте -E для довідкового списку доступних рушіїв.
-t, --threads <threads>
Використати кількість потоків <threads> для зображування та інших операцій [1-64], 0 дає їх визначення головним процесором системи.

Опції Формату¶

-F, --render-format <format>

Задати формат для результату зображування. Правильними опціями є TGA RAWTGA JPEG IRIS IRIZ AVIRAW AVIJPEG PNG BMP

Формати, що можуть бути компільовані у Blender'і, доступні не на всіх системах: HDR TIFF EXR MULTILAYER MPEG FRAMESERVER CINEON DPX DDS JP2

-x, --use-extension <bool>
Задати опцію для додання розширення файлу у кінець його імені.

Опції Відтворення Анімації¶

-a <options> <file(s)>

Відтворює файл(и) <file(s)>, діє таким чином лише тоді, коли не здійснюється у фоні.

-p <sx> <sy>
Відкрити у нижньому лівому кутку по координатах <sx>, <sy>.
-m
Читати з диска (Не з буфера)
-f <fps> <fps-base>
Визначити початкову частоту кадриків за секунду (FPS).
-j <frame>
Задати крок перестрибування кадрів аргументом <frame>.
-s <frame>
Відтворити із кадру <frame>.
-e <frame>
Відтворити до кадру <frame>.

Опції Вікна¶

-w, --window-border
Форсувати відкриття з границями.
-W, --window-borderless
Форсувати відкриття без границь.
-p, --window-geometry <sx> <sy> <w> <h>
Відкрити у нижньому лівому кутку по координатах <sx>, <sy> та із шириною і висотою як <w>, <h>.
-con, --start-console
Запустити з відкритим вікном консолі (ігнорується якщо задано -b), (лише для Windows).
--no-native-pixels
Не використовувати рідний розмір пікселя, для високо роздільних показів (MacBook Retina).

Специфічні Опції Ігрового Рушія¶

-g Специфічні опції Ігрового Рушія

fixedtime
Запускати з частотою 50 герц без пропускання кадрів.
vertexarrays
Використовувати Масиви Вершин (Vertex Arrays) для зображування (зазвичай швидше).
nomipmap
Без Багаторівневого розкладання Текстур (Texture Mipmapping).
linearmipmap
Прямолінійне, лінеарне Багаторівневе Розкладання Текстур Texture Mipmapping замість Найближчого (Nearest) (стандартно).

Опції Python¶

-y, --enable-autoexec
Увімкнути автоматичне виконання скрипту Python (стандарт).
-Y, --disable-autoexec
Вимкнути автоматичне виконання скрипту Python (pydrivers & startup scripts).
-P, --python <filename>
Запустити даний файл скрипту Python.
--python-text <name>
Запустити текстовий блок даного файлу скрипту Python.
--python-expr <expression>
Запустити даний вираз як скрипт Python.
--python-console
Запустити Blender з інтерактивною консоллю.
--python-exit-code
Задати код завершення у [0..255] для виходу, якщо сталося виключення Python (лише для скриптів виконуваних з командного рядка), значення «нуль» вимикає це.
--addons
Розділений комами список додатків (без пробілів).

Опції Зневадження¶

-d, --debug

Увімкнути зневадження.

  • Вмикає виявлення помилок пам'яті
  • Вимикає захоплення мишею (для взаємодії з зневадником у деяких випадках)
  • Зберігає Python'овий sys.stdin, щоб не задавати його як «Нема» — None
--debug-value <value>
Задати значення зневадження <value> при запуску.
--debug-events
Увімкнути повідомлення зневадження для подій системи.
--debug-ffmpeg
Увімкнути повідомлення зневадження від бібліотеки FFmpeg.
--debug-handlers
Увімкнути повідомлення зневадження для оброблення подій.
--debug-libmv
Увімкнути повідомлення зневадження від бібліотеки libmv.
--debug-cycles
Увімкнути повідомлення зневадження від Cycles.
--debug-memory
Увімкнути повно охоронюване розподілення та зневадження пам'яті.
--debug-jobs
Увімкнути профілювання часу для фонових завдань.
--debug-python
Увімкнути повідомлення зневадження від Python.
--debug-depsgraph
Увімкнути повідомлення зневадження від графа залежностей.
--debug-depsgraph-no-threads
Перемкнути граф залежностей в однопотокове оцінювання.
--debug-gpumem
Увімкнути статистику пам'яті GPU у рядку стану.
--debug-wm
Увімкнути повідомлення для віконного менеджера, також виводить кожен виклик оператора.
--debug-all
Увімкнути усі повідомлення зневадження.
--debug-io
Увімкнути повідомлення зневадження для Імпорту/Експорту (collada, ...).
--debug-fpe
Увімкнути дійсночислові винятки.
--disable-crash-handler
Вимкнути обробник збою.

Різні Опції¶

--factory-startup
Skip reading the startup.blend in the user's home directory.
--env-system-datafiles
Задати змінну середовища BLENDER_SYSTEM_DATAFILES.
--env-system-scripts
Задати змінну середовища BLENDER_SYSTEM_SCRIPTS.
--env-system-python
Задати змінну середовища BLENDER_SYSTEM_PYTHON.
-nojoystick
Вимкнути підтримку джойстика.
-noglsl
Вимкнути відтінення GLSL.
-noaudio
Форсувати звукову систему як «Нема» — None.
-setaudio
Форсувати звукову систему для певного пристрою. NULL SDL OPENAL JACK.
-h, --help
Вивести цей текст довідки та вийти.
-R
Зареєструвати розширення blend-file, потім вийти (лише Windows).
-r
Тихо зареєструвати розширення blend-file, потім вийти (лише Windows).
-v, --version
Вивести номер версії Blender'а та вийти.
--
Закінчити обробку опції, наступні аргументи пропускаються без змін. Доступ через Python'овий sys.argv.

Експериментальні Функції¶

--enable-new-depsgraph
Використати новий граф залежностей.
--enable-new-basic-shader-glsl
Використати новий базовий шейдер GLSL.

Інші Опції¶

/?
Вивести цей текст довідки та вийти (лише Windows).
--debug-freestyle
Увімкнути повідомлення зневадження для FreeStyle.
--debug-gpu
Увімкнути вміст та інформацію зневадження щодо GPU для OpenGL 4.3+.
--disable-abort-handler
Вимкнути обробник припинення.
--verbose <verbose>
Задати рівень проколювання багатослівності.

Аналіз Аргументів¶

Аргументи повинні бути відокремлені пробілом, наприклад:

blender -ba test.blend

...буде ігноруватися a.

blender -b test.blend -f8

...буде ігноруватися 8, оскільки відсутній жодний пробіл між -f та значенням кадру.

Порядок Аргументів¶

Аргументи виконуються у порядку, в якому вони задані, наприклад:

blender --background test.blend --render-frame 1 --render-output '/tmp'

...не буде рендеритися у /tmp, оскільки --render-frame 1 рендериться перед тим, як задано шлях виводу.

blender --background --render-output /tmp test.blend --render-frame 1

...не буде рендеритися у /tmp, оскільки завантаження цього blend-file перезапише вивід рендера, який було задано.

blender --background test.blend --render-output /tmp --render-frame 1

...працюватиме, як очікується.

Змінні Середовища¶

BLENDER_USER_CONFIG:
 Каталог для файлів конфігурації користувача.
BLENDER_USER_SCRIPTS:
 Каталог для скриптів користувача.
BLENDER_SYSTEM_SCRIPTS:
 Каталог для розширювальних системних скриптів.
BLENDER_USER_DATAFILES:
 Каталог для файлів даних користувача (іконки, переклади, ..).
BLENDER_SYSTEM_DATAFILES:
 Каталог для розширювальних системних файлів даних.
BLENDER_SYSTEM_PYTHON:
 Каталог для системних бібліотек Python.
TEMP:Тут зберігатимуться тимчасові файли.
TMP:або $TMPDIR Тут зберігатимуться тимчасові файли.
SDL_AUDIODRIVER:
 Аудіо драйвер LibSDL - alsa, esd, dma.
PYTHONHOME:Шлях до каталогу Python, наприклад, /usr/lib/python.
