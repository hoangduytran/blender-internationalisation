Шаблони Застосунку -- Application Templates¶

Використання¶

Шаблони застосунку — це функція, що дозволяє вам визначати повторно використовувану конфігурацію, яка може бути обрана замість стандартної конфігурації, без потреби окремого іншого установлення Blender'а або перезапису ваших персональних параметрів.

Використання шаблонів з меню «Файл».

Вибір шаблону на екрані заставки.

Шаблони застосунку можна вибрати на екрані заставки або з меню «Файл» (як показано вище).

Коли у меню «Файл» відсутні шаблони, то вони і не будуть показуватися і на екрані заставки.

Нові шаблони застосунку можуть бути установлені за допомогою меню «Файл».

Якщо ви бажаєте зберегти поточний шаблон застосунку активним при запуску Blender, то збережіть ваші уподобання користувача.

Мотивація¶

У деяких випадках не достатньо записати одиничний скрипт або додаток, та очікувати, що хтось замінить його в уподобаннях користувача та запусковому файлі, установить скрипти та змінить для них розкладку клавіш.

Призначення шаблонів застосунку -- це допомога у перемиканні на власну конфігурацію без руйнування ваших наявних устав та установлення.

Це означає, що люди можуть надбудовувати власні застосунки поверх Blender, які можна буде легко розповсюджувати.

Подробиці¶

Шаблон застосунку може бути визначеним через:

Запусковий Файл -- Startup File
Стандартний файл для завантаження за допомогою цього шаблону.
Уподобання Користувача -- User Preferences

З шаблону використовуються лише певні уподобання користувача:

  • Теми.
  • Додатки.
  • Розкладки клавіш.
  • Освітлення оглядвікна.
Екран Заставки -- Splash Screen
Шаблони можуть надавати свої власні зображення екрану заставки.
Скрипти Python

Коли шаблони мають доступ до такої самої функціональності, як і будь-які інші скрипти, то їх типові операції включають:

  • Модифікування та заміну частин інтерфейсу користувача.
  • Визначення нових меню, розкладок ключів та інструментів.
  • Визначення власного шляху додатків для визначених шаблоном додатків.

Шаблони також мають свою власну конфігурацію користувача, а тому збереження запускового файлу на основі шаблону не перезаписує стандартний запусковий файл.

Розстава Тек¶

Шаблони можуть бути розміщені в одному з двох місць у межах теки scripts.

Місця шаблонів:
{BLENDER_USER_SCRIPTS}/startup/bl_app_templates_user
{BLENDER_SYSTEM_SCRIPTS}/startup/bl_app_templates_system

Конфігурація користувача зберігається у під-теці:

Без шаблону:
./config/startup.blend
./config/userpref.blend
Із шаблоном:
./config/{APP_TEMPLATE_ID}/startup.blend
./config/{APP_TEMPLATE_ID}/userpref.blend

Дивіться getting-started_installing-config-directories для подробиць про місця скриптів і конфігурацій.

Вміст Шаблону¶

Кожен з наступних файлів може використовуватися шаблонами застосунку, але це необов'язково.

startup.blend
Використовується заводський запусковий файл для цього шаблону.
userpref.blend

Використовується заводський файл уподобань користувача для цього шаблону.

(Як зазначено попередньо, це використовується тільки для піднабору уподобань).

splash.png, splash_2x.png

Екран заставки заміщує стандартну ілюстрацію Blender'а (не включаючи текст заголовка).

Він має мати розмір 501x230 або 1002x460 (що використовується для моніторів HiDPI).

__init__.py
Скрипт Python, що повинен містити функції register та unregister.

Ghi chú

Пакетні blend-файли startup.blend і userpref.blend вважаються Заводськими Уставами та ніколи не перезаписуються.

Користувач може мати свої власні установки запуску/уподобань і при використанні такого шаблону, він їх перезапише.

Початкові установки шаблону можуть бути завантажені за допомогою команди: «Завантажити Заводські Установки Шаблону» — Load Template Factory Settings з меню «Файл», таким же чином, як відновлюються загальні заводські установки командою «Завантажити Заводські Установки» Load Factory Settings.
