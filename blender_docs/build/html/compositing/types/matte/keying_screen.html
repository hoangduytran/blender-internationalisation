Вузол «Ключування Екрану» -- Keying Screen Node¶

Вузол «Ключування Екрану» -- Keying Screen Node.

Вузол «Ключування Екрану» -- Keying Screen node створює пластини для використання, як колірного орієнтира для вузлів ключування. Він генерує градієнти з відібраних кольорів на точках відстеження руху у відеокліпах. Він може використовуватися для боротьби з нерівномірними кольорами зелених екранів.

Уводи -- Inputs¶

Цей вузол немає жодних увідних роз'ємів.

Властивості¶

Відеокліп -- Movie Clip
Дає змогу вибрати блок даних кліпу, що буде використаний як увід для градієнта кольорів.
Об'єкт Простеження -- Tracking Object
Об'єкт Простеження для генерування градієнта. Ви можливо схочете створити новий об'єкт простеження на панелі «Об'єкт» -- Object у Редакторі Відеокліпів, оскільки стежки, що використовуються для градієнтів не можуть фактично вживатися для простеження камери/об'єкта. Після цього такі стежки можуть бути поміщені у місця, де мають бути відібрані кольори для градієнта. Ці стежки можуть бути простежені або переміщені вручну, так що градієнти будуть оновлюватися автоматично впродовж фільму. Стежки можуть мати зсув для легшого простеження невиразних екранів.

Виводи -- Outputs¶

Екран -- Screen
Вивід зображення градієнта.

Приклад¶

Розглянемо уклад вузлів для вилучення зеленого екрану за допомогою вузла «Ключ Кольору» -- Color Key:

Часто, освітлення є нерівномірним по тлі.

Example from the Mango Open Movie, Tears of Steel.

Так, може отриматися погана маска.

Приклад поганої маски: Деякі частини тла є безпрозорими, а деякі частини зброї на передньоплані є прозорими.

If you increase the tolerances on the keying node, it will accept more shades of green to mask out. But it may also incorrectly mask out more of the foreground.

Замість збільшення діапазону прийнятних відтінків для відмаскування, вузол «Ключування Екрану» -- Keying Screen дозволить вам вирізнити відтінок зеленого (або іншого кольору), що використовується різними частинами зображення.

Розпочнемо у Редакторі Відеокліпів -- Movie Clip Editor. Відкриємо боковини, праву «Властивості» -- Properties Region та ліву «Полиця Інструментів» -- Tool Shelf для показу конфігурації простеження. Стежки, що використовуються для градієнтів, не корисні для обчислення камери, оскільки вони не відстежуються добре. Тому, створимо новий об'єкт для простеження на панелі «Об'єкти» -- Objects правої боковини. Розмістимо мітки простеження на кліпі для відбору різних частин тла.

These tracks may be tracked or moved manually, so gradients can be updated over time. If the marker is not enabled for a frame, it will not be used creating the gradient. (Such as the red-colored marker on the arm in the screenshot above)

Після створення стежок, додамо даний вузол у ваш уклад компонування, та виберемо цей об'єкт простеження, щоб він використовувався для тла.

Конфігурація вузлів з пластиною градієнта, генерованою вузлом «Ключування Екрану» -- Keying Screen, подана на увід Ключовий Колір вузла «Ключ Кольору».

Пластина градієнта генерована вузлом «Ключування Екрану» -- Keying Screen.

Результатне зображення тепер має кращу маску.
