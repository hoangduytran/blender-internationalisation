Вузол «Розфокус» -- Defocus Node¶

Вузол «Розфокус» -- Defocus Node.

Вузол «Розфокус» -- Defocus Node розмиває ділянки зображення, базуючись на уводі карти/маски.

Він типово використовується для емуляції глибини різкості (DOF) за допомогою методу після-обробки з уводом Z-буфера. Але також він дозволяє розмивати зображення, що не базуються на Z-глибині.

Уводи -- Inputs¶

Зображення -- Image
Стандартний увід зображення.
Z
Увід Z-буфера (глибини), але може бути також (сіровідтінкове) зображення, використовуване як маска, або також увід одиничного числового значення.

Властивості¶

Тип Боке -- Bokeh Type

Кількість пелюсток райдужки діафрагми віртуальної камери.

Кругове -- Circular або Disk (для емуляції ідеально круглого боке), Трикутне -- Triangular (3 пелюстки діафрагми), Квадрат -- Square (4 пелюстки), П'ятибічне -- Pentagon (5 пелюсток), Шестибічне -- Hexagon (6 пелюсток), Семибічне -- Heptagon (7 пелюсток) або Восьмибічне -- Octagon (8 пелюсток).

Кут -- Angle
Ця кнопка деактивується, якщо Тип Боке -- Bokeh Type установлено як Кругове -- Circular чи Disk. Може використовуватися для додання зсуву обертання форми Боке. Це значення є кутом, що вимірюється у градусах.
Корекція Гамми -- Gamma Correction
Застосовує корекцію гамми на зображенні перед і після його розмивання.
Д-число -- F-Stop
Ця устава керує величиною локального розмиву таким же чином, як для реальної камери. Вона імітує апертуру f райдужки реального об'єктива без модифікування освітленості зображення. Стандартне значення 128 передбачає безкінечність: все знаходиться в ідеальному фокусі. Половина від цього значення буде дублювати величину розмиву. Ця кнопка деактивується, якщо вимкнена опція «Вжити Z-буфер» -- Use Z-Buffer.
Макс Розмив -- Max Blur
Це значення лімітує величину розмиву, задаючи максимальний радіус розмиву. Може використовуватися для оптимізації продуктивності. Стандартне значення «0» означає, що ліміт відсутній.
Поріг -- Threshold

Як різниця у пікселях для розмиву велика, то можуть траплятися деякі артефакти, як-от, просочення, розплив країв. Це значення керує тим, наскільки великою буде відмінність розмиву, щоб вважатися безпечною.

Mẹo

Змінюйте це значення тільки, якщо трапилася проблема з об'єктом у фокусі.

Передогляд -- Preview
При увімкненні використовується обмежена величина (квазі-) випадкових вибірок для зображування, рендерингу передогляду. Цей спосіб відбору вибірок включає додатковий шум, який не буде показуватися на кінцевому зображі, рендері.
Сцена -- Scene
Вибір прив'язаної сцени.
Вжити Z-Буфер -- Use Z-Buffer
Повинна бути активована при відсутності подачі даних Z-буфера, глибини на увідний роз'єм Z. Ця опція автоматично вмикається, коли на роз'єм Z подаються дані не з вузла, що може подавати дані типу «Зображення» -- Image.
Z Масштаб -- Z Scale
Ця опція активується тільки тоді, коли вимкнена опція «Вжити Z-Буфер» -- Use Z-Buffer. Коли вимкнена опція «Вжити Z-Буфер» -- Use Z-Buffer, то ця устава використовується безпосередньо для керування радіусом розмиву (подібно до устави «Д-число» -- f-Stop при використанні Z-буфера). Це параметр може використовуватися для масштабування діапазону уводу Z.

Виводи -- Outputs¶

Зображення -- Image
Стандартний вивід зображення.

Приклади¶

У цьому прикладі -- blend-file example зображення масиву куль розмите, ніби воно було зняте камерою з Д-числом у 2.8, що у результаті дає досить вузьку глибину різкості, центровану на відстані у 7.5 одиниць Blender від камери. При збільшенні відстані куль від камери вони стають розмитіше.

Приклади без вживання Z-Буфера¶

Інколи може знадобитися більше засобів керування для розмиття зображення. Наприклад, ви можете схотіти розмити тільки один об'єкт, при цьому, залишивши все інше у фокусі (чи навпаки), або ви схочете розмити усе зображення однорідно водночас. Цей вузол, тому, дозволяє вам використовувати щось інше, ніж фактичний Z буфер, в якості уводу Z. Наприклад, ви можете під'єднати вузол Image та використати інше сіровідтінкове зображення, в кому колір буде визначати наскільки в цих точках розмивати увідне зображення, де білий визначає максимальний розмив, а чорний — повна відсутність розмиву. Або, ви можете скористатися вузлом Time для однорідного розмиття зображення, де значення часу буде керувати максимумом розмиву для відповідного кадру. Він також може використовуватися для отримання можливо дещо кращого розмиву глибини різкості за допомогою сіровідтінкового зображення підробної глибини замість Z-буфера. (Типовий метод створення зображення підробної глибини різкості -- це використання лінеарної текстури Blend для усіх об'єктів у сцені або використання методу відтінювання підробної глибини «туман/імла» -- "fog/mist"). Це також має перевагу у тому, що зображення підробної глибини може мати загладжування країв -- anti-aliasing, що не можливо з реальним Z-буфером.

Тоді вимкнення опції «Вжити Z-буфер» -- Use Z-Buffer стає тоді головним засобом керування розмивом. Значення з цього уводу повинні бути відмасштабовані, оскільки зазвичай значення текстури знаходяться у лише числовому діапазоні від 0.0 до 1.0.

Устави Камери¶

Устава «Відстань» -- Distance на панелі «Глибина Різкості» -- Depth of Field для камери.

Вузол «Розфокус» -- Defocus використовує дані фактичної камери у вашій сцені, якщо вони подаються у мережу вузлом «Шари Зображу» -- Render Layers.

Для задання точки фокусування камера зараз має параметр «Відстань» -- Distance, назва якого є скороченням від «відстань поля глибини різкості» -- Depth of Field Distance. Використовуйте цей параметр камери для задання фокальної площини камери (об'єкти на Відстані Глибини Різкості від камери знаходяться у фокусі). Задайте уставу «Відстань» -- Distance у на цій панелі «Глибина Різкості» -- Depth of Field у контексті «Камера» -- Camera Редактора Властивостей при виділеній камері.

Для зроблення фокальної точки видимою увімкніть опцію «Ліміти» -- Limits на панелі «Показ» -- Display у контексті Камера і точка фокусування стане видимою у вигляді жовтого хрестика на кінці напрямної лінії огляду камери.

Поради¶

Передогляд -- Preview
Загалом, використовуйте режим передогляду Preview для бажаної зміни параметрів і вимикайте його вже тільки для фінального ренрерингу. Цей вузол напружено обчислюється комп'ютером, а тому дивіться ваше вікно консолі, й воно надасть вам інфо про стан, оскільки у ньому показується обчислення рендерингу кожного рядка розгортки.
Артефакти на краях
Для мінімізації артефактів намагайтесь укладати вашу сцену так, щоб відмінності у відстаннях між двома об'єктами, що можуть у певний момент видимо накладатися, є не надто великими.
«Перевід фокуса» або "Focus Pull"

Пам'ятайте, що це не реальна Глибина Різкості, а тільки після-обробна її імітація. Деякі речі не можуть бути зроблені так, щоб не було жодних проблем з реальною Глибиною Різкості взагалі. Типовий приклад -- сцена з певним об'єктом, що знаходиться дуже близько до камери, а камера фокусується на певній точці далеко позаду нього. У реальному світі, використовуючи невелику глибину різкості, не є неможливим, щоб ближні об'єкти стали повністю невидимими, фактично дозволяючи камері бачити за ними. Голлівудські кінематографісти використовують цю візуальну характеристику для досягнення популярного ефекту «переводу фокусу» -- "focus pull", де фокус зсувається з ближнього на далекий об'єкт, таким чином, що «інший» об'єкт все-таки зникає. Ну, це просто не можливо зробити за допомогою після-обробного методу за одиничний прохід. Якщо ви реально хочете досягти цього ефекту на досить задовільному рівні, то ниже описано, як:

  • Розділіть вашу сцену на «ближні» і «дальні» об'єкти та відзображте, рендерте їх у два проходи.
  • Тепер, скомбінуйте ці два результати, для кожного з них додайте свій вузол «Розфокус» -- Defocus, що керується вузлом «Час» -- Time, але для одного з потоків інвертуйте дані. (наприклад, скористайтесь вузлом «Розкладка Значень» -- "Map Value", на якому задайте для устави «Розмір» -- Size значення -1). Таким чином, в одному потоці розфокус буде зростати, а в іншому -- зменшуватися, з однаковою частотою, створюючи плавний перехід.
Ступінчатість при Низьких Значеннях Д-числа

При дуже низьких значеннях, менших за 5, цей вузол буде вилучати будь-який перебір вибірок та даватиме об'єкти на Відстані Глибини Різкості дуже різкими у фокусі. Якщо такий об'єкт знаходиться на контрастному задньоплані, то це можливо призведе до видимої ступінчатості, яку перебір призначений запобігати. Якщо ви зіткнулися з цією проблемою:

  • Зробіть ваш власний перебір, рендеривши, зобразивши двічі -- у збільшеному і зменшеному розмірі, щоб суміжні пікселі розмилися.
  • Використайте вузол «Розмив» -- Blur зі значенням 2 для устав X та Y.
  • Задайте уставу «Відстань» -- Distance для Глибина Різкості зовсім малою, щоб об'єкт у фокусі ще трохи розмився найменше.
  • Скористайся вищим значенням Д-числа -- F-Stop, яке почне розмив, а потім використайте роз'єм Z, дані з якого подайте на вузол «Розкладка Значень» -- Map Value і далі на вузол «Розмив» -- Blur для покращення ефекту розмиву.
  • Перевпорядкуйте об'єкти у вашій сцені, щоб використати низько-контрастний задньоплан.
Без вживання Z-Буфера
Фінальне слово попередження, оскільки не існує способу виявляти, чи фактичний Z-буфер під'єднується до цього вузла, а тому будьте дуже обережними з перемикачем Use Z-Buffer. Якщо значення Z scale виявиться великим і ви забудете задати його назад меншим, то воно може раптово трактуватися, як значення величезного радіусу розмиву, що спричинить різке зростання часу на обчислення.
