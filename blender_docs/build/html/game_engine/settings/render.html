Устави Зображування, рендерингу -- Render Settings¶

Камера (чи камери), використовувані у рушії Blender Game, широко впливають на спосіб, яким гра рендериться та показується. Здебільшого це керуються за допомогою панелей вкладки даних об'єкта редактора Properties для камер(и), що використовуються у грі.

Є два окремих «відтворювачів, програвачів» гри для її передогляду у ході розроблення. Вбудований Відтворювач -- Embedded Player рендерить 3D сцену в інтерфейсі користувача Blender'а, використовуючи поточні перспективу та рівень зумування 3D передогляду. Окремий Відтворювач -- Standalone Player рендерить сцену з перспективи активної камери сцени або створює нове вікно стільниці чи перемикає у повноекранний режим рендерингу. Зауважте, що при виконанні рушія Game Engine в обох відтворювачах, миша та клавіатура комп'ютера захоплюються грою та стандартно курсор миші не видимий. Для виходу з гри натискайте клавішу Esc.

Рушій Рендера -- Render Engine

Переконайтеся, що рушій рендерингу установлено як Blender Game, для задання цих контролерів, інакше цей опис не буде відповідати тому, що ви бачите!

У редакторі Properties доступні сім панелей щодо рендерингу. Кожна з них розгортається та згортається трикутною кнопкою зліва. Функції кожної панелі описуються нижче.

Вбудований Відтворювач -- Embedded Player¶

Embedded Player panel.

Ця панель надає інформацію для Вбудованого Відтворювача гри, який дозволяє виконувати ігри всередині Blender'а.

Зауважте, що устави секції Resolution не залежать від розміру оглядвікна. Фактично, контролери Resolution, здається, ніби не мають взагалі ефекту. Ці роздільність та пропорція вбудованого передогляду завжди фіксуються для 3D передогляду, який поводиться, як режим кадрування Extend для окремого відтворювача, як описано нижче. Вибір у секції Framing на панелі Display немає жодного впливу на вбудований передогляд.

Старт -- Start
Запускає рушій Game Engine всередині оглядвікна передогляду Blender'а. Скорочення -- P.
Роздільність -- Resolution X/Y
Задає внутрішню роздільність X/Y для рендерингу.

Окремий Відтворювач -- Standalone Player¶

Standalone Player panel.

Ця панель надає інформацію для Окремого Відтворювача гри, який дозволяє виконувати ігри без Blender'а. Дивіться детальніше тут -- Standalone Player.

Семантика керувальників Resolution для Standalone Player відрізняється для Віконного -- Windowed та Повноекранного -- Fullscreen режимів. У Віконному режимі (стяг Fullscreen вимкнено), керувальники Resolution задають початкові розмірності вікна стільниці. Користувач може перерозмірити вікно у будь-який час, що призводить до відповідної зміни роздільності рендерингу. У Повноекранному режимі (стяг Fullscreen увімкнено), керувальники задають внутрішню роздільність рендерингу. Фактична роздільність показу буде краще вписуватися залежно від устаткування користувача. В обох режимах ці пропорція/обрізання/масштабування визначаються вибором у секції Framing панелі Display.

Щодо режиму Fullscreen, то важливо пам'ятати, що устави Resolution у режимі Fullscreen є лише натяками для операційної системи. Кожна комбінація показу та монітора буде мати різний набір роздільностей, який вони здатні показувати; тому може бути мало впевненості, що усі кінцеві користувачі будуть фактично мати пропоновану вами роздільність; допоки ви не виберете одну з найбільш стандартних роздільностей (наприклад, 800x600 або 1024x768). Якщо ви наполягаєте на використанні вищих роздільностей, то ви можете чітко вказати у своїй документації, що підтримуються тільки певні роздільності. У більшості інших випадків машина користувача може обрати роздільність, яка близька до запропонованої; але результати можуть бути непередбачувані, особливо у режимі Letterbox.

Зауважте, що стяг Desktop немає жодного ефекту у Віконному режимі.

Старт -- Start
Launches the current blend-file with the Standalone Player.
Роздільність -- Resolution
X
Задає розмір вікна або роздільність повноекранного показу по X.
Y
Задає розмір вікна або роздільність повноекранного показу по Y.
Повноекранно -- Fullscreen
Вимк -- Off
Відкриває окрему гру як нове вікно.
Увім -- On
Відкриває окрему гру на повний екран.
Стільниця -- Desktop
Вимк -- Off
Пробує дотримуватися значень Resolution, визначених вище, у режимі Fullscreen.
Увім -- On
Зберігає поточну роздільність стільниці у режимі Fullscreen.
Якість -- Quality
Вибірки Загладжування -- AA Samples
Кількість вибірок загладжування -- anti-aliasing samples для використання для MSAA (multisample anti-aliasing -- багатовибірного загладжування).
Бітова Глибина -- Bit Depth
Кількість бітів, використовуваних для представлення кольору кожного пікселя у повноекранному показі.
Частота Освіження -- Refresh Rate
Кількість кадрів за секунду повноекранного показу.

Стерео -- Stereo¶

Stereo panel.

Дає на вибір режим стерео, що буде використовуватися для захвату стерео зображень гри (і також, побічно, які стерео покази будуть використовувати для рендерингу зображень в окремому відтворювачі).

Нема -- None
Рендерить одиничні зображення без стерео.
Стерео -- Stereo
Рендерить подвійні зображення для стерео перегляду з використанням відповідного обладнання. Дивіться детальніше про доступні опції тут -- Stereo Camera.
Купол -- Dome
Забезпечує можливості для затягувального купольного середовища, в якому переглядається гра. Дивіться детальніше про доступні опції тут -- Dome Camera.

Відтінення -- Shading¶

Shading panel.

Визначає режим відтінення, що буде використано у рендерингу гри. Можливості відтінення у Blender, що використовуються для матеріалів -- Materials та текстур -- Textures, по суті є однаковими й для рушія Blender Game Engine. Однак, обмеження реально-часового показу означають, що тільки деякі з цих можливостей доступні.

Багатотекстурно -- Multitexture
Use Multi-texture shading.
GLSL
Використовує відтінення згідно GLSL (скорочено від OpenGL Shading Language), коли це можливо для реально-часового рендерингу зображення.

Система -- System¶

Панель System у вкладці Render редактора Properties Editor дозволяє розробнику ігор визначати опції щодо продуктивності системи, що стосуються відкидання кадрів та обмежень стосовно рендерингу кадру, клавішу для зупинки рушія Blender Game Engine та як зберігати геометрію у внутрішній пам'яті відеокарти.

Панель System у вкладці Render.

Вжити Частоту Кадрів -- Use Frame Rate
Враховує частоту кадрів, а не рендерить якомога більше кадрів. При вимкненні, це буде інформувати Blender про вільне виконання без обмежень частоти кадрів. Ця частота кадрів визначається на панелі Display у вкладці Render редактора Properties Editor. Детальніше про частоту кадрів дивіться тут -- Display panel.
Списки Показу -- Display Lists
Використовує списки показу для пришвидшення рендерингу шляхом зберігання геометрії на GPU. При увімкненні, це буде повідомляти Blender утримувати списки геометрії сітей, розміщені у пам'яті GPU. Це може допомогти пришвидшити рендеринг оглядвікна у ході гри, якщо ви маєте достатньо пам'яті GPU, для розміщення в ній геометрії та текстур.
Обмежити Оновлення Анімації -- Restrict Animation Updates
Обмежує кількість оновлень анімації до частоти кадрів FPS для цієї анімації (це краще для продуктивності, але може спричиняти проблеми для плавного відтворення). При увімкненні, це буде примушувати рушій Game Engine відкидати кадри (навіть у середині перерисування,інколи спричиняючи рвані артефакти), якщо частота кадрів рендерингу на GPU є більшою, ніж визначена на панелі -- Display panel.
Вжити Кешування Матеріалів -- Use Material Caching
Кешує матеріали у конвертері. Це швидше, але може спричиняти проблеми зі старішими одно-текстурними або багато-текстурними іграми.
Вертикальна синхронізація -- Vsync
Змінює устави Вертикальної синхронізації -- Vsync (скорочено від vertical synchronization).
Сховище -- Storage
Задає вузол сховища, використовуваний растеризатором.
Клавіша Виходу -- Exit Key
Ця кнопка визначає, натиснення якої клавіші буде давати вихід з гри.

Показ -- Display¶

Панель Display у вкладці Render редактора Properties дозволяє розробнику ігор визначати максимальну частоту кадрів анімацій, показуваних у ході виконання гри, показувати такі відомості як частота кадрів і профіль, властивості налагодження, візуалізацію геометрії фізики, попередження, визначати чи показувати курсор миші у ході виконання гри, а також задавати опції для визначення стилю кадрування гри для підгонки вікна з визначеною роздільністю.

Панель Display у вкладці Render.

Частота Кадрів Анімації -- Animation Frame Rate
Ця числова кнопка/повзунок визначає максимальну частоту кадрів, з якою гра буде виконуватися. Мінімум -- 1, максимум -- 120.
Властивості Налагодження -- Debug Properties
При увімкненні, значення будь-яких властивостей, які обрані для налагодження на панелі Game Properties, будуть показуватися з вмістом Framerate and Profile.
Частота кадрів і Профіль -- Framerate and Profile
При увімкненні, це буде показувати значення кожного з розрахунків, що Blender робить при виконанні гри, плюс властивості, позначені для налагодження, якщо опція Debug Properties також увімкнена.
Візуалізація Фізики -- Physics Visualization
Показує візуалізацію меж та взаємодій фізики (як корпуси та форми зіткнення) та їх взаємодію.
Попередження про Небажаність -- Deprecation Warnings
Кожен раз, коли розробник гри використовує небажану функціональність (яка у деяких випадках застаріла або з непридатними функціями OpenGL відеокарт), то система буде видавати попередження про таку небажану функцію.
Курсор Миші -- Mouse Cursor
Показувати чи ховати курсор миші при виконанні гри.
Кадрування -- Framing

Вибирає, як сцена буде вписувати у вікно показу або екран. Доступні три типи кадрування:

Леттербокс -- Letterbox
У Віконному режимі:
Утримує пропорцію 4:3, масштабуючи для вписування розмірності поточного вікна без обрізання, перекриваючи будь-які частини показу, що лежать поза цією пропорцією кольоровими смугами.
У Повноекранному режимі:
Поведінка цієї комбінації багато у чому залежить від устаткування користувача. Результат може бути досить непередбачуваним, особливо, коли роздільність та пропорція надто сильно відрізняються від можливостей машини. З цієї причини режим Extend повинен бути пріоритетним для повноекранних Fullscreen застосунків.
Розшир -- Extend
Цей режим веде себе майже як режим Letterbox, підтримуючи пропорцію 4:3, масштабуючи скрізь, де можливо, за винятком того, що піраміда камери розширюється або звужується кожен раз, коли необхідно заповнити будь-які частини показу, що лежать поза цією пропорцією, замість перекривання цих частини сцени кольоровими смугами, як при режимі Letterbox, або замість спотворення сцени, як при режимі Scale.
Масштаб -- Scale
У цьому режимі не робляться жодні спроби для підтримання певної пропорції. Сцена та об'єкти в ній будуть розтягуватися або стискатися для точного вписування у показ.
Смуга Кольору -- Color Bar
Це дозволить розробнику гри вибрати колір для смуг при використанні режиму кадрування Letterbox.

Запікання -- Bake¶

Панель Bake у вкладці Render редактора Properties дуже подібно до її аналога для рушія Blender Render та служить з тією ж метою. Дивіться детальніше тут -- Render Baking.

Запікання -- Bake
Запікає текстури зображення виділених об'єктів.
Режим Запікання -- Bake Mode

Інформація відтінення для запікання у зображення.

Повний Зображ -- Full Render
Запікає усі матеріали, текстури та освітлення, за винятком бліковості (specularity) та підповерхневого розсіювання (SSS).
Загорода Оточення -- Ambient Occlusion
Запікає загороду оточення, як вона визначена на панелях вкладки «Світ» -- World. Ігнорує усі освітлювачі у сцені.
Тіні -- Shadows
Запікає тіні та освітлення.
Нормалі -- Normals
Запікає дотичні та з простору камери нормалі (серед багатьох інших) у зображення RGB.
Текстури -- Textures
Запікає тільки кольори матеріалів та текстур без відтінення.
Зміщення -- Зміщення
Подібно до запікання карт нормалей, карти зміщення можуть також бути запечені з високо роздільного об'єкта та розкладені на низько роздільний об'єкт, використовуючи опцію «Виділене на Активне» -- Selected to Active.
Похідна -- Derivative
Запікає карту похідних.
Кольори Вершин -- Vertex Colors
Запікає кольори вершин.
Емісія -- Emission
Запікає колір емітування -- Emit або сяяння -- Glow матеріалу.
Альфа -- Alpha
Запікає значення альфа або, іншими словами, прозорість матеріалу.
Інтенсивність Дзеркалення -- Mirror Intensity
Запікає значення інтенсивності дзеркалення.
Кольори Дзеркалення -- Mirror Colors
Запікає кольори дзеркалення.
Інтенсивність Блікування -- Specular Intensity
Запікає значення інтенсивності блікування.
Кольори Блікування -- Specular Colors
Запікає кольори блікування.
Запекти з Багатороздільності -- Bake from Multiresolution
Bake directly from a multi-resolution object.
Нормалізовано -- Normalized
У режимі Displacement:
Нормалізує відстань.
У режимі Ambient Occlusion:
Вмикає нормалізацію, використовуючи устави матеріалу.
Простір Нормалей -- Normal Space

Нормалі можуть бути запечені у різних просторах:

Простір Камери -- Camera space
Стандартний метод.
Простір Світу -- World space
Нормалі у координатах світу, залежно від трансформацій та деформацій об'єкта.
Простір Об'єкта -- Object space
Нормалі у координатах об'єкта, незалежно від трансформацій об'єкта, але залежно від його деформацій.
Простір, дотичний до об'єкта -- Tangent space
Нормалі у координатах дотичного простору, незалежно від трансформацій та деформацій об'єкта. Це є новий стандарт, та є правильним вибором у більшості випадків, оскільки далі така карта нормалей може використовуватися також для анімованих об'єктів.
Запекти у Кольори Вершин -- Bake to Vertex Color
Запікає у кольори вершин замість в UV-розкладене зображення.
Зчищення -- Clear
Якщо увімкнено, то зчищає зображення в обраний фоновий колір (стандартним є чорний) перед запіканням зобража.
Припуск -- Margin
Запечений результат розширюється на цю кількість пікселів поза границею кожного «острова» UV для пом'якшення швів у текстурі.
Виділене на Активне -- Select to Active

Запікає відтінення поверхні виділених об'єктів на активний об'єкт.

Відстань -- Distance
Максимальна відстань в одиницях Blender'а від активного об'єкта до іншого об'єкта.
Відхил -- Bias
Відхил в одиницях Blender'а у напрямку граней, що далі від цього об'єкта.
Розділення -- Split

Метод, використовуваний для розділення чотирибічників на два трикутники для запікання.

Фіксовано -- Fixed
Розділяє чотирибічники передбачувано (0,1,2)(0,2,3).
Фіксовано Альтернативно -- Fixed Alternate
Розділяє чотирибічники передбачувано (1,2,3)(1,3,0).
Автоматично -- Automatic
Розділяє чотирибічники так, щоб у результаті запікання було найменше спотворення.
Масштаб Користувача -- User Scale
Застосовує користувацький масштаб для карта похідних замість нормалізування у стандарт (0.1).
