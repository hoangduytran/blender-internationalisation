Устави Об'єкта -- Object Settings¶

Рівень Деталізації -- Level of Detail¶

При створенні візуальних активів часто бажано мати високу величину детальності в активі для близького перегляду. Однак, ця висока величина детальності є марною, якщо об'єкт переглядається на відстані, та знижує продуктивність сцени. Для вирішення цього актив може підмінюватися на певних відстанях перегляду. Це зазвичай називають як «системою рівнів деталізації» -- level of detail system. Кожен візуальний крок активу відомий як рівень деталізації. Рівні деталізації найкраще доцільно використовувати, коли ви маєте велику сцену, де певні об'єкти можуть переглядатися як зблизька, так і на відстані.

Устави -- Settings¶

Ghi chú

Модифікатори на Об'єктах Рівнів Деталізації

Будь-які об'єкти рівнів деталізації, що мають модифікатори, не показуються коректно у рушії Game Engine. Ви повинні будете застосувати всі модифікатори для об'єктів рівнів деталізації, щоб вони показувалися коректно. Виправлення цього розглядається.

Устави рівня деталізації можна знайти серед устав Об'єкта, коли як рендерер установлено рушій Blender Game. На панелі «Рівень Деталізації» -- Levels of Detail є кнопка для додання нового рівня деталізації для поточного об'єкта. Устави для кожного рівня деталізації показуються в його власній рамці. Винятком з цього є базовий рівень деталізації. Це автоматичний уклад при установленні для поточного об'єкта устави відстані як 0. Для вилучення рівень деталізації клацайте на кнопці Х у верхньому правому кутку рамки для рівня, який має бути вилучений.

Об'єкт -- Object
Об'єкт для використання для цього рівня деталізації.
Відстань -- Distance
Відстань, на якій цей рівень деталізації стає видимим.
Вжити Сіть
Коли ця опція увімкнена, то сіть з цього об'єкта рівня деталізації використовується, допоки нижчий рівень деталізації не замістить її.
Вжити Матеріал -- Use Material
Коли ця опція увімкнена, то матеріал з цього об'єкта рівня деталізації використовується, допоки нижчий рівень деталізації не замістить його.

Інструменти -- Tools¶

Деякі інструменти рівня деталізації для легшого управління та створення можна знайти у меню вибірника біля кнопки додання на панелі Levels of Detail.

Задати За Іменем -- Set By Name¶

Шукає сцену за конкретно іменованими об'єктами та пробує установити їх як рівень деталізації на поточно виділеному об'єкті. Виділений об'єкт повинен бути на базовому рівні деталізації (наприклад, LOD0). Це може бути корисним для швидкого укладання рівня деталізації на імпортованих активах. Щоб використовувати цей інструмент іменування у вас повинно бути послідовним та кожен рівень повинен мати у назві префікс або суфікс "lodx", де x -- це рівень, для якого призначається цей об'єкт. Регістр символів слова "lod" повинен також бути послідовним для усіх об'єктів. Нижче наведені кілька прикладів імен, які цей інструмент буде розпізнавати.

  • LOD0_Box, LOD1_Box, LOD2_Box
  • Box.lod0, Box.lod1, Box.lod2
  • LoD0box, LoD1box, LoD2box

Генерування -- Generate¶

Цей інструмент генерує та укладає рівень деталізації на основі виділеного об'єкта. Генерування робить за допомогою модифікатора «Спрощення» -- Decimate Modifier. Генерування не застосовує модифікатор, щоб дозволити подальші зміни його устав. Генеровані об'єкти автоматично іменуються на основі назви рівня, для якого вони генеруються. Нижче деякі устави для цього оператора (доступні у нижній частині Полиці Інструмента 3D Огляду після його виклику).

Рахунок -- Count
Бажана кількість рівнів для генерування. Цей оператор створює Count-1 нових об'єктів.
Розмір Цілі -- Target Size
Устава пропорції для модифікатора Decimate на останньому рівні деталізації. Устави пропорції для інших рівнів визначаються шляхом лінеарної, прямолінійної інтерполяції.
Пакування у Групу -- Package into Group
З увімкненням цієї устави оператор здійснює деякі додаткові завдання для зроблення цього активу готовим для легкого прив'язування у новий файл. Базовий об'єкт та усі його рівні деталізації поміщаються у групу на основі імені базового об'єкта. Рівні, інші ніж базовий, ховаються як для оглядвікна, так і рендерингу. Це спрощує вигляд системи та не впливає на вигляд базового об'єкта. Нарешті, усі рівні приріднюються до базового об'єкта для вилучення плутанини в Outliner.

Зчистити Усе -- Clear All¶

Зчищає устави рівнів деталізації з поточного об'єкта.
