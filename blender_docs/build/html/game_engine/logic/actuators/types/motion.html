Актуатор «Рух» -- Motion Actuator¶

Актуатор Motion установлює об'єкт у рух. Існує два режими операції, простий -- Simple або серво -- Servo, в яких об'єкт може телепортуватися та обертатися або динамічно переміщатися.

Властивості -- Properties¶

Тип Руху -- Motion Type

Визначає тип руху:

Простий Рух -- Simple Motion
Застосовує зміну в локації та/або обертанні безпосередньо.
Серво Керування -- Servo Control
Задає цільову швидкість, а також, як швидко досягається ця швидкість.

Простий Рух -- Simple Motion¶

Simple Motion дає керування над позицією та скорістю, але робить це як миттєве зміщення; об'єкт ніколи не проходить жодної координати між стартовою та кінцевою позиціями. Це може втручатися у фізичну імітацію інших об'єктів та може спричиняти проходження об'єкта крізь інший об'єкт. Актуатор Servo Control не страждає цим, оскільки він продукує фізично коректні скорості та залишає оновлення позиції до фізичної імітації.

Актуатор Motion з типом Simple Motion.

Loc -- Локація
Об'єкт стрибає на кількість заданих тут одиниць Blender'а, кожен раз при отриманні пульсу.
Rot -- Обертання
Об'єкт обертається на визначену тут величину, кожен раз при отриманні пульсу.
L -- Локаль
Перемикає визначення координат як Глобальні (позначено сірим) або Локальні (позначено білим).

Серво Керування -- Servo Control¶

Актуатор типу Servo Control впливає на скорість об'єкта гри шляхом застосування сил, що дають у результаті коректну поведінку при стиканні з іншими об'єктами, що керуються імітацією фізики. Величина необхідної сили визначається пропорціонально-інтегрально-диференціальним контролером -- PID controller, типом контролера, що часто використовується у системах керування. Цей актуатор впливає тільки на позиційну скорість; він не керує обертання взагалі та керує позицією тільки опосередковано.

Керування позицією не є необхідним у цьому відношенні; для гравця залишається рухати об'єкт за допомогою контролерів типу «напрямок» (таких як клавіші WASD у шутері від першої особи). У такому сценарії кожен сенсор клавіші напрямку повинен бути прикріплений до різних актуаторів з типом Servo Control із заданням для них різних цільових скоростей.

Mẹo

Для використання актуатора з типом Servo Control необхідно задати Тип Фізики об'єкта як "Dynamic" або "Rigid Body" та позначити об'єкт як "Actor" на тій же панелі. Цей актуатор не працює з типом фізики Character.

Серво Керування -- Servo Control.

Об'єкт Орієнтира -- Reference Object
Визначає об'єкт, який цей актуатор використовує як орієнтир для скорості. При заданні він буде використовувати скорість відносно такого об'єкта, а не абсолютну (тобто, відносно світу) скорість. Використовуйте це для об'єкта гравця на рухомій платформі.
Лінеарна Скорість -- Linear Velocity
Цільова лінеарна скорість для об'єкта.
L -- Локаль
Визначає, чи Лінеарна Скорість визначається у Локальних (кнопка натиснута) або Глобальних (кнопка відпущена) координатах.
Ліміти сили по X, Y, Z
Задає мінімальні та максимальні ліміти для сили, застосованої до об'єкта. Якщо вимкнуто (тобто кнопки X, Y або Z натиснуті), то застосована сила не лімітується.

Наступні три коефіцієнти визначають відповідь на помилку скорості, яка є різницею між цільовою скорістю та фактичною скорістю об'єкта.

Пропорційний Коефіцієнт -- Proportional Coefficient
Це керує реакцією, пропорційно до помилки скорості. Малі значення спричиняють плавні (але можливо надто повільні) зміни у скорості. Вищі значення спричиняють швидкі зміни, але можуть давати перевищення.
Інтегральний Коефіцієнт -- Integral Coefficient
Це керує реакцією відносно суми помилок. Використання тільки Пропорційного компонента дає систематичну помилку скорості, якщо там присутнє тертя; для продукування сили, що компенсує це тертя, необхідна деяка дельта скорості. Використання Інтегрального компонента придушує цей ефект (цільова скорість досягається усередненням), але може створювати коливання; цей контролер буде пришвидшувати компенсацію помилки початкової скорості. Для запобігання коливанню Пропорційний компонент повинен використовуватися разом з Інтегральним компонентом (Пропорційний компонент гасить це керування) Ось чому графічний інтерфейс користувача задає Пропорційний Коефіцієнт систематично, коли ви змінюєте Інтегральний Коефіцієнт.
Коефіцієнт Похідної -- Derivative Coefficient
Задає Коефіцієнт Похідної. Це гасить прискорення, коли цільова скорість майже досягнута.
