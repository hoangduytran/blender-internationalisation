Стани -- States¶

У BGE об'єкт може мати різні «стани». У будь-який момент при відтворенні гри поточний стан об'єкта визначається його поведінкою. Наприклад, персонаж у вашій грі може мати стани, що представляють пробудження, спання або мертвість. У будь-який момент їх поведінка у відповідь на гучний грюк буде залежати від їх поточного стану; вони можуть припасти вниз (насторожитися); пробудитися (замліти) або нічого робити (мертвість).

Як працюють стани¶

Стани установлюються та використовуються через контролери; зауважте, що тільки контролери, не актуатори та сенсори, безпосередньо скеровуються системою станів. Кожен об'єкт має ряд станів (до 30; стандартно = 1) і може знаходитися тільки в одному стані у певний момент часу. Контролер повинен завжди визначати стан, яким він буде оперувати -- це дасть тільки вивідний пульс, якщо а) умови його логіки відповідні, і б) об'єкт поточно знаходиться у визначеному Стані. Стани укладаються та редагуються в уставах Контролера об'єкта (детальніше дивіться нижче).

Mẹo

Устави стану є автоматичними у простих іграх. Стандартно, кількість станів для кожного об'єкта є 1 та усі контролери установлюються для використання Стану 1. Тому, якщо гра не потребує одночасно кількох станів, все буде працювати без явного задання станів -- вам не потрібно турбуватися про стани взагалі.

Один з таким же іменем актуаторів -- State може задавати або скидати біти Стану об'єкта і, звідси, дозволяти реакцію об'єкта на сигнал сенсора залежно від цього об'єкта поточного стану. Тому, у згадуваному вище прикладі, актор буде мати ряд контролерів, під'єднаних до сенсора «гучний грюк», для кожного зі станів «пробудження», «завмирання» або «мертвості». Вони будуть оперувати різними актуаторами залежно від поточного стану актора і деякі з цих актуаторів можуть перемикати стан актора під відповідні умови.

Редагування станів¶

State Panel button.

Стани задаються та редагуються за допомогою стовпця Контролер (центр) панелі Game Logic. Щоб бачити панель Стану клацніть на кнопці панелі Стану. Ця панель показує дві області для кожного із 30 доступних станів; видимі стани -- Visible та початкові -- Initial стані (дивіться нижче). Установлення системи Станів для гри здійснюється через вибір відповідного стану для кожного контролера у логіці об'єкта.

Показ логіки стану об'єкта та інше господарство здійснюється через панель Стану для об'єкта, що вмикається та вимикається цією кнопкою. Панель ділиться на дві половини -- Visible та Initial.

State Panel visible.

Видимі Стани -- Visible States¶

In the Visible area, each of the 30 available states is represented by a light-gray square. This panel shows what logic is visible for the logic brick displayed for the object. At the right is the All button; if clicked, then all the object's logic bricks are displayed (this is a toggle), and all State Panel squares are light gray. Otherwise, individual states can be clicked to make their logic visible. (Note that you can click more than one square). Clicking the square again deselects the state.

States for the object that are in use (i.e. the object has controllers which operate in that state) have dots in them, and squares are dark gray if these controllers are shown in the Game Logic display. The display of their connected sensors and actuators can also be controlled if the State buttons at the head of their columns are ticked.

State Panel initial.

Початковий Стан -- Initial State¶

В області «Початкове» -- Initial кожен з 30 доступних станів також представлено світло-сірим квадратиком. Один з цих станів може буде увімкнено як стан, в якому об'єкт починає при запуску гри.

Справа є кнопка ; при увімкненні його та стяга Game ‣ Show Debug Properties поточний стан об'єкта показується у верхньому лівому кутку екрану при виконанні гри.
