Вступ¶

The Blender Game Engine (BGE) is Blender's tool for real-time projects, from architectural visualizations and simulations to games.

Попередження, перед тим, як почати будь-який великий або серйозний проект з Blender Game Engine, ви повинні врахувати, що він поточно не дуже підтримується та те, що існують плани для його перенацілення й переробки коду, які принаймні призведуть до втрати сумісності. Для отримання додаткової інформації ви повинні зв'язатися з розробниками за допомогою списку розсилки чи IRC та прочитати цю дорожню карту розвитку -- development roadmap.

Випадки використання та зразки ігор¶

Blender has its own built-in Game Engine that allows you to create interactive 3D applications or simulations. The major difference between Game Engine and the conventional Blender system is in the rendering process. In the normal Blender engine, images and animations are built off-line -- once rendered they cannot be modified. Conversely, the Blender Game Engine renders scenes continuously in real-time, and incorporates facilities for user interaction during the rendering process.

Екранознімок з гри "Yo Frankie", продукованої за допомогою Blender Game Engine.

Рушій Ігор Blender'а відстежує ігрову петлю, яка обробляє логіку, звук, фізику та рендеринг симуляцій у послідовному порядку. Цей рушій написано на C++.

By default, the user has access to a powerful, high-level, Event Driven Logic Editor which is comprised of a series of specialized components called "Logic Bricks". The Logic Editor provides deep interaction with the simulation, and its functionality can be extended through Python scripting. It is designed to abstract the complex engine features into a simple user interface, which does not require experience with Programming. An overview of the Logic Editor can be found in the Game Logic Screen Layout.

Рушій Ігор є тісно інтегрованим з базою наявного коду Blender'а, що дозволяє швидко переходити між набором функцій традиційного моделювання та специфічною для ігор функціональністю, що надаються цією програмою. У цьому сенсі Рушій Ігор може ефективно використовуватися в усіх областях розробки ігор, від прототипів до фінального випуску.

Рушій Ігор може імітувати вміст у межах Blender'а, однак, він також включає здатність експортувати двійковий виконуваний пакет у Linux, macOS та MS-Windows.

Існує ряд потужних бібліотек, переваги яких Рушій Ігор використовує:

  • Audaspace: Звукова бібліотека для керування аудіо. Використовує OpenAL чи SDL.
  • Bullet: Рушій фізики з функцією виявлення 3D зіткнень, динамікою м'якого тіла та динамікою твердого тіла.
  • Detour: A pathfinding and spatial reasoning toolkit.
  • Recast: Сучасний набір засобів мистецької навігації та конструювання сіті для ігор.

При створенні гри або симуляції в BGE існує чотири основні кроки:

  1. Створення візуальних елементів, що можуть бути рендерені. Це можуть бути 3D моделі або зображення.
  2. Увімкнення взаємодії у межах сцени, використовуючи цеглини логіки для скриптування власної поведінки та визначення того, як вона викликається (використовуючи відповідні «датчики, сенсори», такі як клавіатури або джойстики).
  3. Створення однієї (або більше) камери для надання піраміди, у межах якої рендерити сцену та модифікувати параметри для підтримки середовища, в якому буде показуватися дана гра, такого як рендеринг Стерео.
  4. Запуск гри, використовуючи внутрішній відтворювач або експортуючи виконуваний файл на відповідну платформу.
