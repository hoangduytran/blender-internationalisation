Фізика контролера автотранспорту -- Vehicle Controller Physics¶

Вступ¶

Vehicle Controller -- Контролер Автотранспорту -- це спеціальний тип фізики об'єктів, які розпізнає рушій фізики Bullet.

Він складається з типу rigid body, що представляє шасі, та набору колір, для яких установлено тип no collision. Особливо важливо підкреслити відмінності між об'єктом у рушії Game Engine, об'єктом з Логікою та для Рендера та їх представленням для рушія фізики Physics Engine.

Імітувати транспорт як справжнє тверде тіло, що є поверх також твердо-тілих коліс, з реальною системою підвіски, зроблених за допомогою шарнірів, було б надто складно та нестабільно. Автівки та інші автотранспорти є складними механічними пристроями і найбільш часто імітувати тільки те, що «діє як очікується». Vehicle Controller існує для надання окремого способу симулювання поведінки транспорту без потреби імітувати усю фізику, що дійсно відбувається у реальному світі. Він абстрагує всю складність, забезпечуючи простий інтерфейс з налаштовуваними параметрами, такими як сила підвішування, згасання та стиснення.

Як це працює¶

Підхід Bullet до контролера транспорту називається "Raycast Vehicle" (відкидальний промінь автотранспорт). Виявлення зіткнення для коліс приблизно здійснюється відкиданнями променя, а тертя шин є анізотропною моделлю тертя.

Raycast vehicle працює, відкидаючи промінь для кожного колеса. Використовуючи точку перетинання променя, ми можемо розрахувати довжину підвішування та, звідси, силу підвіски, що дає застосовується до шасі, утримуючи її від потрапляння на землю. По суті, шасі автотранспорту «пливе» уздовж цих променів.

Сила тертя розраховується для кожного колеса там, де промінь контактує із землею. Це застосовується як вперед та убік спрямована сила.

Ви можете перевірити підхід Kester Maddock до імітації автотранспорту тут -- here. Він включає деякі загальні проблеми, обходи їх, підказки та хитрощі.

Як використовувати¶

Поточно Vehicle Controller може використовуватися тільки як примус через Python. Існують плани додання його в інтерфейс.

Укладання¶

Ви повинні мати тіло, що діє як шасі, та установити для нього тип фізики 'Rigid Body'.

Колеса повинні бути окремими об'єктами з типом фізики 'No Collision'. Vehicle Controller буде розраховувати зіткнення як промені, а тому, якщо ви установите його на щось інше, то він буде розраховувати їх двічі різними способами та давати чудові результати.

Зіткнення -- Collisions¶

Циліндр -- Cylinder, зазвичай, є хорошою формою зіткнення для коліс. Для шасі форма зіткнення повинна бути грубою, як коробка -- Box. Якщо автотранспорт є дуже складним, то ви повинні розділити його на простіші об'єкти та приріднити їх (з їх формами зіткнення) до об'єкта з Vehicle Controller, щоб вони слідували за ним. Якщо ваш автотранспорт навіть має рухомі частини (зброю, боєприпаси, причепи тощо), то вони повинні також імітуватися окремо та з'єднуватися з автотранспортом через зчеплення.

Python¶

Збирання автотранспорту¶

Загальні кроки:

  1. Створити примус для автотранспорту та зберегти його ID для подальшого на нього посилання.
  2. Прикріпити колеса.
  3. Задати параметри для коліс: вплив, тугість, згасання, стиснення та тертя.
  4. Ініціювати змінні.

Ви можете бачити приклад у файлі нижче.

Керування автотранспортом¶

Це робиться у двох частинах та повинно моделюватися відповідно до бажаної поведінки. Ви повинні подумати про ігровий процес та відшукати відповідні функції для увідних даних. Наприклад, чи може автотранспорт розвертатися? стрибати? зноситися? чи повертає він повільно? Скільки часу потрібно для гальмування чи набирання повної швидкості? Перша частина -- це реакція на клавіші. Кожен раз, коли гравець натискає клавішу, ви повинні для цього задати відповідне значення, наприклад, прискорення. Приклад:

if key[0] == events.UPARROWKEY:
    logic.car["force"] = -15.0
elif key[0] == events.RIGHTARROWKEY:
    logic.car["steer"] -= 0.05

Друга частина -- це обчислення переміщення відповідно до функцій:

## apply engine force ##
for i in range(0, totalWheels):
    vehicle.applyEngineForce(logic.car["force"],i)
...
## slowly ease off gas and center steering ##
logic.car["steer"] *= 0.6
logic.car["force"] *= 0.9

Обидві повинні виконуватися у кожному кадрі.
