Фізика «Сенсор» -- Sensor Physics¶

Об'єкти з цим типом виявляють об'єкти з Static та Dynamic, але не об'єкти з іншими зіткненнями. Об'єкт з типом Sensor подібний до об'єктів фізики, що є під сенсорами Near та Radar. Як об'єкт з Near та Radar, цей об'єкт є:

  • Статичним та привидом.
  • Стандартно невидимим.
  • Завжди активним для гарантування коректного виявлення зіткнення.
  • Здатним для виявлення обох -- статичних і динамічних об'єктів.
  • Ігнорує зіткнення зі своїм предком.
  • Здатним до широкофазного фільтрування на основі:
    • Опції Actor: зіштовхуваний з ним об'єкт повинен мати увімкнений стяг Actor, щоб бути виявленим.
    • Властивість/матеріал: як вказано у сенсорах типу Collision, прикріплених до нього.

Широкофазне фільтрування важливе з причини продуктивності: точки зіткнення будуть обчислюватися тільки для об'єктів, що проходять широкофазний фільтр.

  • Автоматично вилучається із симуляції, коли жоден із сенсорів типу Collision не є активним на ньому.

На відміну від об'єктів з Near та Radar, цей об'єкт може:

  • Приймати будь-яку форму, включаючи трикутникову сіть.
  • Бути зробленим видимим для налагодження (просто використайте актуатор Visibility).
  • Мати одночасно кілька сенсорів типу Collision, що використовують його.

Окрім цього всього, об'єкти з типом Sensor є звичайними об'єктами. Ви можете переміщувати їх вільно або приріднювати їх. При прирідненні до об'єкта Dynamic вони можуть давати просунуте керування зіткненнями для цього об'єкта.

Тип спроможності зіткнення залежить від форми:

  • Box, sphere, cylinder, cone, convex hull забезпечують виявлення об'єму.
  • Triangle mesh забезпечує виявлення поверхні, але ви можете додати деякий об'єм до поверхні, збільшивши припуск -- margin. Припуск застосовується на обох сторонах поверхні.

Підказки щодо продуктивності

  • Об'єкти із Sensor працюють краще, ніж з Near та Radar: вони зменшують синхронізацію через оптимізації графа сцени й вони можуть мати одночасно кілька сенсорів Collision на них (з різним фільтруванням властивості, наприклад).
  • Завжди надавайте перевагу простій формі (коробка, сфера) перед складними формами, там де це можливо.
  • Завжди використовуйте широкофазове фільтрування (уникайте сенсора типу Collision з порожніми властивістю/матеріалом).
  • Використовуйте сенсор Collision тільки там, де це потрібно. Коли жоден сенсор Collision не активний на об'єкті з Sensor, то цей об'єкт вилучається з імітації та не витрачає жодний ресурс CPU.

Відомі обмеження

  • При виконанні Blender'а у режимі налагодження ви побачите один рядок попередження у консолі:

    warning btCollisionDispatcher::needsCollision: static-static collision!
In release mode this message is not printed.

  • Припуск зіткнення не впливає на форми Sphere, Cone та Cylinder.
