Фізика Світу -- World Physics¶

BGE World Physics panel.

Панель «Фізика» -- Physics¶

Панель Physics для світу сцени, розміщена у вкладці Scene чи World, визначає тип фізичних правил, що управляють сценою Рушія Ігор та використовуваним значенням гравітації. На основі обраного рушія фізики в імітаціях фізики у рушії Game Engine, Blender буде автоматично рухати акторів Actors у напрямку (-Z) вниз. Після того, як ви впорядкуєте акторів і вони рухатимуться, як ви бажаєте, ви можете далі запекти цей обчислений рух у ключкадри (дивіться детальніше тут -- Копання Глибше -- Digging Deeper).

Рушій Фізики -- Physics Engine

Задає тип рушія фізики для використання.

Bullet
Стандартний рушій фізики, в активному розробленні. Він обробляє виявлення переміщення та зіткнення. Речі, що стикаються, передають момент зіштовхнутому об'єкту.
Нема -- None
Жодна фізика не використовується. На речі не впливає гравітація і вони можуть літати по віртуальному просторі. Об'єкти у русі залишаються у цьому русі.
Гравітація -- Gravity
Гравітаційне прискорення, m.s-2 (в одиницях метрів за секунду у квадраті), цього світу. Кожен об'єкт, що є актором має масу та повзунок розміру. У поєднанні з частотою кадрів (дивіться підрозділ Render), Blender використовує це інфо для розрахування, як швидко об'єкт повинен прискоритися вниз.
Відкидання Загороди: Роздільність -- Occlusion Culling: Resolution
Розмір буфера відкидання загороди у пікселях, використовуйте вище значення для кращої точності (повільніше). Оптимізований Bullet DBVT для перегляду піраміди та відкидання загороди стандартно активується внутрішньо.
Кроки Фізики -- Physics Steps
Макс, максимум -- Max
Sets the maximum number of physics steps per game frame if graphics slow down the game. higher value allows physics to keep up with real-time.
Підкроки -- Substeps
Sets the number of simulation sub-steps per physics time step. Higher value give better physics precision.
FPS -- Кадри за Секунду
Задає номінальну кількість кадрів гри за секунду. Фіксований крок часу фізики = 1/fps, незалежно від фактичної частоти кадрів.
Кроки Логіки -- Logic Steps
Задає максимальну кількість кадрів логіки за кадр гри, якщо графіка уповільнює гру, вище значення дозволяє кращу синхронізацію з фізикою.
Деактивація Фізики -- Physics Deactivation

Ці устави керують порогом, за яким фізика дезактивується. Ці устави допомагають скоротити обробку, витрачену на імітацію Фізики у ході гри.

Лінеарний Поріг -- Linear Threshold
Ліміт швидкості, нижче якого тверде тіло впадатиме у сплячку (перестане рухатися), якщо воно залишається нижче цих лімітів на час, що дорівнює або довший ніж, час дезактивації (спання вимикається, коли час дезактивації задається у 0).
Кутовий Поріг -- Angular Threshold
Подібно, як лінеарний поріг, але для ліміту обертання (у радіани/сек)
Час -- Time
Величина часу, упродовж якого об'єкт повинен мати рух, нижче цих порогів для фізики, щоб бути вимкненим (0.0 вимикає дезактивацію фізики).

Максимальна довжина контуру краю.

Вершини на Полігон -- Vertices Per Poly

Імітація Перешкоди -- Obstacle Simulation¶

Імітація, використовувана для уникання перешкоди у рушії Game Engine, на основі принципу RVO (Reciprocal Velocity Obstacles -- Взаємні Перешкоди Швидкості). Мета тут -- запобігти, що один чи кілька акторів зіштовхнулися з перешкодами.

See Pathfinding and steering behaviors for more details.

Тип -- Type
Нема -- None
Імітація перешкоди вимкнена, актори не у змозі уникати перешкоди.
RVO (cells -- комірки)
Імітація перешкоди базується на методі RVO method з відбором комірок.
RVO (rays -- промені)
Імітація перешкоди базується на методі RVO method з відбором променів.
Висота Рівня -- Level height
Максимальна різниця у висотах перешкод для вмикання їх взаємодії. Використовується для визначення мінімального припуску між перешкодами за висотою, коли вони обробляються як так, що розташовані одна над іншою, тобто вони не впливають одна на іншу.
Візуалізація -- Visualization
Вмикає візуалізацію налагодження для імітації перешкоди.
