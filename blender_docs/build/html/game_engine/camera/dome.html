Купольна Камера -- Dome Camera¶

Ця функціональна можливість дозволяє митцям візуалізувати їх інтерактивні проекти у межах вражального купольного середовища. Щоб зробити його розширюваним інструментом, ми підтримуємо Повний купол, Урізані (спереду і ззаду) куполи, Планетарії та куполи зі сферичними дзеркалами.

Mẹo

The Dome camera uses a multi-pass texture algorithm as developed by Paul Bourke and was implemented by Dalai Felinto with sponsorship from SAT -- Society for Arts and Technology within the SAT Metalab immersion research program, that involves rendering the scene four times and placing the subsequent images onto a mesh designed especially such that the result, when viewed with an orthographic camera, is a fisheye projection.

Ghi chú

Не забувайте використовувати Blender у «повноекранному режимі», щоб максимально використати ваш проектор.

To accomplish that launch Blender with the command-line argument -W. Also to get away of the top menu on Blender try to join all areas (buttons, 3D View, text, etc.) in a single one. Otherwise if you only maximize it (Ctrl-Up) you cannot get the whole screen free to run your game (the top menu bar takes about 20 pixels).

Устави Купольної Камери¶

Тип Купола -- Dome Type
Це меню дозволяє вам обрати, який тип купольної камери використовувати. Вони наведені нижче з відповідними їх уставами.
  • Режим Риб'ячого ока -- Fisheye Mode
  • Режим Урізаного Спереду Купола -- Front-Truncated Dome Mode
  • Режим Урізаного Ззаду Купола -- Rear-Truncated Dome Mode
  • Режим Кубічної Карти -- Cube Map Mode
  • Режим Сферичної Панорами -- Spherical Panoramic Mode

Доступні устави камери змінюються залежно від обраного Типу Купола:

Роздільність -- Resolution
Задає роздільність Буфера. Зменшення цього значення збільшує швидкість, але зменшує якість.
Теселяція -- Tessellation
This is the tessellation level of the mesh (not available in Cube Map mode).
Кут -- Angle
Задає поле зору купола у градусах, від 90 до 250. (Доступно у режимах Fisheye та обох Truncated).
Відхил -- Tilt
Set the camera rotation in the horizontal axis (available in Fisheye and Truncated modes).
Дані Жолоблення Сіті -- Warp Data Mesh
Використовує власний файл даних жолоблення сіті.

Режим Риб'ячого ока -- Fisheye Mode¶

Ортогональний огляд з ефектом «Риб'яче око» від 90º до 250º градусів.

  • З 90º до 180º ми використовуємо чотири рендери.
  • З 181º до 250º ми використовуємо п'ять рендерів.

Режим Риб'ячого ока.

Режим Урізаного Спереду Купола -- Front-Truncated Dome Mode¶

Розроблено для урізаних куполів, цей режим вирівнює зображення з риб'ячим оком з верхом вікна та торкаючись боків.

  • Поле зору поширюється від 90º до 250º градусів.
  • З 90º до 180º ми використовуємо чотири рендери.
  • З 181º до 250º ми використовуємо п'ять рендерів.

Режим Урізаного Спереду Купола.

Режим Урізаного Ззаду Купола -- Rear-Truncated Dome Mode¶

Розроблено для урізаних куполів, цей режим вирівнює зображення з риб'ячим оком з низом вікна та торкаючись боків.

  • Поле зору поширюється від 90º до 250º градусів.
  • З 90º до 180º ми використовуємо чотири рендери.
  • З 181º до 250º ми використовуємо п'ять рендерів.

Режим Урізаного Ззаду Купола.

Режим Кубічної Карти -- Cube Map Mode¶

Режим Cube Map може використовуватися для попереднього генерування анімованих зображень для Кубічних Карт.

Ми використовуємо шість рендерів для цього. Порядок зображень слідує формату файлу внутрішніх карт середовища EnvMap у Blender'і: - перший ряд: справа, ззаду. - Другий ряд: знизу, зверху, спереду.

Режим Карти Середовища.

Режим Сферичної Панорами -- Spherical Panoramic Mode¶

Режим повної сферичної панорами.

  • Ми використовуємо тут шість камер.
  • Нижня і верхня починають отримувати точність за допомогою Definition, задану зі значенням як 5 чи більше.

Режим Повної Сферичної Панорами.

Дані Жолоблення Сіті -- Warp Data Mesh¶

Багато середовищ проектування вимагають зображень, що не є простими перспективними проекціями, які є нормою для плоско екранних дисплеїв. Приклади включають корекцію геометрії для циліндричних показів та деякі нові методи проектування на планетарні куполи або прямовисні куполи, призначені для віртуальної реальності VR.

Детальніше про цей формат сіті дивіться -- Paul Bourke's article.

Для продукування цих зображень ми використовуємо специфічний формат файлу.

Шаблон файлу:

mode
width height
n0_x n0_y n0_u n0_v n0_i
n1_x n1_y n1_u n1_v n1_i
n2_x n1_y n2_u n2_v n2_i
n3_x n3_y n3_u n3_v n3_i
(...)

First line is the image type the mesh is support to be applied to: 2 = rectangular, 1 = radial Next line has the mesh dimensions in pixels. Rest of the lines are the nodes of the mesh.

Each line is compound of x, y, u, v, i. (x, y) are the normalized screen coordinates, (u, v) texture coordinates, i a multiplicative intensity factor.

x varies from negative screen aspect to positive screen aspect. y varies from -1 to 1. u and v vary from 0 to 1. i ranges from 0 to 1, if negative do not draw that mesh node.

  1. Вам необхідно створити файл та додати його у Редактор Тексту, щоб вибрати як файл даних Warp Mesh.
  2. Відкрити Редактор Тексту Editor Types ‣ Text Editor.
  3. Відкрити файл даних сіті (наприклад, myDome.data) у Редакторі Тексту (Text ‣ Open чи Alt-O).
  4. Перейти у Game Framing Settings -- Editor Types ‣ Properties editor ‣ Scene.
  5. Увімкнути режим Dome.
  6. Увести ім'я файлу у поле Warp Data (наприклад, myDome.data).

Для створення вами власних Warp Meshes доступний інтерактивний інструмент під назвою meshmapper, який є частиною програмного пакета Paul Bourke's Warpplayer (вимагає повну версію).

Quan trọng

Оглядвікно розраховується на основі пропорції ширини полотна до його висоти. Звідси, різні розміри екрану будуть вимагати різні файли жолоблення сіті. Також, щоб отримати коректну пропорцію вашого проектора, вам необхідно використовувати Blender у Повноекранному режимі.
