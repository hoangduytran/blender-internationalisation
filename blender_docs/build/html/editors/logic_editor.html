Редактор Логіки -- Logic Editor¶

Редактор Логіки -- Logic Editor забезпечує головний метод укладання та редагування логіки гри для різних акторів (тобто, об'єктів), які складають цю гру. Логіка для об'єктів, які поточно виділені в асоційованому 3D Огляді -- 3D View, показується у вигляді логічних брикетів, які відображаються як таблиця з трьох стовпців: сенсорів -- sensors, контролерів -- controllers та актуаторів -- actuators, відповідно. Зв'язки, що сполучають ці логічні брикети, проводять імпульси по шляху сенсор-контролер та далі -- контролер-актуатор.

Щоб дати вам краще розуміння Редактора Логіки, ілюстрація нижче демонструє типовий вміст цього редактора, головні компоненти якого позначені. Ми розглянемо кожен з них окремо.

Різні частини Редактора Логіки -- Logic Editor.

1) Property Region, 2) Object Name, 3a) Links, 3b) Link socket, 4) Sensor column, 5) Controller Column, 6) Actuator Column.

Main View¶

Ім'я Об'єкта -- Object Name
This toggle shows the name of the object which owns the logic bricks below.
Зв'язки -- Links

Зв'язки (ЗА -- на ілюстрації) вказують напрямок потоку логіки між об'єктами. Лінії зв'язків рисуються тягненням LMB від одного роз'єму Зв'язка -- Link socket (3B) до іншого. Зв'язки можуть бути прорисовані тільки від Сенсорів -- Sensors до Контролерів -- Controllers та від Контролерів -- Controllers до Актуаторів -- Actuators. Ви не можете безпосередньо зв'язувати Сенсори -- Sensors до Актуатори -- Actuators; аналогічно, Актуатори не можуть бути пов'язані назад до Сенсорів (однак, доступні окремо спеціальні типи актуаторів та сенсорів, що допускають ці з'єднання).

Вузли Відсилання -- sending nodes (чорні кільця на правій стороні Сенсорів та Контролерів) можуть відсилати одночасно до кількох вузлів Приймання -- Reception nodes (білі кільця на лівій стороні Контролерів та Актуаторів). Вузли Приймання -- Reception nodes можуть аналогічно приймати одночасно кілька зв'язків.

Зв'язки можуть створюватися між логічними брикетами, що належать різним об'єктам. Для видалення зв'язка між двома вузлами протягніть LMB між цими двома вузлами.

Стовпець Сенсорів -- Sensor Column¶

Цей стовпець містить список усіх сенсорів, що належать активному об'єкту (та будь-яким іншим виділеним об'єктам). Нові сенсори для активного об'єкта створюються за допомогою кнопки «Додати Сенсор» -- "Add Sensor". Більш глибоко про вміст, розставу та доступні операції у цій області дивіться тут -- Sensors.

Стовпець Контролерів -- Controller Column¶

Цей стовпець містить список усіх контролерів, що належать активному об'єкту (та будь-яким іншим виділеним об'єктам). Нові контролери для активного об'єкта створюються за допомогою кнопки «Додати Контролер» -- "Add Controller", разом зі створенням станів для активного об'єкта. Більш глибоко про вміст, розставу та доступні операції у цій області дивіться тут -- Controllers.

Стовпець Актуаторів -- Actuator Column¶

Цей стовпець містить список усіх актуаторів, що належать активному об'єкту (та будь-яким іншим виділеним об'єктам). Нові актуатори для активного об'єкта створюються за допомогою кнопки «Додати Актуратор» -- "Add Actuator". Більш глибоко про вміст, розставу та доступні операції у цій області дивіться тут -- Actuators.

Регіон Властивості -- Properties Region¶

Властивості гри подібні до змінних в інших мовах програмування. Вони використовуються для зберігання та доступу до даних, асоційованих з об'єктом. Доступні різні типи властивостей. Властивості оголошуються клацком на кнопці «Додати Властивість Гри» -- Add Game Property у цьому регіоні. Більш глибоко про вміст, розставу та доступні операції у цій області дивіться тут -- Properties.
