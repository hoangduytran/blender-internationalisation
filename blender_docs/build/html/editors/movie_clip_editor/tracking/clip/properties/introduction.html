Вступ¶

Панель «Об'єкти» -- Objects Panel¶

Objects panel.

Ця панель містить огляд списком -- list view усіх об'єктів, які можуть бути використані для простеження, обчислення камери або об'єкта. Стандартно тут є тільки один об'єкт у цьому списку, який використовується для обчислення камери. Він не може бути видалений, а інші об'єкти не можуть бути використані для обчислення камери; усі додані об'єкти використовуються для тільки простеження та обчислення об'єктів. Ці об'єкти можуть посилатися примусами «Слідування Стежкою» -- Follow Track та «Обчислювач Об'єкта» -- Object Solver. «Слідування Стежкою» -- Follow Track стандартно використовує об'єкт камери.

Якщо деякі стежки були додані та простежені для невірного об'єкта, то вони можуть бути скопійовані для іншого об'єкта за допомогою «Простеження > Копіювати Стежки» -- Track ‣ Copy Tracks та «Простеження > Вставити Стежки» -- Track ‣ Paste Tracks.

Використання для всіх видів об'єктів (використовується для простеження камери та об'єкта) є однаковим: простеження вирізнень, задання даних камери, обчислення руху. Дані камери є спільними для всіх об'єктів та уточнення характеристик камери відбувається тільки тоді, коли обчислюється рух камери.

Панель «Простеження» -- Track Panel¶

Track panel.

Ім'я -- Name
Ім'я стежки може бути змінене за допомогою цього поля. Імена стежок використовуються для пов'язування даних простеження з іншими областями, наприклад, з примусом «Слідування Стежкою» -- Follow Track.
Увімкнення -- Enable (іконка з оком)
Цей перемикач контролюється стягом увімкнення мітки. Якщо мітка вимкнена, то її позиція не використовується ні для обчислювача, ні для примусів.
Блокування -- Lock (іконка замка)
Цей перемикач керує блокуванням стежки. Блоковані стежки не можуть редагуватися взагалі. Це допомагає запобігти випадковим змінам для стежок, які є «закінчені» (простежені точно по всьому фільмоматеріалі).

Віджет «Передогляд Стежки» -- Track Preview Widget¶

Віджет на цій панелі називається «Передогляд Стежки» -- "Track Preview" та він показує вміст області взірця. Це допомагає перевірити, як точно вирізнення було простежене (контролювати, чи не було сковзань від оригінальної позиції), а також допомагає пересувати стежку назад у правильну позицію. Стежка може бути пересунута безпосередньо за допомогою цього віджета перетягом миші.

Якщо використовується анкер (позиція у зображенні, яке простежується, є відмінною від позиції, яка використовується для приріднення), то віджет передогляду буде показувати область навколо позиції анкера. Ця конфігурація допомагає при маскуванні деяких речей, коли немає доброго вирізнення у позиції, де куток маски повинен бути розміщений. Деталі цієї техніки буде описано пізніше.

Нижче віджета передогляду є маленька область, яка може використовуватися для збільшення вертикального розміру віджета передогляду (ця область підсвічується двома горизонтальними лініями).

Вагомість -- Weight

Коли кілька стежок використовуються для 3D реконструкції камери, то можливо призначити скорочену вагомість для деяких стежок, щоб керувати їх впливом на результат обчислення. Це параметр може (і часто потребує) анімування.

Зміна вагомостей проблемних міток простеження може коректувати або значно скорочувати небажані стрибки при зниканні вирізнення або ускладненні простеження.

Інше використання вагомостей стежок -- коли ви хочете реконструювати сцену з вашого обчислення камери. У такому випадку ви можете спершу ретельно простежити та обчислити вашу сцену, після цього заблокувати усі ваші мітки за допомогою Ctrl-L, далі задати вагомість для простежника на панелі «Додаткові Устави» -- Extra Settings на нуль та використати виявлення вирізнення для швидкого додання багатьох міток. Далі, простежте їх та обчисліть сцену знову. Оскільки, їх вагомість дорівнює нулю, то вони не впливатимуть на ваше обчислення взагалі, але ви буде мати багато добрих орієнтувальних точок у вашій сцені.

Вагомість Стабілізації -- Stabilization Weight
У той час, як параметр «Вагомість» -- Weight використовується для 3D реконструкції, «Вагомість Стабілізації» -- Stabilization Weight (Stab Weight) використовується для керування 2D стабілізацією.
Передустави Кольорів -- Color Presets
Передустава для опції «Власний Колір» -- Custom Color.
Власний Колір -- Custom Color
This setting overrides the default marker color used in the Clip editor and 3D View, and it helps to distinguish different type of features (for example, features in the background vs. foreground and so on). Color also can be used for "grouping" tracks so a whole group of tracks can be selected by color using the Select Grouped operator.

Панель «Стежка Площини» -- Plane Track Panel¶

Plane Track panel.

Її властивості показуються тільки тоді, коли вибране простеження площини. Спершу, показується ім'я виділеної стежки площини. Воно може також бути змінене тут.

Авто Ключкадр -- Auto Keyframe
Перемикає авто-ключкадрування для кутків стежки площини. З увімкненням цього ключкадри будуть автоматично вставлятися, коли будь-який з кутків пересуватиметься.
Зображення -- Image
Field to select an image which will be displayed inside the plane track. This image is for preview purposes in the Movie Clip editor only. To include it in your final render, see Plane Track Deform node.
Безпрозорість -- Opacity
Використовується для задання безпрозорості цього зображення. Знову ж, це тільки для цілей показу і не впливає на фінальний зображ, рендер.

Панель «Устави Простеження» -- Tracking Settings Panel¶

Ця панель містить устави простежника -- tracker settings для кожної мітки.

Панель «Нарисний Олівець» -- Grease Pencil Panel¶

Grease pencil strokes can be enabled/disabled with the checkbox in the panel header. It is a standard Grease pencil panel where new Grease pencil layers and frames can be controlled. There is one difference in the behavior of the Grease pencil from other areas -- when a new layer is created "on-demand" (when making a stroke without adding a layer before this) the default color for the layer is set to pink. This makes the stroke easy to notice on all kinds of movies.
