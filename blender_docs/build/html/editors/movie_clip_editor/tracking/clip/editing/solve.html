Обчислення -- Solve¶

Панель «Стежка Площини» -- Plane Track Panel¶

Оператор «Створити Стежку Площини» -- Create Plane Track створює нову стежку площини. Площинне простеження має перевагою той факт, що часто у фільмоматеріалі присутні площинні поверхні, до точок яких можна прикріпити мітки. Це може використовуватися для заміщення таких речей, як плакати або екрани на фільмоматеріалі, на інші зображення або відео. Це також може бути корисним для маскування.

Ця кнопка створить об'єкт площини, який деформується так само, як площина, визначена усіма виділеними стежками її точок. Принаймні чотири вирізнені точки простежуються по фільмоматеріалу, що належать площині, яку передбачається замінити. Більше стежок дасть кращу оцінку руху площини.

Вирізнені точки, використані для оцінки руху площини, може бути використані з будь-якого місця площини, тобто не обов'язково вони мають знаходитися на її кутках. Кутки не завжди легко простежити, вони можуть загороджуватися. У цьому випадку ви можете позиціонувати простежувані вирізнення, щоб вони лежали на одній площині, рівно віддаленій від фактичної площини, яка повинна бути замінена.

This provides more information about the possible deformation of the marker in following frames, and such markers can be tracked even if partially occluded (appear and disappear during the time). It is only required that two neighbor frames have at least four common tracks.

Зображення може бути проектоване на таку площину за допомогою вузла компонування Вузол «Деформа Стежки Площини» -- Plane Track Deform Node.

Панель «Обчислення» -- Solve Panel¶

Тринога -- Tripod¶

Опція «Тринога» -- Tripod може використовуватися для фільмоматеріалу, при зйомці якого камера не рухалася, а тільки оберталася. Такий фільмоматеріал не може бути простежено загальним обчислювачем, і неможливо визначити фактичні вирізнені точки у просторі через нестачу інформації. А тому цей обчислювач буде обчислювати тільки відносне обертання камери, а потім перепроектовувати вирізнені точки на сферу з однаковою відстанню між вирізненням та камерою для усіх вирізнених точок.

Ghi chú

Будь ласка, зауважте, що це є спеціальний тип обчислювача камери й він поводиться інакше, ніж звичайний обчислювач. Це означає, що використання більше стежок не призведе до більш точного обчислення. Наявність 5-10 стежок на кадрі зазвичай має бути достатнім для використання цього роду обчислювача.

Ключкадр -- Keyframe¶

Ключкадр -- Keyframe
Автоматично вибирає ключкадри для початкової реконструкції. Ця опція вмикає складний алгоритм, який намагається знайти пару ключкадрів з мінімальною помилкою реконструкції та найкращим припущення щодо масштабу сцени.
Ключкадр А, В -- Keyframe A, B Q, E
Стартовий (А) та Кінцевий (В) кадри діапазону, що використовується для реконструкції.

Уточнення -- Refine¶

Опція «Уточнення» -- Refine визначає, які параметри повинні бути уточнені у ході обчислення. Таке уточнення є корисним, коли ви не впевнені про деякі характеристики камери, та обчислювач повинен спробувати знайти найкращий параметр для цих характеристик. Але ви все ще маєте знати приблизні початкові значення -- він не зможе знайти правильні значення, якщо вони були задані зовсім неправильно спочатку.

Обчислити Рух Камери/Об'єкта -- Solve Camera/Object Motion¶

Оператор обчислення руху камери -- Camera Motion обчислює рух камери, використовуючи усі стежки, поміщені на фільмоматеріалі, та два ключкадри, визначені на цій панелі. Існують кілька вимог:

  • На обох вибраних ключкадрах повинно бути задано принаймні по вісім загальних стежок.
  • Між цими двома ключкадрами повинен бути помітний ефект паралаксу.

If everything goes smoothly during the solve, the average reprojection error is reported to the information space and to the Clip editor header. Reprojection error means the average distance between reconstructed 3D position of tracks projected back to footage and original position of tracks. Basically, reprojection error below 0.3 means accurate reprojection, (0.3 - 3.0) means quite nice solving which still can be used. Values above 3 means some tracks should be tracked more accurately, or that values for focal length or distortion coefficients were set incorrectly.

Панель «Вичищення» -- Cleanup Panel¶

Ця панель містить один оператор та його устави. Цей оператор вичищає погані стежки: стежки, які не простежуються досить довго або які не вдалося реконструювати точно.

Кадри -- Frames
ToDo >2.61.
Помилка -- Error
Значення порогу помилки.
Дія -- Action

Різні дії можна здійснити при вичищенні поганих стежок:

Виділене -- Selected
Вони можуть просто бути виділеними.
Видалити Стежку -- Delete Track
Стежка повністю може бути видалена.
Видалити Сегменти -- Delete Segments
Погані сегменти простеженої послідовності можуть бути вилучені.

3D Мітки у Сіть -- 3D Markers to Mesh

Пов'язати Порожняк до Стежки -- Link Empty to Track
ToDo 2.61.

Панель «Орієнтація» -- Orientation Panel¶

Засоби орієнтації сцени можуть бути використані для орієнтування об'єкта за набором міток.

Підлога -- Floor
Select three markers that should lay on the floor plane. ToDo >2.61.
Стіна -- Wall
Визначає орієнтацію світу на основі точок на стіні.
Задати Початок -- Set Origin
ToDo >2.61.
Задати X, Y Вісь -- Set X, Y Axis
ToDo >2.61.
Задати Масштаб -- Set Scale
Об'єкт має отримати масштаб для визначення «глибини» у просторі камери.
Застосування -- Apply
Застосовує масштаб при обчисленні сцени.
Відстань -- Distance
ToDo >2.61.
