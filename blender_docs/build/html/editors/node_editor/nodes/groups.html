Групи Вузлів або Групові Вузли -- Node Groups¶

Групування вузлів може спрощувати дерево вузлів, дозволяючи примірникувати (використовувати повторно примірники частин дерева) та ховати частини дерева. Можуть групуватися, як матеріалові (material), так і компонівні (composite) вузли.

Концептуально, групування вузлів дозволяє вам визначати набір вузлів, який ви можете обробляти, як ніби це був би «тільки один вузол». Групи вузлів подібні до функцій у програмуванні. Далі ви можете повторно використовувати їх всередині, які далі названі "NodeGroups", або в іншому blend-файлі(ах), при прилученні названі "NodeTrees".

Як приклад: Якщо ви створили матеріал, який ви б хотіли використати з іншими уводами, наприклад, колір розсіяння чи дифузний колір: червоний пластик, зелений пластик. Ви можете створити різні матеріали і зробити їх однокористувацькими за допомогою команди «Зробити Однокористувацьким» -- Make Single User для кожного відтінку кольору з копією частини дерева, що описує матеріал пластику. Якщо ви схочете редагувати цей матеріал, то вам необхідно буде повторити всі ці правки на усіх цих матеріалах. Кращим методом повторного використання є створення груп вузлів, з виставленням тільки уводів, що будуть змінюватися (наприклад, дифузний колір).

При групуванні також підтримуються вкладені один в один групові вузли. Тобто, група вузлів може бути вставлена або створена всередині іншої групи вузлів.

Ghi chú

Рекурсія

Рекурсивні групові вузли заборонені для усіх поточних систем вузлів з метою запобігання безкінечній рекурсії, «замикання на собі». Група вузлів ніколи не може вміщувати сама себе (або іншу групу, що вміщує її).

Зроблення Групи -- Make Group¶

Орієнтир

Режим:Усі Режими
Меню:«Група > Зробити Групу» -- Group ‣ Make Group
Скорочення:Ctrl-G

Для створення групи вузлів у Редакторі Вузлів -- Node editor виділіть вузли, які ви хочете включити у цю групу, далі натисніть Ctrl-G або у меню виберіть «Група > Зробити Групу» -- Group ‣ Make Group. Груповий вузол буде мати забарвлену зеленим смужку заголовку. Усі виділені вузли будуть відтепер вміщені всередині цієї групи вузлів. Стандартне іменування для групи вузлів -- "NodeGroup", "NodeGroup.001" тощо. На груповому вузлі передбачене поле імені, на якому ви можете клацнути та змінити його на потрібне. Змінюйте ім'я групи вузлів на щось значуще. При прилученні груп вузлів з одного blend-файлу в інший Blender не робить різниці між матеріаловими та компонівними груповими вузлами, а тому рекомендується мати певну умовність іменування, яка дозволить вам легко відрізняти між собою ці дві категорії вузлів.

Mẹo

Що не включається у ваші групи (усі режими Редактора Вузлів)

Пам'ятайте, що основоположна ідея полягає у тому, що група вузлів повинна бути легко повторно використовуваним, автономним програмним компонентом. Матеріалові групові вузли не повинні включати:

Вузли типу «Увід» -- Input nodes
Якщо ви включите джерельний вузол у вашу групу, то у результаті джерельний вузол буде присутнім двічі: один всередині групи, а інший поза групою у новій мережі вузлів матеріалу.
Вузли типу «Вивід» -- Output node
Якщо ви включите вивідний вузол у групу, то будуть відсутні вивідні роз'єми для виводу даних із групи!

Правка Групи -- Edit Group¶

Орієнтир

Режим:Усі Режими
Меню:«Група > Правка Групи» -- Group ‣ Edit Group
Заголовок:«Перейти у Предківське Дерево Вузлів» -- Go to Parent Node Tree
Скорочення:Tab, Ctrl-Tab

With a node group selected, Tab expands the node to a frame, and the individual nodes within it are shown. You can move them around, play with their individual controls, re-thread them internally, etc. just like you can if they were a normal part of the editor view. You will not be able, though, to thread them to a node outside the group; you have to use the external sockets on the side of the node group. To add or remove nodes from the group, you need to ungroup them. While Tab can be used to both enter and exit a group, Ctrl-Tab only exits.

Інтерфейс -- Interface¶

Інтерактивно¶

Коли груповий вузол створено, то у ньому генеруються вузли «Увід Групи» -- Group Input та «Вивід Групи» -- Group Output для представлення потоку даних у та з даної групи. При створенні, з'єднання до увідних роз'ємів невиділених вузлів стануть прикріпленими до нових роз'ємів на вузлі Group Input. Аналогічно, вихідні з'єднання до увідних роз'ємів невиділених вузлів стануть прикріпленими до нового вузла Group Output.

Якщо у ході розроблення групи вузлів необхідно буде подати додатковий параметр, що має входити у цю групу, то повинен бути доданий додатковий роз'єм на вузлі Group Input. Це легко робиться шляхом додання з'єднання від порожнього роз'єма на правій стороні вузла Group Input до бажаного увідного роз'єма на вузлі, що вимагає уводу. Цей процес є подібним і для Group Output відносно даних, які ви хочете зробити доступними поза даною групою.

Панель¶

Орієнтир

Режим:Усі Режими
Панель:«Регіон Властивості > Інтерфейс» -- Properties region ‣ Interface

Панель інтерфейсу для правлення груп.

На цій панелі можна додавати, пере-впорядковувати або вилучати роз'єми для групи, можливо додавати описові для них імена, а також тут можуть визначатися деталі значення увідних даних.

Якщо ви маєте одночасно кілька уводів або виводів, то воним можуть бути пере-впорядковані шляхом вибору потрібного роз'єму у цьому списку і далі пересування його уверх або вниз за допомогою кнопок зі стрілками на правому боці цієї панелі. Великі кнопки зі знаком плюс нижче списку будуть додавати непід'єднаний роз'єм такого ж типу, як і вибраний роз'єм, або роз'єм значення, якщо нічого не вибрано. Малий обведений кільцем знак плюс внизу списку має функції фільтрування для сприяння узнаходженні вузлів, якщо група має велику кількість роз'ємів.

Розгрупування -- Ungroup¶

Орієнтир

Режим:Усі Режими
Меню:«Група > Розгрупування» -- Group ‣ Ungroup
Скорочення:Alt-G

Засіб за скороченням Alt-G розгруповує, вилучаючи властивість групи, всі вкладені у груповому вузлі інші вузли та поміщає їх у робочому просторі редактора. Жодні внутрішні з'єднання не втрачаються і ви далі можете зв'язувати ці колишні елементи групи з іншими вузлами у вашому робочому просторі.

Відокремлення -- Separate P

Відокремлює виділені вузли від групи вузлів.

Копіювання -- Copy
Копіює у предківське дерево вузлів, зберігаючи групу незачепленою.
Переміщення -- Move
Переміщує у предківське дерево вузлів, вилучаючи з групи.

Вставляння у Групу -- Group Insert¶

Орієнтир

Режим:Усі Режими
Меню:«Група > Вставити у Групу» -- Group ‣ Group Insert

При виділенні набору потрібних вузлів, з виділенням останнім вузла групи призначення, та натискання «Група > Вставити у Групу» -- Group ‣ Group Insert перемістить ці вузли у цю групу. Ці переміщені вузли збируться у свою групу, зберігаючи свої з'єднання, маючи свої вузли уводу та виводу групи. Наявні вузли уводу та виводу групи оновляться новими роз'ємами, від нових вузлів. Дана група вузлів повинна бути поправлена, щоб містити лише один вузол Group Input та тільки один вузол Group Output.

Прилучення Груп Вузлів¶

Орієнтир

Редактор:Редактор Інфо -- Info Editor
Режим:Усі Режими
Меню:«Інфо Редактор > Файл > Пов'язання/Прилучення» -- Info Editor ‣ File ‣ Link/Append

Після прилучення Дерева Вузлів іззовні у ваш blend-файл ви можете його використовувати у Редакторів Вузлів -- Node Editor, натиснувши Shift-A або «Додання > Група» -- Add ‣ Group, і далі вибрати потрібну з прилучених груп. «Керувлаьна панель» цієї Групи являє собою індивідуальні керувальники для згрупованих у ній вузлів. Ви можете змінювати їх, працюючи з цим Груповим вузлом так само, як і з іншими вузлами.
