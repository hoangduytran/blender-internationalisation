Вступ¶

Редактор Вузлів -- Node Editor.

The Node Editor is used to work with node-based workflows. The node tree type can be changed using the buttons in the Node editor header. However, here we will only give an overview of what the Node Editor is. In the list below it shows a list of different types of node trees and where each is documented.

Іконка Ім'я Документація
Матеріалові Вузли -- Material Nodes Оскільки, є два різних рушії зображу, рендеру, то документація розділена між Blender Internal та Cycles.
Компонівні Вузли -- Composite Nodes Документація може бути знайдена у секції -- Compositing.
Текстурові Вузли -- Texture Nodes Текстурові Вузли розглядаються у документації внутрішнього рушія Blender Internal.

Після того, як ви вибрали бажаний контекст вузлів для використання, ви маєте увімкнути вузли за допомогою кнопки «Вжити Вузли» -- Use Nodes.

Інтерфейс -- Interface¶

Заголовок -- Header¶

Заголовок -- Header містить різні меню, кнопки та опції, що частково базуються на поточному типі дерева вузлів.

Common Node editor header options.

Огляд -- View
Це меню змінює огляд редактора.
Виділ -- Select
Дане меню дозволяю вам виділяти вузол або групу вузлів.
Додання -- Add
Це меню дозволяє вам додавати вузли.
Вузол -- Node
Робить різні операції з виділеними вузлами, як ніби з вершинами.
Кнопки перемикання на режим Матеріалових, Компонувальних або Текстурових вузлів
Вузли груповані у три категорії, дивіться список Node Tree Types.
Вжити Вузли -- Use Nodes
Каже рушію зображу, рендера чи використовувати, вживати карту вузлів при обчисленні кольору матеріалу або рендерингу, зображуванні фінального зображення, чи ні. Якщо ні, то карта вузлів ігнорується застосовуються параметри матеріалу з базових вкладок для зображу, рендера або зі сцени.
Вжити Пришпилене -- Use Pinned
When enabled, the editor will retain the material or texture, even when the user selects a different object. A node tree can then be edited independent of the object selection in the 3D View.
Йти до Предка -- Go to Parent button
Ця кнопка дозволяє вам перейти у предківське дерево вузлів, наприклад, залишивши групу.

Авто-зсув -- Auto-offset¶

When you drop a node with at least one input and one output socket onto an existing connection between two nodes, Auto-offset will, depending on the direction setting, automatically move the left or right node away to make room for the new node. Auto-offset is a feature that helps organizing node layouts interactively without interrupting the user workflow.

Авто-зсув стандартно увімкнено, але його може вимкнути у заголовку Редактора Вузлів -- Node Editor.

Ви можете перемикати напрямок зсуву при руханні нового вузла, натискаючи T.

Припуск зсуву може бути змінено за допомогою устави «Припуск Авто-зсуву» -- Auto-offset Margin у вкладці «Правка» -- Editing вікна «Уподобання Користувача» -- User Preferences.

Подальші меню¶

Підхоп -- Snap
Перемикає режим підходу для вузлів у Редакторі Вузлів -- Node Editor.
Вибірник Елемента Підходу Вузлів -- Snap Node Element Selector

This selector provide the following node elements for snap:

Сітка -- Grid:(стандарт) Підхоплює до сітки Редактора Вузлів.
Х Вузла -- Node X:
 Підхоплює до лівої/правої границі вузла.
Y Вузла -- Node Y:
 Підхоплює до верхньої/нижньої границі вузла.
X/Y Вузла -- Node X/Y:
 Підхоплює до будь-якої границі вузла.
Ціль Підхопу -- Snap Target

Яка частина буде підхоплюватися до цілі.

Найближче -- Closest:
 Підхоплює найближчу точку до цілі.
Центр -- Center:
 Підхоплює центр до цілі.
Серединне -- Median:
 Підхоплює середину до цілі.
Активне -- Active:
 Підхоплює активне до цілі.
Копіювати Вузли -- Copy Nodes
Ця кнопка дозволяє вам скопіювати виділені вузли у буфер обміну.
Вставити Вузли -- Paste Nodes
Ця кнопка дозволяє вам вставити вузли з буфера обміну в активне дерево вузлів.

Полиця Інструментів -- Tool Shelf¶

Полиця Інструментів -- Tool Shelf -- це контекстно-чутливий регіон, що початково містить інструменти для Нарисного Олівця -- Grease Pencil та кнопки для додання вузлів. Полиця Інструментів організована за допомогою вкладок.

Регіон Властивості -- Properties Region¶

Регіон «Властивості» -- Properties Region містить властивості для поточно виділеного вузла, а також специфічні устави для Редактора Вузлів.
