Інтегратор -- Integrator¶

Інтегратор -- це алгоритм зображування, рендеринга, який використовується для обчислення освітлення. Cycles поточно підтримує інтегратор простеження шляхів з вибиранням прямого освітлення. Він добре працює для різних укладів освітлення, але не настільки підходить для каустики й деяких інших складних ситуацій освітлення.

Промені простежуються від камери у сцену у ході їх відскакування від об'єктів сцени, допоки вони не натраплять на джерело світла, таке як лампа, світло емітувальний об'єкт або фон світу. Для знаходження ламп та світло емітувальних поверхонь використовуються обидва види відбору вибірок, відбір побічного освітлення (слідування променя по поверхні BSDF) та відбір прямого освітлення (вказування джерела світла та простеження променя у напрямку до нього).

Відбір -- Sampling¶

Орієнтир -- Reference

Panel:Render ‣ Sampling
Метод Відбору -- Sampling Method
Є два можливих режими інтегратора: «Простеження Шляху» -- Path Tracing та «Простеження Розгалуженого Шляху» -- Branched Path Tracing.
Вибірки до Квадрата -- Square Samples
Підносить до квадрата (у степінь 2) кількість вибірок.
Базис -- Seed

Значення базису для інтегратора для отримання різних взірців шуму.

Анімування Базису -- Animate Seed (іконка годинника)
Ця кнопка справа від устави Seed вмикає задання різних значень базису у різних кадрах. Рекомендується вмикати її при зробленні анімації, оскільки у реальному світі кожен кадр має різний взірець шуму.
Затиск Прямих -- Clamp Direct

Ця опція лімітує максимальну інтенсивність вибірки з променів, які ще не відскакнули, та можуть привнестися у піксель. Це скорочує шум ціною точності. Задання цієї устави як 0.0 вимикає затискання взагалі. Нижчі значення мають більший вплив (затемнені вибірки) на результатне зображення, ніж вищі значення.

Ghi chú

Поширеною проблемою методу Path Tracing є поява «світлячків» -- "fireflies": неймовірні вибірки, що привносять дуже високі значення для пікселів. Ця устава забезпечує спосіб лімітування цього. Однак, зауважте, що коли ви затискаєте такі значення, то інші яскраві підсвітки/відбиття також будуть затемнені.

Слід бути обережним при використанні цієї устави, щоб знайти баланс між послабленням світлячків та навмисною втратою яскравих частин. Часто корисно затискати побічні відскоки окремо, оскільки вони більше тяжіють спричиняти світлячки, ніж прямі відскоки. Дивіться детальніше про уставу Clamp Indirect.

Затиск Побічних -- Clamp Indirect
Так само як і Clamp Direct, але для променів, які вже відскакнули багато разів.
Поріг Відбору Освітлення -- Light Sampling Threshold
Ймовірно припиняє вибірки освітлення, коли внесок освітлення нижче цього порогу (більше шуму, але швидший рендеринг). Нуль вимикає цю перевірку та ніколи не ігнорує освітлювачі. Це корисно, тому що у великих сценах з багатьма джерелами освітлення, деякі з них можуть здійснювати малий внесок у фінальне зображення, що недоцільно збільшує час на рендеринг. Використання цієї устави може зменшити час на рендеринг, необхідний для розрахування променів, що у кінцевому підсумку мають дуже маленький вплив на зображення.
Взірець -- Pattern

Випадковий взірець відбору, використовуваний інтегратором.

Соболь -- Sobol
Використовує взірець на основі послідовності Соболя як випадковий взірець відбору, використовуваний інтегратором. Дивіться детальніше тут -- Sobol sequence.
Корельоване Багато-Тремтіння -- Correlated Multi-Jitter
Використовує взірець Корельованого Багато-Тремтіння як випадковий взірець відбору, використовуваний інтегратором Дивіться детальніше тут -- this Pixar paper.
Вибірки Шарів -- Layer Samples

Коли шари зображів мають для кожного з них задану кількість вибірок, ця опція визначає, як їх використовувати.

Вживання -- Use
Вибірки шару зобража будуть заміщати задані вибірки для сцени.
Обмежено -- Bounded
Обмежує вибірки шару зобража вибірками сцени.
Ігнорування -- Ignore
Ігнорує устави вибірок шару зобража.

Простеження Шляху -- Path Tracing¶

Інтегратор Path Tracing є простежником простого шляху; при кожному потраплянні він буде простежувати відскок світла в одному напрямку та відбирати один освітлювач для приймання з нього освітлення. Це пришвидшує кожну індивідуальну вибірку для обчислення, але зазвичай вимагає більше вибірок для вичищення шуму.

Зображ: Вибірки -- Render Samples
Кількість шляхів для простеження для кожного пікселя на фінальному рендері. Чим більше вибірок задано, тим результат стає менш шумним та більш точним.
Передогляд: Вибірки -- Preview Samples
Кількість вибірок для рендерингу в оглядвікні.

Простеження Розгалуженого Шляху -- Branched Path Tracing¶

Не-прогресивний інтегратор Branched Path Tracing дозволяє точніше керувати відбором вибірок. Він подібний на інтегратор Path Tracing, але при першому потраплянні він буде розділяти шлях для різних компонент поверхні та буде враховувати усі, а не тільки один, освітлювачі для відтінення.

Це уповільнює кожну вибірку, але дасть скорочення шуму, особливо у сценах, в яких домінують пряме або освітлення з одним відскоком. Для отримання такої ж кількості вибірок розсіяння, дифузії, як в інтеграторі Path Tracing, зауважте, що наприклад, 250 вибірок простеження простого шляху = 10 AA вибірок x 25 diffuse вибірок простеження розгалуженого шляху. Панель «Відбір» -- Sampling показує загальну кількість вибірок.

Зображ: Вибірки Загладжування
Кількість вибірок для відбирання для кожного пікселя у фінальному рендері. Більше вибірок покращуватиме загладжування -- anti-aliasing.
Передогляд: Вибірки Загладжування -- AA Preview Samples
Кількість вибірок для рендерингу в оглядвікні.
Вибірки: Розсіяння -- Diffuse Samples
Кількість вибірок відскоків розсіяння, дифузії для відбору для кожної вибірки загладжування -- AA sample.
Вибірки: Глянець -- Glossy Samples
Кількість вибірок відскоків глянцю для відбору для кожної вибірки загладжування.
Вибірки: Пропускання -- Transmission Samples
Кількість вибірок відскоків пропускання світла для відбору для кожної вибірки загладжування.
Вибірки: Загорода Оточення -- AO Samples
Кількість вибірок загороди оточення -- ambient occlusion для відбору для кожної вибірки загладжування.
Вибірки: Світна Сіть -- Mesh Light Samples
Кількість вибірок світної сіті для відбору для кожної вибірки загладжування.
Вибірки: Підповерхня -- Subsurface Samples
Кількість вибірок підповерхневого розсіювання для відбору для кожної вибірки загладжування.
Вибірки: Об'єм -- Volume Samples
Кількість вибірок розсіювання об'ємом для відбору для кожної вибірки загладжування.
Вибірка Усіх Освітлювачів Прямих -- Sample All Direct Lights
При увімкненні Cycles буде здійснювати вибірки усіх освітлювачів у сцені для прямих відскоків, замість здійснення відбору одного випадкового освітлювача. Вимкнення цієї опції може підвищити продуктивність при використанні багатьох AA Samples, у будь-якому випадку, для вичищення рендера.
Вибірка Усіх Освітлювачів Побічних -- Sample All Indirect Lights
Подібно до Sample All Direct Lights, але відбираються освітлювачі для побічних відскоків. Це може скоротити шум у сценах з багатьма освітлювачами.

Відскоки -- Bounces¶

Відскоки: Макс -- Max Bounces
Максимальна кількість відскоків світла. Для найкращої якості це повинно бути установлено на максимум. Однак, на практиці, може бути добре задати для цієї устави нижчі значення для швидшого рендерингу. Задання цього максимального значення відскоків як 0 дасть відбір лише з прямого освітлення.
Відскоки: Мін -- Min Bounces
Мінімальна кількість відскоків світла для кожного шляху, нижче якої інтегратор використовує Російську Рулетку для завершення шляхів, що дають менший внесок у зображення. Задання вищих значень дає менше шуму, але може також значно збільшувати час на рендеринг. При низьких значеннях відскоків настійно рекомендується задавати мінімальну кількість рівною максимальній кількості.
Відскоки: Розсіяння -- Diffuse Bounces
Максимальна кількість відскоків розсіяння.
Відскоки: Глянець -- Glossy Bounces
Максимальна кількість відскоків глянцю.
Відскоки: Пропускання -- Transmission Bounces
Максимальна кількість відскоків пропускання.
Відскоки: Об'єм -- Volume Bounces
Максимальна кількість відскоків розсіювання об'ємом.

Прозорість -- Transparency¶

Прозорість: Макс -- Transparency Max
Максимальна кількість відскоків прозорості.
Прозорість: Мін -- Transparency Min
Мінімальна кількість відскоків прозорості, нижче якої використовується Російська Рулетка для завершення шляхів, що дають менший внесок у зображення.
Прозорість: Тіні
Для відбору відскоків прямого світла використовує прозорість між поверхнями для продукування тіней, спричинених прозорістю цих поверхонь.

Каустики та Фільтр Глянцю -- Caustics & Filter Glossy¶

Каустики Відбиття -- Reflective Caustics
Хоча, у принципі, простеження шляху підтримує рендеринг каустик за допомогою достатньої кількості вибірок, на практиці ж, він може бути неефективним у тому, що дає надто багато шуму. Ця опція може бути вимкненою для вимикання рендерингу каустик відбиття.
Каустики Заломлення -- Refractive Caustics
Подібно як Reflective Caustics, тільки щодо каустик заломлення.
Фільтр Глянцю -- Filter Glossy

При використанні значення, вище за 0.0, це буде розмивати глянцеві відбиття після відскоків розмиву для скорочення шуму ціною точності. 1.0 -- хороше значення для початку підправлення.

Деякі шляхи світла мають низьку ймовірність виявлення, оскільки привносять багато світла у піксель. У результаті ці шляхи світла будуть виявлені у деяких пікселях, а в інших ні, що дасть світлячки -- fireflies. Прикладом такого складного шляху може бути наявність малого освітлювача, що спричиняє малу блікову підсвітку на різкому глянцевому матеріалів, який ми бачимо крізь грубий глянцевий матеріал. Фактично, у такому випадку, ми практично маємо каустику.

За допомогою простеження шляху важко виявити таку блікову підсвітку, але, якщо ми збільшимо значення шорсткості для матеріалу, то ці підсвітки стануть більшими та м'якшими, а звідси легшими для їх виявлення. Часто це розмивання буде ледь помітне, оскільки ми дивимося на нього крізь розмитий матеріал, у будь-якому випадку, але є також випадки, де це буде призводити до втрати детальності освітлення.
