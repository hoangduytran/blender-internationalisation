Адаптивне Підподілення -- Adaptive Subdivision¶

Орієнтир -- Reference

Panel:Modifier ‣ Subdivision Surface

Ghi chú

Втілення цього методу ще не завершено і він відноситься до експериментального набору функцій -- Experimental Feature Set.

При використанні набору експериментальних функцій -- Experimental Feature Set панель модифікатора «Підподіл Поверхні» -- Subdivision Surface Modifier змінюється для керування підподіленням сіті під час рендерингу. Для цього, усі інші устави залишаються такими ж, за винятком устав View та Render. Попередній вигляд цих устав вилучено/перейменовано та такі устави додаються:

Модифікатор «Підподіл Поверхні» -- Subdivision Surface.

Передогляд -- Preview

Рівні -- Levels
Рівні підподілення для перегляду у 3D Огляді -- 3D View, це працює так само як устава View для оригінального Subdivision Modifier.

Зображ, Рендер -- Render

Адаптовано -- Adaptive

Use OpenSubdiv to give different subdivision levels to near and far objects automatically. This allows nearer objects to get more subdivisions and far objects to get less.

Норма, масштаб Нарізання -- Dicing Rate

При увімкненні Adaptive властивість Render: Levels змінюється на Dicing Rate, ця властивість використовується для помноження на норму нарізання для сцени -- scene dicing rate.

Subdivision Вимк/Увімк, Dicing Rate: 1.0 - 0.3 - 0.05 (мавпи виглядають ідентично в оглядвікні, без модифікаторів).

Рівні -- Levels
Рівні підподілення, видимі на фінальному рендері, це працює так само, як устава Render на оригінальному Subdivision Modifier.

Відомі обмеження¶

  • Відсутня підтримка для підподілення UV.
  • Складки, наразі, не відповідають складкам Blender'а.
  • Багато-користувацькі дані об'єкта поточно робляться одиничними користувачами, що призводить до збільшення використання пам'яті. Для них краще все ще використовувати не-адаптивне підподілення.
  • Багато-оглядні рендери можуть мати певні неузгодженості між оглядами.

Cảnh báo

Примірники Частинок -- Particle instances, Примірники Груп -- Group instances, Дубль-Вершини -- Dupliverts та Дубль-Групи -- Dupligroups не теселюються, коміркуються індивідуально. Натомість, оригінальний об'єкт теселюється, коміркується, а потім дублюється на усі примірники. Щоб скористатися перевагою обох технік, адаптивного підподілення та примірникування, ви повинні помістити оригінальний об'єкт у позицію примірника, який знаходиться найближче до камери.
