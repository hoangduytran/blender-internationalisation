Розкладання -- Mapping¶

Текстури потребують розкладання координат для визначання того, як вони будуть розкладені на об'єкт. Розкладання визначає те, як текстура у кінцевому підсумку обгорне об'єкт.

For example, a 2D image texture could be configured to wrap itself around a cylindrical-shaped object.

Панель «Розкладка» -- Mapping panel.

Координати -- Coordinates¶

Розкладка працює, використовуючи набір координат для керування процесом розкладання. Ці координати можуть походити будь-звідки, зазвичай, з об'єкта, на який розкладається дана текстура.

Глобально -- Global
Глобальні 3D координати сцени. Це також корисно для анімацій; якщо ви рухаєте об'єкт, то текстура рухається на ньому. Це може бути корисним для отримання появи чи зникання об'єкта у певній позиції простору.
Об'єкт -- Object

Використовує об'єкт як джерело для координат. Часто використовується з порожняком -- Empty, це легкий спосіб розмістити маленьке зображення у заданій точці на об'єкті. Цей об'єкт може також бути анімованим для пересування текстури по поверхні.

Об'єкт -- Object
Дає змогу вибрати ім'я об'єкта.
Генероване -- Generated
Оригінальні недеформовані координати об'єкта. Це стандартна опція для розкладання текстур.
UV

UV mapping is a very precise way of mapping a 2D texture to a 3D surface. Each vertex of a mesh has its own UV coordinates which can be unwrapped and laid flat like a skin. You can almost think of UV coordinates as a mapping that works on a 2D plane with its own local coordinate system to the plane on which it is operating on. This mapping is especially useful when using 2D images as textures, as seen in UV Mapping. You can use multiple textures with one set of UV coordinates.

Карта UV -- UV Map
Розкладка, карта UV, що використовується для розкладання.
Пасмо -- Strand / Частинка -- Particle
Використовує нормалізовану 1D текстурну координату пасма або вік (X) та позицію сліду (Y) частинки. Використовується, коли текстура розкладається на пасма волосся або частинки.
Вікно -- Window
Координати вікна рендереного зображення. Це дуже добре підходить для змішання вигляду двох об'єктів.
Нормаль -- Normal
Використовує напрям вектора нормалі поверхні як координати. Це дуже корисно при створенні певних спеціальних ефектів, що залежать від кута огляду.
Відбиття -- Reflection
Використовує напрям вектора відбиття як координати. Це корисно для додання карт, розкладок відбиття. Вам знадобиться цей увід при розкладанні середовища -- Environment Mapping.
Напруженість -- Stress
Використовує різницю довжини ребра, краю, у порівнянні з оригінальними координатами сіті. Це корисно, наприклад, коли сіть деформується модифікаторами.
Дотично -- Tangent
Використовує додатково дотичний вектор як текстурні координати.

Проекція -- Projection¶

Плоско -- Flat
Плоске розкладання дає найкращі результати на одиничних площинних гранях. Це продукує цікаві ефекти на сфері, але у порівнянні зі сферичним розкладанням на сфері результат виглядає більш плоским. На гранях, що не знаходяться на площині розкладання, останній піксель текстури розтягується, що продукує смужки на кубі та циліндрі.
Куб -- Cube
Кубічне розкладання часто дає найбільш коричні результати, коли об'єкти не є надто криві та органічні за формою (зверніть увагу на шви на сфері).
Труба -- Tube
Трубкове розкладання розкладає текстуру навколо об'єкта подібно до етикетки на пляшці. Тому така текстура є більш розтягнутою на циліндрі. Це розкладання, звичайно, є дуже добрим для виготовлення етикетки на пляшці або прикріплення наклейок на округлих об'єктах. Однак, це не є циліндричним розкладанням, тому, кінці такого трубки-циліндра не визначені.
Сфера -- Sphere
Сферичне розкладання є найкращим типом для розкладання на сфері й воно ідеально підходить для зроблення вигляду поверхонь планет та подібних об'єктів. Це часто дуже корисно для створення органічних об'єктів. Це також продукує цікаві ефекти на циліндрі.

Успадковування координат від Предківського об'єкта¶

З Дублів -- From Dupli
Дублі примірникуються з вершин, граней або частинок та успадковують текстурні координати від своїх цих предків.

Зсув, масштабування та трансформування розкладки координат¶

Зсув -- Offset
The texture coordinates can be translated by an offset. Enlarging of the Offset moves the texture towards the top left.
Розмір -- Size

Ці кнопки дозволяють вам змінювати розкладку осей між власною системою координат текстури та обраною вами системою координат (Generated, UV тощо). Більш точно, кожній осі текстури відповідає один з чотирьох варіантів, що дозволяє вам вибрати, по які осі у системі розкладання вона розкладеться! Це передбачає кілька моментів:

  • Для 2D текстур (таких як зображення) відповідними є тільки два перші рядки, оскільки вони не мають жодних даних Z.
  • Ви можете обернути 2D картинку на четвертний поворот, задавши перший ряд (тобто, вісь X текстури) на Y та другий ряд (вісь Y текстури) на X.
  • Коли ви розкладаєте без вказування жодної осі текстури (тобто ці три кнопки задані як «порожні» або «none»), то ви отримаєте суцільну однорідну текстуру, ніби скористалися нульовим виміром (тобто, точки або пікселя) для цього (і далі Blender розтягує або повторює колір цієї точки по усіх осях).
  • Коли ви розкладаєте тільки на одну вісь текстури (тобто дві інші кнопки задані як «порожні» або «none»), то ви використовуєте тільки один вимір (тобто, лінію чи ряд пікселів) (і далі Blender розтягує цю лінію уздовж двох інших осей).
  • Це ж відбувається і для 3D текстур (тобто, процедурних), коли одна вісь розкладається на ніщо, Blender розтягує її проекцію («скибку») уздовж відповідної третьої осі.
