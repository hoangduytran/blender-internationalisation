Загладжування -- Anti-Aliasing¶

Генероване комп'ютером зображення складається з пікселів; кожен піксель може, звичайно, бути тільки одного кольору. У процесі зображування, рендерингу рушій зобража повинен призначати один колір для кожного пікселя на основі того, який об'єкт показується у тому пікселі. Це часто призводить до поганих результатів, особливо на різких границях або у місцях тонких ліній, і це особливо очевидно для похилих ліній.

Для долання цієї проблеми, яка називається «ступінчатість» -- Aliasing, можна вдатися до техніки «анти-ступінчатість» чи «загладжування» -- Anti-Aliasing. Базово, кожен піксель «перебирається» через зображування, рендеринг його, ніби це п'ять або більше пікселів, та на основі цього призначається для рендереного пікселя «усереднений» колір.

The buttons to control Anti-Aliasing, or Oversampling (OSA), are below the rendering button in the Render Panel.

Вибірки -- Samples

Кількість вибірок на піксель. Вищі значення дають кращі краї, але уповільнюють зображування, рендеринг.

5, 8, 11, 16

Стандартно, ми використовуємо у Blender'і фіксовану таблицю «розподіленого тремтіння» -- "Distributed Jitter". Вибірки у межах пікселя розподіляються та відхиляються у спосіб, що забезпечує дві характеристики:

  1. Кожна вибірка має рівні відстані до сусідніх для неї вибірок.
  2. Вибірки охоплюються усі під-піксельні позиції однаково, як горизонтально, так і вертикально.

Ілюстрація нижче показує взірці вибірок у Blender'і для 5, 8, 11 та 16 вибірок. Для показу того, що розподіл вирівнюється по кільком пікселям, також прорисовані взірці сусідніх пікселів. Зауважте, що кожен піксель має ідентичний взірець.

Фільтрування -- Filtering¶

Після зображування, рендерингу вибірок ми отримали інформацію про колір та альфа для кожної вибірки. Далі важливо визначити, наскільки кожна вибірка робить внесок для пікселя.

Найпростіший метод -- усереднити усі вибірки та зробити результат кольором пікселя. Це називається -- скористатися «коробковим -- Box фільтруванням». Недоліком цього методу -- він не враховує те, що деякі вибірки є дуже близько до краю пікселя, і, звідси, можуть також впливати на колір сусіднього(іх) пікселя(ів).

Меню «Фільтр Пікселя» -- Pixel Filter дозволяє обрати тип для «усереднення» вибірок:

Коробка -- Box

Низько якісна коробково-формова крива.

Ghi chú

Цей фільтр дає відносно низьку якість. Ви можете бачити, що вибірки у межах тільки самого пікселя додаються у колір пікселя. Для інших фільтрів їх формула гарантує, що певна кількість кольору вибірки розподіляється також на інші пікселі.

Розмір фільтру -- Filter Size¶

Зменшення значення «Розмір» -- Size для фільтру буде стискати вибірки більше до центру та більше розмивати зображення. Більший Розмір фільтру дає різкіший результат. Зауважте, що два останні фільтри також мають від'ємну частину; це дає результат з додатковою різкістю.
