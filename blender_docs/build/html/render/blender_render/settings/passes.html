Проходи Зобража -- Render Passes¶

Render Passes are necessary because of the different things the Blender Renderer must calculate to give you the final image. In each 'pass' the engine calculates different interactions between objects.

Проходи зобража у деталях¶

Все, що ви бачите при зображуванні, рендерингу, повинно бути обчислене для фінального зображення. Усі взаємодії між об'єктами у вашій сцені, освітлення, камери, фонові зображення, устави світу тощо повинні всі бути окремо обчислені у різних проходах з різних причин, таких, зокрема, як обчислення тіней.

При зображуванні кожен піксель обчислюється кілька разів для гарантування того, що він показуватиме правильний колір для правильної частини зображення. Різні речі, що обчислюються при стандартному зображуванні, включають:

  • Де відкидають тіні?
  • Як освітлення оточення у середовищі блокується (загороджується) об'єктами у сцені?
  • Як світло відбивається від дзеркальних поверхонь? Як і тіні, промені обчислюються, за винятком того часу, коли вони йдуть від камери та відбиваються від дзеркальних поверхонь, так, що коли ці промені вдаряються в об'єкт, то рушій обчислює це, як те, що камера повинна бачити.
  • How is light bent (refracted) as it passes through transparent objects? Does it go straight through? Does it bend? If so, at what depth in the object?
  • Які визначені об'єкти є у сцені та яких у них контур? Чи повинні ці об'єкти виглядати розмитими або чи мають вони виглядати у різкому фокусі?
  • Наскільки швидко щось рухається (скорість)? Повинно це виглядати розмитим при заданій частості кадрів або воно є настільки повільним, щоб бути все ще фокусованим?
  • How far away from the camera are objects' surfaces (Z depth)? Can the object's surfaces be seen at all, or are they being blocked by another object's geometry?
  • Does an object have a normal vector (bump map)? Do shadows and apparent geometry need to be calculated for any objects?
  • Is there any specularity? Are objects with textures such as metal shiny at all?

Відповідь на кожне з цих питань вище -- це зображення або карта, розкладка даних, як показано нижче:

Each Render Pass puts out an image or a map. For the purposes of this example, a Render Layer was defined to produce all possible outputs. When a Render Layer input node was added to the node diagram and the Render Layer node was subsequently associated with the Render Layer, all of the layer's outputs appeared as connection points on the right-hand (output) side of the node.

Render Passes that produce Images can be directly viewed in a viewer, or, if they are the only pass that is rendered, saved as the render image. If the pass is enabled, it can be saved in a multi-layer OpenEXR format.

Якщо вивід Прохода Зобража не є зображенням, а картою, розкладкою даних, то він має бути перекладено у щось, що ми можемо бачити. Наприклад, карта глибини Z -- це масив значень, що визначають, наскільки далеко від камери є кожен піксель; значення у діапазоні між +/-3,000,000 Одиниць Blender або біля того. Проміжний вузол, який ви бачите вище, між вивідним роз'ємом вузла Render Layers та увідним роз'ємом вузла Viewer, Map Value робить цей переклад або співвідношення. Ви повинні використати цей специфічний тип вузла перекладу для отримання хороших результатів, якщо намірені оперувати такою картою, як зображенням. Ви повинні далі, після зроблення будь-яких коригувань, запустити карту назад через таких вузол для повторного встановлення відношень назад в оригінал перед збереженням.

Вибирання проходів зобража¶

Проходи Зобража -- це різні відмінні виводи, які зображувач, рендерер здатний генерувати. Усі наступні виводи зобража зазвичай комбінуються в єдиний вивід, відомий, відповідно, як комбінований вивід Combined. Але ви можете також вибрати будь-який з них для виводу як окремий прохід. (Якщо ви так зробите, у більшості випадків ви можете обрати, чи також продовжувати включати його у вивід Combined).

Деякі з цих виводів повинні бути увімкненні та використовуватися у межах вашої сцени (а не тільки вибраними на у вкладці Render Layers), щоб хоч щось було видно на зображі. Наприклад, якщо ви не маєте жодних освітлювачів у вашій сцені або ці освітлювачі установлено так, що вони не відкидають тіней, або об'єкти у центрі уваги не мають матеріалів, які б приймали тіні, то прохід Shadow буде порожнім; просто жодної інформації щодо тіней на зображі показано не буде. Якщо ви не увімкнули опцію Ambient Occlusion в уставах World, то прохід AO буде порожнім, навіть якщо його увімкнено.

Для заощадження часу та простору на диску ви маєте повідомити Blender'у про кожен з проходів, що він має рендерити) у вкладці Render Layers (про яку ми говорили тут -- the previous page):

Комбіновано -- Combined
This renders everything in the image, even if it is not necessary. ("The whole enchilada", so to speak). This is all the options below, blended into a single output, except those options which you have indicated should be omitted from this pass, as indicated with the camera button.
Z
The Z-depth map; how far away each pixel is from the camera. Used for depth of field (DOF). The depth map is inverse linear (1/distance) from the camera clip start.
Вектор -- Vector
Напрямок та швидкість речей, що рухаються у кадрі. Використовується разом Векторним Розмивом -- Vector Blur.
Нормаль -- Normal
Calculates lighting and apparent geometry for a bump map (an image which is used to fake detail on an object) or for changing the apparent direction of light falling on an object.
UV
Дозволяє текстурування після зображування, рендерингу. Дивіться інформацію про вузол UV.
Імла -- Mist
Видає прохід фактора Mist.
Індекс Об'єкта -- Object Index
Маскує обрані об'єкти. Дивіться детальніше тут -- Вузол «Маска за ID» -- ID Mask Node.
Колір -- Color
Колір матеріалів без відтінення.
Розсіяння -- Diffuse
Розсіяне, дифузне відтінення матеріалів.
Блікування -- Specular
Блікові підсвітки.
Тінь -- Shadow
Відкидання тіней. Переконайтеся, що увімкнено відкидання тіней вашими освітлювачами та приймання їх матеріалами. Використовуйте цей прохід, змішуючи через множення з проходом Diffuse.
Емітування -- Emit
Прохід емісії.
Загорода Оточення -- Ambient Occlusion
Переконайтеся, що загорода увімкнена для Світу сцени та що увімкнено Простеження Променя -- Ray Tracing.
Середовище -- Environment
Освітлення середовища.
Побічне -- Indirect
Прохід побічного, непрямого освітлення.
Відбиття -- Reflection
Відбиття від дзеркальних та інших відбивних поверхонь (високо вощені білі підлоги, наприклад). Змішуйте через Add з проходом Diffuse.
Заломлення -- Refraction
Заломлення, рефракція кольорів через прозорі сіті. Змішуйте через Add з проходом Diffuse.

Після вмикання цього проходу відповідний роз'єм на вузлі Render Layers показується як магія та може використовуватися, як показано на прикладі вище.

Виключення проходів зобража¶

Як ми вже говорили, вивід «Комбіновано» -- Combined є поєднанням різних виводів, які також доступні окремо. Коли ви вибираєте один з цих виводів, то вони подаються окремо, а також включно у проході Combined.

When you enable the camera icon that is beside several of the pass options, the particular pass will be excluded from the combined pass. They will be made available separately but not included in the combined pass.

Використання проходів зобража¶

Первинне призначення Проходів Зобража -- дозволити вам обробляти різні виводи різними способами, будуючи мережі вузлів зображування. Ви можете домогтися багато спеціальних ефектів та значно зекономити час на рендеринг складних сцен, творчо та ефективно використовуючи цю можливість. Ми покажемо вам кілька прикладів цього лише за мить.

Чимало інформації про типові використання деяких з проходів також в інших місцях цього посібника:

  • Зображення -- Image: Оскільки це головний продукт, все у Blender'і використовує його.
  • Альфа -- Alpha: Дивіться про вузол Alpha Over та усі вузли категорії Matte.
  • Z: Дивіться про вузол Defocus.
  • Вектор -- Vector: Дивіться про вузол Vector Blur.
  • Нормаль -- Normal: Дивіться про вузол Normal.

Перезабарвлення тіней¶

Давайте запустімо буфер Тіні через уклад вузлів для забарвлення, далі перекомпонуємо його; усі ваші тіні будуть штучно забарвлені. Тут на ілюстрації показано багато потоків у цьому укладі вузлів, а тому давайте пройдімося по ньому. Зліва увідний вузол Render Layers: він стосується одного з шарів зобража, що ми визначити для нашої сцени. У сцені ми маємо відбивний м'яч на п'єдесталі, що стоїть попереду фону. Все (за винятком м'яча) є сірим. Ми використали стандартну оснастку з чотирьох освітлювачів: заднє заповнення поміщене високо, два бокових заповнення на рівні основи та ключовий освітлювач вище та зліва від камери. Сюзанна, геометрія мавпячої форми, стоїть перед ключовим освітлювачем, так що її тінь відкидається на підлогу сцени. М'яч відкидає тіні на фон та підлогу.

Вивідні канали для вузла Render Layers було визначено на однойменній вкладці Редактора Властивостей. Два зверху вузли Viewer показують вам зображення виводу, що використовує тінь Shadow як Alpha канал, а вузол біля них -- тільки канал тіні Shadow. Там, де Тінь є темною, зображення на лівому Viewer є прозорим. Ми використали прохід Shadow для відрізання частин зображення.

Далі ми беремо тінь через вузол RGB Curve, який установлює просте збільшення Blue на 75%; так що сіра тінь RGB(40, 40, 40) стає RGB(40, 40, 40×1.75=70). Така підбарвлена синім тінь показується на нижньому Viewer. Тепер ми маємо два варіанти: Alpha Over та Mix. Для обох варіантів:

  • Використовується карта Тіні як Фактор.
  • Подається Синя Тінь на Верхній Роз'єм.
  • Подається базове зображення на Нижній Роз'єм.

Результатне зображення однакове у будь-якому випадку; синя тінь. Зверніть увагу, що відбиття від Сюзанни не є синім; там для цього є інший Прохід Зобража.

You could just as easily swap in another image entirely; for example, the shadow map from another render layer. You can even take an image from another project entirely and use that instead (by using the Image Input node), to get a different effect. (For example, an effect similar to a Star Wars Episode One movie poster, where Anakin Skywalker already casts the shadow of Darth Vader).

Компонування загороди оточення¶

AO is a geometry-based dirt shader, making corners darker. It is separately enabled in the World settings and computed as a separate pass. When enabled, it has one of three Modes: Add, Subtract, Both, and a variable Energy level (which changes the intensity of the shading). The third variable is the amount of Ambient light that the material receives. If it does not receive any, then ambient occlusion does not affect it. Based on these variables, Blender computes an AO pass. If you call it out as a separate pass and wish to composite it back into your image, you will need to enable the Color and Diffuse pass as well.

Для конфігурування вашого укладу розгляньте приклад на ілюстрації тут.

  1. Спершу, залежно від обраного режиму АО зробіть наступне: Якщо режим АО -- Add: прямо використовуйте прохід АО. Якщо режим АО є Sub: Обчислюєте як AO - 1, а якщо режим АО -- Both: Обчислюйте як 2 × AO - 1.
  2. Помножте вивід Кроку 1 на рівень Energy для АО.
  3. Помножте вивід Кроку 2 на значення Ambience для матеріалу. Якщо у вас є матеріали, які приймають різні значення Ambience (0.5 є стандартом), потрібно створити карту оточення -- ambience map на основі Object ID.
  4. Помножте вивід Кроку 3 на прохід Color.
  5. Додайте вивід Кроку 4 до проходу Diffuse.

Якщо задіюються тіні, забарвлені освітленням оточення, бліковістю, відбиттями та/або заломленнями, то їх значення потрібно додати до проходу Diffuse перед доданням до конвертованого проходу АО.

Тіні векторного розмиву¶

При використання Векторного Розмиву -- Vector Blur замість Розмиву Рухом -- Motion Blur, об'єкти у русі є також розмитими, але об'єкти у спокої (відносно камери) не розмиваються. Схема вузлів на ілюстрації тут -- це тінь об'єкта у русі. Для цього прикладу ми маємо куб у русі по площині основи. Якщо ми просто пропустимо прохід Combined через вузол Vector Blur, то зможемо побачити результат у нижньому правому кутку; коробка є розмитою, але тінь від неї є різкою у фокусі, і, звідси, зображення не виглядає реалістично.

Тому, ми потребуємо відокремити проходи Diffuse та Shadow від підлоги, створивши шар зобража "Floor". Цей шар зобража має увімкненими проходи Diffuse та Shadow та рендерить тільки об'єкт підлоги (шар 2). Інший шар зобража ("Cube") рендерить проходи Z та Vector та зображує тільки куб (на шарі 1). Використавши вузол Blur, ми розмиваємо прохід Shadow і далі комбінуємо розсіяння та розмиту тінь, помноживши їх між собою на вузлі Mix Multiply; тепер ми отримали розмиту тінь на чіткій площині основи. Далі ми змішуємо її з векторно-розмитим об'єктом для забезпечення реалістичного на вигляд зображення.

Висновок¶

Проходами Зобража можна маніпулювати для отримання майже повного контролю над вашим фінальним зображенням. Спричинення об'єктів відкидати тіні, що реально не мають власних тіней, примушення об'єктів бути поза фокусом або у фокусі, як і при реальній камері, маніпулювання кольорами тільки для фінального після оброблення або лише для переконфігурації проходів зобража для заощадження часу -- все це є тим, чим можливо ви захочете маніпулювати для рушія зображування, рендерингу.
