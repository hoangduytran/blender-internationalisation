Ray-Traced Shadows¶

Опції затінення на основі променя для ламп.

Більшість типів ламп («Точка» -- Point, «Прожектор» -- Spot та «Сонце» -- Sun) мають загальними деякі опції для генерування промінь-простежених тіней, які описуються тут нижче. Зауважте, що лампа типу «Площина» -- Area, навіть хоча використовує більшість з цих опцій, має деякі особливості для неї, описані тут -- own ray-traced shadows page.

Тінь Променя -- Ray Shadow
Кнопка «Тінь Променя» -- Ray Shadow вмикає для цього джерела освітлення генерування промінь-простежених тіней. Коли ця кнопка Ray Shadow активована, то ще й інший набір опцій стає для неї доступним:
Тип генератора вибірки тіні -- Shadow sample generator type

Метод для вибірок генерування тіні: Adaptive QMC є найшвидшим, Constant QMC є менш шумним, але повільніший. Це дозволяє вам вибрати, який алгоритм буде використовуватися для генерування вибірок, що будуть слугувати для обчислення промінь-простежених тіней (наразі, доступні два варіанти алгоритму Квазі Монте-Карло, дивіться детальніше -- Quasi-Monte Carlo method):

Постійний QMC -- Constant QMC
Метод Constant QMC використовується для обчислення значень тіні дуже однорідним, рівномірно розподіленим чином. Цей метод дає дуже добре обчислення значення тіні, але не є таким швидким, як при використанні методу Adaptive QMC; хоча, Constant QMC є більш точним.
Адаптивний QMC -- Adaptive QMC
Методи Adaptive QMC використовується для обчислення значень тіні у дещо менш однорідний та розподілений спосіб. Цей метод дає добре обчислення значення тіні, але не настільки, як Constant QMC. Перевага використання Adaptive QMC полягає у тому, що загалом він більш швидкий, при цьому, дає, у загальному не набагато гірші результати, ніж Constant QMC.
Вибірки -- Samples
Кількість взятих додаткових вибірок (вибірки на вибірки). Цей повзунок задає максимальну кількість вибірок, яку обидва Constant QMC та Adaptive QMC будуть використовувати для здійснення обчислення тіні. Максимальне значення -- 16: реальна кількість вибірок фактично є піднесеним до квадрату значенням, а тому значення 3 реально означає 32 = 9 вибірок, що будуть відібрані.
Розмір М'якості -- Soft Size

Розмір світла для відбору тіні променя. Цей повзунок визначає розмір зони розпливчастості/розсіяння/півтіні навколо краю тіні. Soft Size визначає тільки ширину розміру м'якої тіні, а не те, наскільки тінь є градуйованою або згладженою. Якщо ви хочете, щоб широка тінь була також м'якою та гарно градуйованою, ви повинні також задати кількість вибірок для устави Samples більше, ніж 1; інакше ця устава не матиме видимого ефекту, а генеровані тіні не матимуть м'якого краю. Максимальне значення для Soft Size -- 100.0.

Нижче наведена таблиця зображів з різними значеннями устав Soft Size та Samples, яка показує різні рівні м'якості країв тіні:

Soft Size: 1.0, Samples: 2.

Soft Size: 1.0, Samples: 4.

Soft Size: 1.0, Samples: 6.

Soft Size: 2.0, Samples: 2.

Soft Size: 2.0, Samples: 4.

Soft Size: 2.0, Samples: 6.

Soft Size: 3.0, Samples: 2.

Soft Size: 3.0, Samples: 4.

Soft Size: 3.0, Samples: 6.
Поріг -- Threshold
Поріг для адаптивного відбору. Ця устава використовується методом обчислення тіні Adaptive QMC. Це значення використовується для визначення, чи вибірне обчислення тіні Adaptive QMC може пропускатися на основі порогу того, наскільки вже є затіненою область. Максимальне значення Threshold -- 1.0.

Метод Квазі Монте-Карло -- Quasi-Monte Carlo¶

Метод Монте-Карло -- Monte Carlo -- це метод взяття серії вибірок/читань значень (значення будь-якого роду, такі як значення світла, кольору, стани відбивань) у або навколо області у довільному порядку, так, щоб при визначенні коректних дій для взяття у певні обчислення, які зазвичай вимагають одночасно багато значень вибірок, визначалася загальна точність цих обчислень. Метод Монте-Карло намагається бути максимально випадковим; це часто може призводити до того, що області, які були відібрані, мають великі нерегулярні проміжки між ними (місця, що не відібрані/зчитані). Це, у свою чергу, може спричиняти проблеми для певних обчислень (таких як обчислення тіні).

Вирішенням цього стало знайдення методу Квазі Монте-Карло -- Quasi-Monte Carlo.

Метод Квазі Монте-Карло -- Quasi-Monte Carlo також є випадковим, але намагається гарантувати, що вибірки/зчитування також мають бути краще розподіленими (залишаючи менш нерегулярні проміжки в областях їх вибірок) і більш поширеними по області. Це надає перевагу, оскільки інколи дає більш точні обчислення на основі вибірок/зчитувань.
