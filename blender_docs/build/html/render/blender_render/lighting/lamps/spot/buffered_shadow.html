Буферовані тіні Прожектора -- Spot Buffered Shadow¶

Увімкнено режим «Буферна Тінь» -- Buffer Shadow для лампи «Прожектор» -- Spot.

Spotlights can use either Ray-traced Shadows or buffered shadows. Either of the two can provide various extra options.

Ray-traced shadows are generally more accurate, with extra capabilities such as transparent shadows, although they are quite slower to render.

Буферовані тіні є більш складними для укладання та вимагають більше хитрування, але швидкість їх зображування, рендерингу є визначальною перевагою їх вживання. Як би там не було, прожектор має спільні, як і для інших типів ламп, опції тіней, що описані тут -- Панель «Тінь» -- Shadow Panel.

Типи буферних тіней¶

When the Buffer Shadow button is activated, the currently selected Spot light generates shadows, using a "shadow buffer" rather than using ray tracing, and various extra options and buttons appear in the Shadow panel.

Тип Буфера -- Buffer Type
Існує більше ніж один спосіб генерування буферованих тіней. Тип буфера генерування тіні керує тим, який буде використовуватися генератор.
Є чотири типи генерування тіні:
  • Класично -- Classical
  • Класично-Усереднено -- Classic-Halfway
  • Нерегулярно -- Irregular
  • Глибоко -- Deep

Детальніше про ці різні методи генерування тіні дивіться інформацію за наступними посиланнями:

  • Development Release Logs 2.43: Irregular Shadow Buffer.
  • Blender Nation: Blender Gets Irregular Shadow Buffers.
  • Development Release Logs 2.43: Shadow Buffer Halfway Average.

«Класично» -- "Classical" та «Класично-Усереднено» -- "Classic-Halfway"¶

Тип буфера для Буферної Тіні -- «Класично-Усереднено» -- Classic-Halfway.

Класично -- Classical

Даний тип генерації тіні раніше використовувався Blender'ом як стандарт та є унікальним методом генерування буферованих тіней. Він використовував старий спосіб генерування буферованих тіней, але мав певні проблеми з точністю генерованих тіней і міг бути дуже чутливим до роздільності розміру буфера тіні -- Shadow Buffer ‣ Size, різних значень Bias, а також мав усі проблеми щодо само-затінення.

Метод генерування тіней Classical є застарілим та фактично ще присутній тут лише для зворотної сумісності зі старшими версіями Blender'а. У більшості інших випадків вам бажано використовувати замість нього метод Classic-Halfway.

Класично-Усереднено -- Classic-Halfway

Цей тип буфера тіні є покращеним методом буферування тіні та є наразі стандартного вибраною для цього опцією у Blender'і. Він працює, беручи усереднене зчитування першого та другого найближчих значень глибини Z, що дозволяє знизити значення устави Bias та не мати багато проблем із само-затіненням.

Відсутність необхідності збільшення значень Bias допомагає з точністю тіні, оскільки великі значення Bias можуть означати, що малі грані можуть втрачати їх тіні, а також давати надмірний зсув тіней.

Classic-Halfway не працює дуже добре при перекриванні граней і можуть траплятися проблеми з відхилянням.

Наразі такі опції є специфічні для цих методів генерування:

Розмір -- Size

Кнопка з числом Size може мати значення від (512 до 10240). Size представляє роздільність, що використовується для створення розкладки, карти тіні. Ця карта, розкладка тіні далі використовується для визначення місць, де лягатимуть тіні у межах сцени.

Як приклад, якщо ви маєте Size зі значенням 1024, то це для вас вказує, що дані цієї тіні будуть записані у буфер, що матиме квадратну роздільність 1024×1024 пікселів/вибірок з даного прожектора.

Чим вище значення Size, тим вище роздільність та точність результатних тіней, при цьому припускається, що всі інші властивості освітлювача і сцени залишаються без змін, хоча, при цьому, буде використовуватися більше пам'яті та часу на обчислення. Зворотне також вірно -- якщо значення Size знижене, то результатні тіні можуть мати нижчу якість, але буде використовуватися менше пам'яті та часу на обчислення.

Окрім значення Size, вплив на якість генерованих тіней має й інша властивість ламп типу Spot -- кут прожектора (що задається однойменною уставою Size на панелі Spot Shape).

При збільшенні значення Size на панелі Spot Shape якість відкинутих тіней знижується. Це стається тому, що, коли світлова область лампи Spot робиться більшою (через збільшення розміру форми прожектора), то й область буферу тіні розтягується та масштабується, щоб вписатися у розмір нової освітленої області.

Устава Size на панелі Shadow для роздільності буферованої тіні не змінюється, щоб компенсувати розмір світлової плями, а тому якість тіней знижується. Якщо ви хочете зберегти для генерованих тіней ту ж якість, то при збільшенні значення Size на панелі Spot Shape ви повинні також збільшити відповідно значення Size на панелі Shadow для буфера тіні.

Вищенаведене базово зводиться до такого

Якщо ви маєте світлову пляму прожектора, яка збільшена, то вам необхідно мати збільшений розмір Size і для буфера тіні, щоб зберегти для тіней добру якість. Зворотне також вірно -- якість генерованих тіней буде зазвичай покращуватися (до певного моменту) при зменшенні площі світлової плями лампи Spot.

Тип Фільтра -- Filter Type

Типи фільтрів Box, Tent та Gauss керують тим, як алгоритм фільтрування використовується для загладжування буферованих тіней.

Вони тісно пов'язані зі значенням устави Samples, оскільки при заданні для неї значення як 1 фільтрування тіні вимикається та жоден з цих фільтрів не матиме жодного ефекту.

Коробка -- Box
Буферовані тіні будуть загладжуватися за допомогою методу фільтрування "box". Це оригінальний фільтр, використовуваний у Blender'і. Він дає відносно низьку якість та використовується для низько роздільних зображів, рендерів, оскільки продукує дуже різке загладжування. Коли це фільтр використовується, то враховуються тільки ті дані перебору, які потрапляють у межі одиничного пікселя, та не враховуються вибірки навколишніх пікселів. Це часто корисно для зображень, які мають елементи з гострими кутами та горизонтальними/вертикальними лініями.
Намет -- Tent
The buffered shadows will be anti-aliased using the "tent" filtering method. It is a simple filter that gives sharp results, an excellent general-purpose filtering method. This filter also takes into account the sample values of neighboring pixels when calculating its final filtering value.
Гаус -- Gauss
Буферні тіні будуть загладжуватися за допомогою методу фільтрування "Gaussian". Він продукує дуже м'яке/розмите загладжування. Як результат, цей фільтр є відмінним для високо роздільних зображів, рендерів.

Більше інформації про різні методи фільтрування/розподілу дивіться тут -- Anti-Aliasing page.

Вибірки -- Samples

Числова кнопка Samples може мати значення між (1 та 16). Вона керує кількість вибірок, взятих на піксель, при обчисленні карт, розкладок тіней.

Чим вище це значення, тим будуть більш фільтрованими, згладженими та загладженими тіні, відкинуті для поточної лампи, але й потрібно буде більше пам'яті та часу на їх обчислення. Використовуваний метод загладжування визначається активованою кнопкою Box, Tent або Gauss (дивіться вище).

Наявність значення Samples як 1 аналогічна вимкненню загладжування для буферованих тіней.

М'якість -- Soft
Кнопка з числом Soft може мати значення між (1.0 до 100.0). Вона визначає, наскільки широко вибирається область при здійсненні загладжування буферованих тіней. Чим більше значення Soft, тим більш градуйована/м'яка область, що загладжується/пом'якшується, на краях генерованих тіней.
Буфери Вибірок -- Sample Buffers

Устава Sample Buffers може мати значення (1, 4 або 9) та представляє кількість буферів тіні, що буде використовуватися при здійсненні загладжування буферованих тіней.

Ця опція використовується в особливих випадках, зокрема, коли маленькі об'єкти рухаються та потребують генерування дійсно маленьких тіней (наприклад, пасма, нитки). Це з'являється, коли звичайним чином тіні з шириною у піксель не загладжуються належно, збільшення Buffer Size більше не допомагає.

Тому, ця опція дозволяє вам мати свого роду додатковий прохід вибірки, що здійснюється поверх звичайного (який керується уставами Box / Tent / Gauss, Samples та Soft).

Стандартне значення 1 вимикає цю опцію.

Вищі значення продукують плавніше загладжування -- але будьте обережні: використання значення Sample Buffers як 4 буде вимагати у чотири рази більше пам'яті та часу на обчислення, оскільки Blender'у прийдеться таку кількість раз обчислювати кожну вибірку.

«Нерегулярно» -- "Irregular"¶

Тип буфера для Буферної Тіні -- «Нерегулярно» -- Irregular.

Irregular shadow method is used to generate sharp/hard shadows that are placed as accurately as ray-traced shadows. This method offers very good performance because it can be done as a multi-threaded process.

Цей метод підтримує прозорі тіні. Для цього вам спершу необхідно укласти властивості -- опцію «Відкидати Буферні Тіні» та уставу «Відхил Буфера», для об'єкта, який буде отримувати прозору тінь -- Material ‣ Shadow ‣ Cast Buffer Shadows and Buffer Bias.

Метод генерування «Глибоко» -- Deep¶

Тип буфера для Буферної Тіні -- «Глибоко» -- Deep.

Типу буфера тіні «Глибоко» підтримує прозорість та краще фільтрування, за рахунок більших затрат пам'яті та часу на обчислення.
Стиснення -- Compress
Поріг стиснення карти, розкладки глибоких тіней.

Загальні опції¶

Наступні устави є загальними для усіх методів генерування буферованої тіні.

Відхил -- Bias

Кнопка з числом Bias може мати значення між (0.001 to 5.0). Bias використовується для додання невеликого зсуву відстані між об'єктом та тінню, яку він відкидає. Це іноді необхідно, через неточності в обчисленні, яке визначає чи є ця область об'єктом, чи є тінню.

Making the Bias value smaller results in the distance between the object and its shadow being smaller. If the Bias value is too small, an object can get artifacts, which can appear as lines and interference patterns on objects. This problem is usually called "self-shadowing", and can usually be fixed by increasing the Bias value, which exists for that purpose!

Other methods for correcting self-shadowing include increasing the size of the Shadow Buffer Size or using a different buffered shadow calculation method such as Classic-Halfway or Irregular.

Self-shadowing interference tends to affect curved surfaces more than flat ones, meaning that if your scene has a lot of curved surfaces it may be necessary to increase the Bias value or Shadow Buffer Size value.

Наявність надмірно великих значень Bias не тільки розмістить тіні дуже далеко від об'єктів, що їх відкидають, але може також те, що дуже маленькі об'єкти взагалі не будуть відкидати тіні. У такий момент для виправлення цієї проблеми може вимагатися, серед інших, і зміна значень Bias, Shadow Buffer Size або Spot Size.

Тонке підлаштування Bias

Тепер ви можете тонко підлаштовувати Bias окремо для кожного матеріалу -- Material, використовуючи повзунок Buffer Bias (вкладка Material, панель Shadow). Це значення є фактором, на який множиться значення устави Bias буферованої тіні кожної лампи типу Spot кожен раз, коли світло від неї потрапляє на об'єкт з цим матеріалом. Значення (0.0 та 1.0) є еквівалентними. Вони не змінюють оригінальне значення Bias для лампи.

Старт Відсіку -- Clip Start та Кінець Відсіку -- Clip End

При увімкненні Буферованої Тіні для лампи Spot на візуальної її представленні у 3D Огляді -- 3D View з'являється додаткова лінія.

Стартова точка цієї лінії представляє значення устави Clip Start, а кінцева точка -- значення устави Clip End. Clip Start може мати значення між (0.1 до 1000.0), а Clip End -- між (1.0 до 5000.0). Обидва значення вимірюються в Одиницях Blender.

Clip Start вказує на точку, після якої буферовані тіні можуть бути присутніми у межах світлової області лампи Spot. Будь-яка тінь, яка може бути представлена перед цією точкою, ігнорується та жодна тінь не генерується.

Clip End вказує на точку, після якої буферовані тіні не будуть генеруватися у межах світлової області лампи Spot. Будь-яка тінь, яка може бути представлена після цієї точки, ігнорується та жодна тінь не генерується.

Область між Clip Start та Clip End буде здатна мати генеровані буферовані тіні.

Зміна значень Clip Start та Clip End допомагає керувати тим, де тіні можуть бути генеровані. Зміна діапазону між Clip Start та Clip End може допомогти пришвидшити зображування, рендеринг, заощадити пам'ять та зробити результатні тіні більш точними.

При використанні лампи Spot з буферованими тінями для збереження або покращення якості генерованих тіней корисно коригувати ці Clip Start та Clip End, оскільки їх значення тісно пов'язані з областями, в яких генеруються тіні. Мінімізація діапазону між Clip Start та Clip End мінімізує область, в якій обчислюються тіні та, звідси, може допомогти збільшити якість тіні у більш обмеженій області.

Старт Авто відсіку -- Autoclip Start / Кінець Авто відсіку -- Autoclip End

Окрім ручного задання устав Clip Start та Clip End для керування тим, де буферовані тіні починаються і закінчуються, також є можливим дозволити Blender'у автоматично самостійно вибирати найкраще значення для кожної з цих Clip Start та Clip End устав.

Blender робить це, переглядаючи, де знаходяться видимі вершини з позиції лампи Spot.

Поради¶

Будь-який об'єкт у Blender'і може діяти як камера у 3D Огляді -- 3D View. Звідси, ви можете виділити освітлювач Spot та перемкнутися на погляд з його точки зору, натиснувши Ctrl-Numpad0.
