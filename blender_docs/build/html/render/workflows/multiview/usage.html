Уклад Оглядів -- Views Setup¶

Перейдіть на панель Render Layers та увімкніть Views для цієї сцени.

Scene render views.

Ghi chú

При увімкненні вами Views для сцени ви отримуєте передогляд 3D в оглядвікні, а також кілька панелей, що відтепер доступні по всьому інтерфейсу користувача.

Оглядвікно з візуалізацією 3D.

Камера -- Camera¶

Для підправлення параметрів стерео 3D виділіть камеру в Outliner. На панелі Camera перейдіть у вкладку Stereoscopy та змініть устави Відстані Збігання -- Convergence Distance.

Оглядвікно буде відповідати у реальному часі на ці зміни, дозволяючи вам передоглядати поточне значення глибини сцени.

Stereo convergence distance.

Оглядвікно -- Viewport¶

Перед тонким підлаштуванням параметрів камери ви можете установити площину збігання в оглядвікні на основі розстави глибини вашої сцени. Вийдіть з огляду камери й ви одразу побачите площину збігання перед камерою.

Ви можете перемикати цю та інші устави показу на панелі Stereoscopy регіону Properties у 3D View. На наступній ілюстрації також увімкнено видимість об'ємів усічених пірамід камери.

Viewport plane and volume stereo preview.

Показ стерео 3D¶

Якщо ви маєте справжній 3D дисплей, то у певний момент ви можете змінити режим показу 3D у меню Window, викликавши оператор Stereo 3D. Майте на увазі, що деякі режими вимагають повноекранного режиму для роботи.

Window menu, stereo 3D operator.

Передогляд з OpenGL -- OpenGL Preview¶

Перед рендерингом вашої сцени ви можете зберегти передогляд з OpenGL анімації для тестування при фінальному показі. На панелі Render: Output ви можете обрати потрібний вивід із секції Views Format.

Опції тут включають окремі файли для кожного огляду, зверху-вниз, анагліф, серед інших. Виберіть той, що відповідає вашим вимогам показу.

Рендеринг та UV/Image Editor¶

Як тільки ви задоволені результатами, ви можете рендерити фінальну анімацію. В UV/Image Editor ви можете перевірити окремі огляди та результат стерео.

Формати зображень -- Image Formats¶

Ваша фінальна анімація може бути збережена у більш надійних форматах, ніж ті, що використовуються передоглядом рендера OpenGL. У цьому прикладі ми зберегли як навскісне пліч-о-пліч стерео 3D.

Side-by-side cross-eye format.

Фінальні міркування¶

Як цей посібник показав, що рендеринг стерео 3D -- це більше, ніж просте генерування двох зображень. Чим раніше у конвеєрі буде укладено устави стерео, тим плавнішим буде його процес. Наступні секції є більш глибоким оглядом окремих компонентів, які ми побачили при огляді ходу роботи.

Показ Стерео 3D у Вікні -- Window Stereo 3D Display¶

Важливим компонентом конвеєру Стереоскопії є можливість показу стерео зображення на належному дисплеї. Blender підтримує від високотехнологічних 3D дисплеїв до простих червоно-блакитних окуляр. До цього всього, ви можете задавати різний режим показу для кожного вікна.

Режим показу можна змінити через меню Window або створене вами самостійно скорочення для оператора wm.set_stereo_3d.

Window menu, stereo 3D operator.

Режим Показу -- Display Mode¶

Анагліф -- Anaglyph
Рендерить два по-різному фільтровані кольорові зображення для кожного ока. Вимагаються анагліфні стереокуляри. Ми підтримуємо стереокуляри Red-Cyan, Green-Magenta та Yellow-Blue.
Чергованість -- Interlace
Рендерить два зображення для кожного ока в одне черговане зображення. Вимагається монітор з підтримкою 3D. Ми підтримуємо Row, Column and Checkerboard Interleaved. Опція обміну Swap Left/Right допомагає відкоригувати зображення на екрані. Цей метод працює краще у повному екрані.
Послідовно у Часі -- Time Sequential
Рендерить почергово огляди для очей -- зображування одного огляду дається в парних кадрах, іншого огляду -- у непарних. Цей метод також відомий як «Перегортувана Сторінка» -- Page Flip. Це вимагає, щоб відеокарта підтримувала почетверений буфер -- Quad Buffer, та працює лише у повному екрані.
Пліч-о-Пліч -- Side-by-Side
Рендерить зображення для лівого та правого ока пліч-о-пліч. Є опція для навскісних окулярів Cross-Eye. Це працює тільки у повному екрані та повинно використовуватися з оператором Full Editor.
Зверху-Вниз -- Top-Bottom
Рендерить зображення для лівого та правого ока одне над іншим. Це працює тільки у повному екрані та повинно використовуватися з оператором Full Editor.

Ghi chú

Режими повного екрана для стерео 3D

Якщо використовуєте 3D дисплей більшість часу і будете вживати його для перегляду у стерео 3D, то ви повинні перейти у повноекранний режим. Фактично, деякі режими стерео працюють тільки у повноекранному режимі, що ховає більшість інтерфейсу користувача. У цьому випадку рекомендується працювати з двома моніторами, використовуючи екран 3D для візуалізації результату стерео, тоді як інший екран може використовуватися для звичайної роботи у Blender'і.

Стерео 3D Камера -- Stereo 3D Camera¶

При використанні укладу огляду сцени Stereo 3D стереопара створюється на льоту та використовується для рендерингу та передвізуалізації. Для всіх цілей це працює як дві камери, що мають спільними більшість параметрів (фокусна відстань, відсікання ...). Камери у стерео парі, однак, зсунутими та можуть мати унікальне обертання та зміщення між собою.

Stereo 3D camera settings.

Міжочна Відстань -- Interocular Distance
Задає відстань між камерами у парі. Хоча збігання стерео пари може бути змінено на після обробці, різні міжочні відстані будуть продукувати різні результати через те, що частини сцени будуть загороджуватися з кожної точки зору.
Відстань до Площини Збігання -- Convergence Plane Distance
Точка збігання для стерео камери. Це часто відстань між проектором та екраном проектування. Ви можете візуалізувати це у 3D View.

Режим Збігання -- Convergence Mode¶

Поза-Осьове -- Off-Axis
Камери стереопари розділяються міжочною відстанню та можуть зсуватися у межах збігання збіжної площини. Це ідеальний режим збігання, оскільки він максимально наближений до того, як працює людське бачення.
Паралельно -- Parallel
Цей метод продукує дві паралельні камери, що не збігаються. Оскільки, цей метод потребує задання збігання вручну, то він не може використовуватися для перегляду. Цей метод є поширеним при комбінуванні реального фільмоматеріалу з рендереними елементами.
Збіжне -- Toe-in
Менш поширений підхід -- обертання камер замість зсування їх пірамід. Метод Toe-in рідко використовується у сучасних виробництвах 3D.
Опорточка -- Pivot
Стерео пара може бути вибудувана навколо активної камери з новою камерою для кожного ока (Center Pivot) або через використання наявних камер (Left або Right). Останнє використовується зазвичай для випадків, коли потрібне тільки одне око для зображування наявного 2D-моно проекту.

Стерео 3D в Оглядвікні -- Viewport Stereo 3D¶

При вмиканні вами 'Views' на у вкладці Render Layers нова панель стає доступною у регіоні Properties редактора 3D View. На цій панелі ви можете обрати, чи переглядати стерео 3D в оглядвікні, або яку камеру вжити для перегляду. Це також дозволяє вам бачити окремі Камери, Площину та Об'єм стерео камер.

Viewport stereo 3D settings.

Камери -- Cameras
При роботі з укладом оглядів для стерео 3D ви можете перевіряти, те, що бачить кожна окрема генерована камера, або комбінований результат з них. У режимі Multi-View ви можете бачити комбінований результат лівої та правої камер (коли доступно) або поточно виділеної камери.
Площина -- Plane
Площина збігання представляє екран, як він сприймається аудиторією. Візуалізація його у 3D Огляді дозволяє вам розставляти вашу сцену на основі глибини поза оглядом камери.
Об'єм -- Volume

Перетин пірамід стерео камер допомагає планувати показ, уникаючи елементів, що будуть видимі тільки для однієї камери. Об'єм визначається відстанями старту та кінця відсікання. Області, що знаходяться у піраміді тільки однієї камери відомі як «області суперництва сітчатки» -- retinal rivalry areas. Вони допускаються у від'ємному просторі (регіон від площини збігання до зображення), але повинні запобігатися за будь-яку ціну у додатному просторі (область від площини збігання до камери).

Viewport 3D: convergence plane and volume display.

Імпорт-експорт зображення Багато-Огляду та Стерео 3D -- Multi-View and Stereo 3D Image I/O¶

Багато-Огляд та Стерео 3D
Multi-view images can be saved in special formats according to the production requirements. By default the system saves each view as an individual file, thus generating as many files as views to be rendered. In stereo 3D productions, for the final deployment or even intermediary previews it is convenient to save stereo 3D images, that are ready to use with 3D displays or simple anaglyph glasses. The formats supported match the display modes available for the window.
Формати з втратами -- Lossy-Formats
Деякі формати стерео 3D представляються зі значною втратою даних. Наприклад, формат Anaglyph буде вирубати цілі колірні канали з оригінального зображення. Стиснений Top-Bottom буде відкидати половину ваших вертикальних даних роздільності. Формат Interlace буде значно розмішувати ваші дані. Після експорту у ці формати ви можете все ще імпортувати зображення назад у Blender для його обробки як Stereo 3D. Хоча, ви повинні узгодити режим показу стерео 3D у вікні та формат зображення стерео 3D.
Формати без втрат -- Lossless Formats
Деякі формати будуть зберігати оригінальні дані, що не призводить до жодних проблем при експорті та імпорті файлів назад у Blender. Опція Individual буде продукувати окремі зображення, що (якщо вони збережені з кодуванням без втрат як PNG або OpenEXR) можуть бути завантажені назад у виробництво без втрати жодних даних. Для форматів Stereo 3D варіантами без втрат є тільки Top-Bottom та Side-by-Side з вимкнутою опцією Squeezed Frame.
Multi-View OpenEXR
Another option is to use multi-view OpenEXR files. This format can save multiple views in a single file and is backward compatible with old OpenEXR viewers (you see only one view though). Multi-view native support is only available to OpenEXR.

Редактор Зображень -- Image Editor¶

Меню «Огляд» -- View Menu

Після того, як ви рендерили вашу сцену з Stereo 3D, ви будете у змозі бачити рендерений результат у комбінованому стерео 3D або зможете перевіряти окремі огляди. Це працює для вузлів Viewer, результатів рендера або відкритих зображень.

Stereo 3D and view menu.

Формат Оглядів -- Views Format

When you drag and drop an image into the UV/Image Editor, Blender will open it as an individual images at first. If your image was saved with one of the Stereo 3D formats, you can change how Blender should interpret the image by switching the mode to Stereo 3D, turning on Use Multi-View and picking the corresponding stereo method.

Views formats and stereo 3D.

Компонівник -- Compositor¶

The compositor works smoothly with multi-view images. The compositing of a view is completed before the remaining views start to be composited. The pipeline is the same as the single-view workflow, with the difference that you can use Images, Movies or Image Sequences in any of the supported multi-view formats.

Compositor, backdrop and Split Viewer node.

Огляди для зображування визначені у поточних оглядах сцени подібним чином, як ви визначаєте роздільність -- resolution на панелі «Зображ» -- Render для поточної сцени, незалежно від роздільностей для вузлів «Зображення» -- Image або «Шари Зображу» -- Render Layers з різних сцен.

Ghi chú

Зображення одиничного огляду

If the image from an Image node does not have the view you are trying to render, the image will be treated as a single-view image.

Вузол «Перемикач Оглядів» -- Switch View Node
If you need to treat the views separately, you can use the Switch View node to combine the views before an Output node.

Продуктивність -- Performance

Стандартно, при компонуванні та рендерингу з інтерфейсу користувача рендеряться та далі компонуються усі огляди. У ході повторів перевірки ви можете вимикати всі, окрім одного, огляди на панелі Views у вкладці Шари Зобража, та повторно вмикати їх, коли ви хочете отримати кінцевий вигляд.
