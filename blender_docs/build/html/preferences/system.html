Система -- System¶

Вкладка «Система» -- System дозволяє вам задавати роздільність, уподобання консолі скриптування, звуку, графічних карт та інтернаціоналізації.

Якщо ваше устаткування не підтримує деякі з опцій, описаних на цій сторінці, то вони або не будуть показані, або будуть скоректовані при запуску. Якщо це трапилося, не хвилюйтеся, ви можете проконсультуватися з підручником щодо вашого комп'ютера для знаходження підтримуваних значень або просто дозволити Blender'у скоректувати їх.

Вкладка «Система» -- System Уподобань Користувача.

Загальне -- General¶

Порт Сервера Кадрів -- Frame Server Port
TCP/IP порт, використовуваний у поєднанні з IP Адресою комп'ютера для рендерингу на сервері. Використовується при роботі з розподіленим рендерингом. Уникайте зміни значення цього порту, інакше це буде конфліктувати з уже наявними сервісними портами, що використовується вашою Операційною Системою та/або іншими програмами. Завжди консультуйтеся з документацією до вашої операційної системи та сервісів або проконсультуйтеся з вашим системним адміністратором перед змінюванням цього значення.
Прокрутка Консолі -- Console Scrollback
The number of lines, buffered in memory of the console window. Useful for debugging purposes and command-line rendering.

Звук -- Sound¶

Пристрій Звуку -- Audio Device

Визначає рушій звуку для використання в обробці та виводі аудіо.

Нема -- None
Без підтримки звуку (смужки звуку все ще можуть звичайно завантажуватися).
SDL
Використовує Simple Direct Media Layer API з libsdl.org для рендерингу звуку безпосередньо через вивід звукового пристрою. Дуже корисно для монтування смужок звуку у відеоряднику.
OpenAL
Забезпечує буферований рендеринг звуку з просторовою підтримкою/3D. Використовується для 3D джерела, що підтримується об'єктами типу «Гучномовець» -- Speaker Objects та ігровим рушієм Game Engine.

Sound Options¶

These settings control how sound behaves during live playback within Blender and are only available with SDL or OpenAL. To control these settings for exporting sound see the Encoding Panel and Audio Panel.

Канали -- Channels
Sets the audio channel count. Available options are: Mono, Stereo, 4 Channels, 5.1 Surround, 7.1 Surround.
Буфер Мікшування -- Mixing Buffer
Sets the number of samples used by the audio mixing buffer. Available options are: 512, 1024, 2048, 4096, 8192, 16384, and 32768.
Частота Вибірки -- Sample Rate
Sets the audio sampling rate. Available options are: 44.1 Khz, 48 Khz, 96 Khz and 192 Khz.
Формат Вибірки -- Sample Format
Визначає формат вибірки звуку. Доступні опції: 32 bit float, 8 bit Unsigned, 16 Bits Signed, 24 Bits Signed, 32 Bits Signed, 32 Bits Float та 64 Bits Float.

Екранозапис -- Screencast¶

These settings are used to control the frame rate for recording a Screencast.

FPS
The frame rate for screencast playback.
Часолік Очікування -- Wait Timer
Час у мілісекундах між кожним записуваним кадром для екранозапису.

Пристрій Обчислення -- Compute Device¶

Опція Compute Device дозволяє користувачу змінювати пристрій обчислення для рушія-рендерера Cycles, що буде використовуватися для рендерингу зображень. Рушій Cycles може використовувати головний процесор CPU або процесори певних відеокарт GPU для рендерингу зображень, детальніше дивіться тут -- GPU Rendering.

Нема -- None
When set to None or when the only option is None: the CPU will be used as the computing device for the Cycles Renderer.
CUDA
Якщо ваша система має установлену сумісну відеокарту Nvidia CUDA, то ви у змозі використати технологію CUDA для рендерингу рушієм Cycles.
OpenCL
Якщо ваша система має установлений пристрій з підтримкою технології AMD OpenCL, то вона буде доступна також на вибір для рендерингу з Cycles.

Обчислення OpenSubdiv -- OpenSubdiv Compute¶

Ці опції задають пристрій обчислення, використовуваний технологією OpenSubdiv для модифікатора «Підподіл Поверхні» -- Subdivision Surface Modifier.

Нема -- None
Вимкнення усіх пристроїв обчислення Opensubdiv -- None -- гарантує використання успадкованого традиційного методу підподілення. Використовуйте цю опцію, коли OpenSubdiv призводить до будь-яких вад або регресій.
CPU
Одно потокова імплементація центрального процесора. Це, головним чином, корисно у випадках, коли обчислення GPU є можливим та потоковий CPU спричиняє артефакти (навряд чи це станеться, але все ще можливе).
OpenMP
Багато потокова імплементація центрального процесора. Використовуйте це для максимальної продуктивності у випадках, коли GPU обчислення не доступне.
GLSL Відклик Трансформацій -- GLSL Transform Feedback
Використовує GPU для здійснення обчислень, має мінімальні вимоги до відеокарти та її драйвера.
GLSL Обчислення -- GLSL Compute
Використовує GPU для здійснення обчислень, вважається більш ефективною, ніж GLSL Transform Feedback, однак має вищі вимоги до відеокарти та її драйвера.

OpenGL¶

Відсікання Альфа -- Clip Alpha
Відсікає альфа нижче цього порогу у 3D Огляді -- 3D View. Зауважте, що стандартно задано низьке значення для запобігання проблемам на деяких GPU.
MIP-карти -- Mipmaps
Scale textures for 3D View using mip-map filtering. This increases display quality, but uses more memory.
Генерування MIP-карти GPU -- GPU Mipmap Generation
Generate mip-maps on the GPU instead of the CPU.
16 -бітні Дійсночислові Текстури -- 16 Bit Float Textures
Enables the use of 16 Bit per component Texture Images (floating point images).
Виділення -- Selection

Метод, що використовується для виділення.

Автоматично -- Automatic
Автоматично вибирає найкращу уставу залежно від ваших ОС, GPU та драйверів.
OpenGL Виділ -- OpenGL Select
Успадкований традиційний метод OpenGL виділення для старішого устаткування.
OpenGL Запити на Загороду -- OpenGL Occlusion Queries
Більш оптимізований метод OpenGL виділення. Вживайте цей метод, якщо ви використовуєте пристрій обчислення OpenSubdiv Compute .
OpenGL Вказування Глибини -- OpenGL Depth Picking

This option uses an alternative method of picking which uses depth information to select the front-most elements. It is only used for selecting with the cursor (not border select, lasso, circle select, etc.).

Performance varies depending on your OpenGL hardware and drivers.

Анізотропна Фільтрація -- Anisotropic Filtering
Встановлює рівень анізотропної фільтрації. Це підвищує якість того, як текстури рисуються, за рахунок продуктивності. Доступні опції: «Вимк» -- Off (Без Фільтрації), 2x, 4x, 8x та 16x.

Метод Прорису Вікна -- Window Draw Method¶

Метод Прорису Вікна -- Window Draw Method

Визначає Метод Прорису Вікна, що використовується для показу вікна(он) Blender'а.

Автоматично -- Automatic
Автоматично установлюється на основі відеокарти та її драйвера.
Потрійний Буфер -- Triple Buffer
Використовує третій буфер для мінімальних перерисувань за рахунок пам'яті. Якщо ви маєте відповідний GPU, то це найкращий та найшвидший спосіб перерисування.
Перекриття -- Overlap
Перерисовує усі перекривні регіони. Мінімальне використання пам'яті, але більше перерисувань. Рекомендується для деяких комбінацій відеокарт та їх драйверів.
Переверт Перекриття -- Overlap Flip
Перерисовує усі перекривні регіони. Мінімальне використання пам'яті, але більше перерисувань (для відеокарт, що здійснюють перевертання). Рекомендується для деяких комбінацій відеокарт та їх драйверів.
Повністю -- Full
Здійснює повне перерисування кожен раз. Використовується тільки орієнтира або коли все інше не працює. Корисно для певних відеокарт з поганим або взагалі без прискорення OpenGL.
Багато-Відбір -- Multi-Sampling

Це вмикає FSAA для згладженого прорису за рахунок деякої величини продуктивності.

Ghi chú

Це відомо спричиняє проблеми виділення на деяких конфігураціях, дивіться тут -- Хибне виділення.

Перекриття Регіону -- Region Overlap

Цей стяг дозволяє Blender'у прорисовувати регіони, перекриваючи 3D Огляд. Це означає, що регіони «Полиця Інструментів» -- Tool Shelf та «Властивості» -- Properties будуть рисуватися, перекриваючи редактор 3D Огляду.

Якщо ви маєте відповідну відеокарту та драйвери з підтримкою Triple Buffer, увімкніть цей стяг та виберіть метод Triple Buffer, який дозволить також прорисовувати регіони з використанням Альфа, показуючи вміст 3D Огляду крізь ці регіони.

Опції Прорису Тексту -- Text Draw Options
Enable interface text anti-aliasing. When disabled, texts are drawn using text straight render (filling only absolute pixels).

Текстури -- Textures¶

Ліміт Розміру -- Limit Size
Лімітує максимальну роздільність картинок, що використовуються у текстурованому показі для заощадження пам'яті. Опції ліміту визначаються у кількості пікселів, піднесеній до квадрата (наприклад: опція 256 означає текстуру 256×256 пікселів). Це корисно для розробників ігор, де ліміт текстури відповідає блокам підкачки текстур у цільову пам'ять відеокарти. Доступні опції: «Вимк» -- Off (Без ліміту), 128, 256, 512, 1024, 2048, 4096 та 8192.
Час Очікування -- Time Out
Час після останнього доступу до GL текстури у секундах, після якого вона вивільняється. Задання в 0 збереже текстури призначеними. Мінімум: 0, Максимум: 3600.
Частота Збирання -- Collection Rate
Кількість секунд між кожним виконанням збирача сміття GL текстури. Мінімум: 0, Максимум: 3600.
Метод Прорису Зображень -- Images Draw Method

Метод для прорису зображень з такими підтримуваними опціями:

2D Текстура -- 2D Texture
Використовує CPU для показу трансформувань та прорисовує зображення як 2D текстури.
GLSL
Найшвидший метод використання GLSL для показу трансформувань та прорисовує зображення як 2D текстури.
Рисування Пікселів -- Draw Pixels
Використовує CPU для показу трансформувань та прорисовує зображення як 2D текстури.

Відеорядник/Редактор Кліпів -- Sequencer/Clip Editor¶

Ліміт Кешу Пам'яті -- Memory Cache Limit
Upper limit of the Sequencer's memory cache (megabytes). For optimum Clip editor and Sequencer performance, high values are recommended.

Суцільні Освітлювачі OpenGL -- Solid OpenGL Lights¶

Суцільні Освітлювачі OpenGL -- Solid OpenGL Lights використовуються для освітлення 3D Огляду здебільшого у режимі відтінення «Суцільно» -- Solid. Освітлення є постійним та базованим на позиції «світу». Існують три такі віртуальні джерела освітлення, також звані як допоміжні лампи OpenGL -- OpenGL auxiliary lamps, що використовуються для освітлення сцен у 3D Огляді та не показуються на рендерах.

Іконки з лампою дозволяють користувачу вмикати або вимикати лампи OpenGL. Принаймні одна з трьох допоміжних ламп OpenGL повинна залишатися увімкненою для 3D Огляду. Ці лампи є рівнозначні й відрізняються своєю позицією та кольорами. Ви можете керувати напрямком цих ламп, а також дифузним чи розсіяним та бліковим чи відсвітним кольорами.

Вживання -- Use
Перемикає конкретну лампу.
Розсіяння -- Diffuse
Це постійний колір лампи.
Блікування -- Specular
Це колір відсвітів від лампи.
Напрям -- Direction
Клацання LMB на сфері та перетяг мишкою дає обертання сфери та дозволяє вам змінювати напрямок лампи. Напрямок лампи показується відтіненням поверхні сфери.

Тип Відбірника Кольору -- Color Picker Type¶

Дає змогу вибрати бажаний колірний простір -- color space для відбору кольору. Він буде показуватися при клацанні вами LMB на будь-якій кнопці вибірника кольору.

Дивіться про різні типи відбірника кольору тут -- Color picker.

Власний Діапазон Малювання Вагомостей -- Custom Weight Paint Range¶

Виваження сітьового покриву використовується для керування тим, наскільки кістка буде деформувати сіть покриву персонажа. Для візуалізації та малювання цих вагомостей Blender використовує колірний градієнт (від синього до зеленого та від жовтого до червоного). Вмикання цього стяга дозволить використовувати альтернативну розкладку, створивши її тут на основі порожнього градієнта. Ви можете створити власну розкладку, використовуючи загальні опції колірного градієнта. Детальніше дивіться тут -- Color ramps.

Шрифти -- Fonts¶

Шрифт Інтерфейсу -- Interface Font¶

Шрифт Інтерфейсу -- Interface Font
Дає змогу замінити стандартний шрифт інтерфейсу користувача.
Моноширинний Шрифт -- Mono-space Font
Дозволяє замінити стандартний моноширинний шрифт для інтерфейсу.

Інтернаціоналізація¶

Blender supports a wide range of languages, enabling this checkbox will enable Blender to support International Fonts. International fonts can be loaded for the User Interface and used instead of Blender default bundled font.

Це також вмикає опції для перекладу Інтерфейсу Користувача на ряд мов, а також перекладу Підказок для інструментів у Blender'і, що з'являються при наведенні вказівника миші над ними.

Blender supports I18N for internationalization. For more Information on how to load International fonts, see: Editing Texts page.
