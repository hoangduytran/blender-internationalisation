Зроблення внеску¶

На цій сторінці код французької (fr) використовується для прикладів, однак, він може бути замінений іншими кодами мов -- languages codes. Тому, переконайтеся, що змінили суфікси /fr у цьому путівнику на код мови, на яку ви перекладаєте!

Для перегляду, які мови поточно доступні, можна подивитися у репозиторії: https://developer.blender.org/diffusion/BMT/browse/trunk/blender_docs/locale

Ghi chú

Перш за все, тут вважається, що ви вже вибудували цей довідник локально. Якщо ви це ще не зробили, то поверніть у секцію Почин.

Установлення¶

Мовні файли¶

З теки, що містить вашу відладку цього довідника, запустіть:

svn checkout https://svn.blender.org/svnroot/bf-manual-translations/trunk/blender_docs/locale/fr locale/fr

Це створить підтеку locale/fr.

Ви повинні отримати таку розставу тек:

blender_docs
   |- locale/
   |  |- fr/
   |  |  |- LC_MESSAGES/
   |- manual/

Ghi chú

При запуску підверсії з командного рядка (для оновлення або фіксування) ви повинні змінити спершу у ньому шлях до тек на locale/fr.

Інакше ви отримаєте повідомлення про помилку: 'locale' is not under version control

Редактор PO Editor¶

Для редагування файлів PO ви повинні інсталювати редактор PO editor. Ми рекомендуємо використовувати редактор Poedit , однак, ви можете використати будь-який редактор цього формату (PO) файлів.

Ghi chú

Для користувачів Linux ви маєте перевірити, щоб завантажити відповідну до виду операційної системи версію Poedit.

Вибудування з Перекладами¶

Ghi chú

Це факультативно, переклади автоматично вибудовуються онлайн, наприклад: https://docs.blender.org/manual/fr/dev/

Тепер ви можете вибудувати цей довідник із застосованим до нього перекладом:

make -e SPHINXOPTS="-D language='fr'"

Якщо ви на MS-Windows і не маєте make, запустіть:

sphinx-build -b html -D language=fr ./manual ./build/html

Тепер ви будете мати побудову довідника із застосованим перекладом.

Редагування Файлів Перекладу¶

Тепер ви можете редагувати файли перекладу РО, наприклад:

Початковий файл RST
manual/getting_started/about_blender/introduction.rst
Генерований файл РО
locale/fr/LC_MESSAGES/getting_started/about_blender/introduction.po

Модифіковані файли .po можуть бути відредаговані та зафіксовані назад у svn.

Підтримування¶

Відстеження Нечітких «Fuzzy» Рядків¶

Коли цей довідник оновлюється, ці переклади, які не актуальні, будуть позначені, як нечіткі (fuzzy). Для відстеження цього ви можете використати інструмент, який ми створили для цього.

Ви можете для цього запустити:

make report_po_progress

Це дасть тільки швидке узагальнення, однак, ви можете отримати більше інформації, запустивши:

python tools/report_translation_progress.py locale/fr/

Ви повинні отримати список усіх цих файлів з інформацією про кількість порожніх та нечітких рядків. Детальніше дивіться:

python tools/report_translation_progress.py --help

Xem thêm

Вказівки на цій сторінці базуються на Sphinx Intl documentation

Оновлення PO файлів¶

Після змін початкового довідника, шаблони потребують оновлення. Зауважте, це не обов'язково робити, оскільки адміністратор зазвичай оновлює файли для всіх мов за раз. Це дозволяє бути усім мовам однакової версії для довідника. Однак, якщо ви потребуєте оновити ці файли самі, то можете зробити наступне:

make update_po

Оновлені шаблони можуть бути далі зафіксовані у svn.
