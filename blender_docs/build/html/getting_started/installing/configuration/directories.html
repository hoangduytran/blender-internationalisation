Конфігурування каталогів¶

Існують три різних каталоги, які може використовувати Blender, їх точне розміщення залежить від операційної системи.

LOCAL
Location of configuration and run-time data (for self-contained bundle).
USER
Розташування файлів конфігурації (зазвичай у домашньому каталогу користувача).
SYSTEM
Location of run-time data for system wide installation (may be read-only).

Для встановлення на систему необхідні обидва каталоги SYSTEM і USER.

For locally extracted Blender distributions, the user configuration and run-time data are kept in the same subdirectory, allowing multiple Blender versions to run without conflict, ignoring the USER and SYSTEM files.

Шляхи залежні від платформи¶

Нижче наведено стандартні локації для кожної з операційних систем:

Ghi chú

Шлях ./|BLENDER_VERSION|/ є відносним для виконуваного файлу Blender'а та використовується для автономних пакетів, що розповсюджуються через офіційні будови з blender.org.

MS-Windows¶

LOCAL
.\2.79\
USER
%USERPROFILE%\AppData\Roaming\Blender Foundation\Blender\2.79\
SYSTEM
%USERPROFILE%\AppData\Roaming\Blender Foundation\Blender\2.79\

macOS¶

LOCAL
./2.79/
USER
/Users/$USER/Library/Application Support/Blender/2.79/
SYSTEM
/Library/Application Support/Blender/2.79/

Ghi chú

macOS зберігає бінарник Blender у ./blender.app/Contents/MacOS/blender, а тому локальний шлях до даних і конфігурації є:

./blender.app/Contents/MacOS/2.79/

Linux¶

LOCAL
./2.79/
USER
$HOME/.config/blender/2.79/
SYSTEM
/usr/share/blender/2.79/

Ghi chú

The USER path will use $XDG_CONFIG_HOME if it is set:

$XDG_CONFIG_HOME/blender/2.79/

Розстава шляхів¶

Це розстава шляхів, що використовується у межах каталогів, описаних вище.

Де ./config/startup.blend може бути ~/.blender/|BLENDER_VERSION|/config/startup.blend , наприклад.

./autosave/ ...

Розташування автозбережених blend-файлів. (лише Windows, каталог «temp» використовується для інших систем).

Порядок пошуку: LOCAL, USER.

./config/ ...

Інформація про стандарти і сесії.

Порядок пошуку: LOCAL, USER.

./config/startup.blend
Стандартний файл для завантаження при запуску.
./config/userpref.blend
Стандартні уподобання для завантаження при запуску.
./config/bookmarks.txt
Закладки Оглядача Файлів -- File Browser.
./config/recent-files.txt
Список для меню останньо використаних файлів.
./datafiles/ ...

Файли виконання.

Порядок пошуку: LOCAL, USER, SYSTEM.

./datafiles/locale/{language}/
Статично передкомпільовані мовні файли для перекладу інтерфейсу користувача.
./scripts/ ...

Скрипти на Python для інтерфейсу користувача та засобів.

Порядок пошуку: LOCAL, USER, SYSTEM.

./scripts/addons/*.py
Додатки на Python, які можуть вмикатися в Уподобаннях Користувача -- User Preferences, включаючи підтримку імпорту/експорту форматів, інтеграцію рушіїв зображування, рендерингу та багато зручних утиліт.
./scripts/addons/modules/*.py
Модулі для використовуваних додатків (додається у системний шлях для Python).
./scripts/addons_contrib/*.py
Інший каталог додатків, який використовується для підтримуваних спільнотою додатків (повинен бути створений вручну).
./scripts/addons_contrib/modules/*.py
Модулі для додатків з addons_contrib (додається у системний шлях для Python).
./scripts/modules/*.py
Модулі для Python, що містять наше ядро API та функції утиліт для інших скриптів для імпорту (додається у системний шлях sys.path для Python).
./scripts/startup/*.py
Скрипти, які автоматично імпортуються при запуску.
./scripts/presets/{preset}/*.py
Presets used for storing user-defined settings for cloth, render formats, etc.
./scripts/templates_py/*.py
Приклади скриптів, які доступні для відкриття через «Редактор Тексту > Шаблони > Python» -- Text Editor ‣ Templates ‣ Python.
./scripts/templates_osl/*.py
Приклади відтінювачів на мові OSL, які доступні для відкриття через «Редактор Тексту > Шаблони > Відкрита Мова Відтінення» -- Text Editor ‣ Templates ‣ Open Shading Language.
./python/ ...

Запакований дистрибутив Python.

Порядок пошуку: LOCAL, SYSTEM.

Тимчасовий каталог¶

The temporary directory is used to store various files at run-time (including render layers, physics cache, copy-paste buffer and crash logs).

Тимчасовий каталог вибирається, базуючись на наступному пріоритеті:

  • Уподобання Користувача -- User Preference (дивіться Шляхи Файлів -- File Paths).
  • Змінні середовища -- Environment variables (TEMP на MS-Windows, TMP і TMP_DIR на інших платформах).
  • Каталог /tmp/.
