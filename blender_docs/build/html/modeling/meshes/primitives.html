Примітиви¶

Довідка

Mode:Object Mode and Edit Mode
Panel:Tool Shelf ‣ Create ‣ Add Primitive/Mesh
Menu:Add ‣ Mesh
Hotkey:Shift-A

Поширеним типом об'єкта, що використовується у 3D-сцені, є полігональна сітка. Blender постачається з рядом "примітивних" сітчастих фігур, з яких можна починати моделювати. Ви також можете додавати примітиви в Режимі Редагування (Edit Mode) на місці 3D-курсору.

Стандартні примітиви Blender'а.

Ghi chú

Planar Primitives

Ви можете зробити планарну полігональну сітку тривимірною, перемістивши одну або декілька вершин з її площини (застосовується до Площини (Plane), Кола (Circle) та Сітки (Grid)). Просте коло насправді часто використовуються як відправна точка для створення навіть найскладніших полігональних сіток.

Gợi ý

Для того, щоб полегшити моделювання, найкраще буде уявити, який тип примітива ліпше підходить для вашої моделі. Якщо ви моделюєте кубоподібну форму, найкраще починати з примітива куба, тощо.

Загальні параметри¶

Ці параметри можна вказати у панелі Оператора (Operator panel) на Полиці Інструментів (Tool Shelf), який з'являється після створення об'єкта. Параметри, що з'являються у більш ніж одному примітиві:

Згенерувати UV-координати (Generate UVs)
Generates a default UV unwrapping of new geometry. This will be defined in the first UV layer (which will get added if needed).
Радіус/Розмір (Radius/Size), Повернути до виду (Align to View), Розташування (Location), Обертання (Rotation)
Див Загальні параметри об'єкта (Common Object Options).

Площина (Plane)¶

The standard plane is a single quad face, which is composed of four vertices, four edges, and one face. It is like a piece of paper lying on a table; it is not a three-dimensional object because it is flat and has no thickness. Objects that can be created with planes include floors, tabletops, or mirrors.

Куб (Cube)¶

Стандартний куб містить 8 вершин, дванадцять ребер та шість граней і є тривимірним об'єктом. Об'єкти, які можна створити із куба включають гральні кості, коробки або ящики.

Коло (Circle)¶

Вершини (Vertices)
Число вершин, що визначають коло або полігон.
Тип заповнення (Fill Type)

Вказує, як коло буде заповнено.

Triangle Fan
Заповнити трикутними гранями, що поділяються вершину посередині.
N-кутник (N-gon)
Заповнити єдиним n-кутником (n-gon).
Нічого (Nothing)
Не заповнювати. Створює лише зовнішнє кільце вершин.

UV Сфера (UV Sphere)¶

Стандартна UV сфера зроблена з чотирикутних граней та віялом трикутників у верхній та нижній частині. Може використовуватися для текстурування.

Сегменти (Segments)
Число вертикальних сегментів. Подібно меридіанам Землі, від полюса до полюса.
Кільця (Rings)

Число горизонтальних сегментів. Подібно до земних паралелей.

Ghi chú

Кільця - це петлі граней (face loops), а не петлі ребер (edge loops), яких було б на один менше.

Ікосфера (Icosphere)¶

An icosphere is a polyhedral sphere made up of triangles. Icospheres are normally used to achieve a more isotropical layout of vertices than a UV sphere, in other words, they are uniform in every direction.

Підрозділи (Subdivisions)
How many recursions are used to define the sphere. At level 1 the Icosphere is an icosahedron, a solid with 20 equilateral triangular faces. Each increase in the number of subdivisions splits each triangular face into four triangles.

Ghi chú

Subdividing an icosphere raises the vertex count very quickly even with few iterations (10 times creates 5,242,880 triangles), Adding such a dense mesh is a sure way to cause the program to crash.

Циліндр (Cylinder)¶

Об'єкти, які можна створити з циліндра, включають ручки та стержні.

Вершини (Vertices)
Число вертикальних ребер між колами використовується для визначення циліндра або призми.
Глибина (Depth)
Встановлює початкову висоту циліндра.
Тип заповнення кришки (Cap Fill Type)
Подібно до кола (див. вище). Коли вибрано нічого (none), створеним об'єктом буде трубка (tube). Об'єкти, що можна створити з трубки, включають труби або склянки (основною відмінністю між циліндром і трубкою є те, що перший має закриті кінці).

Конус (Cone)¶

Об'єкти, які можна створити з циліндра, включають шипи та загострені капелюхи.

Вершини (Vertices)
Число вертикальних ребер між колами або верхівкою, що використовуються для визначення конуса або піраміди.
Радіус 1 (Radius 1)
Встановлює радіус кругової основи конуса.
Радіус 2 (Radius 2)
Sets the radius of the tip of the cone. which will create a frustum (a pyramid or cone with the top cut off). A value of 0 will produce a standard cone shape.
Глибина (Depth)
Встановлює початкову висоту конуса.
Тип заповнення основи (Base Fill Type)
Подібно до кола (див. вище).

Торус (Torus)¶

A doughnut-shaped primitive created by rotating a circle around an axis. The overall dimensions can be defined by two methods.

Пресети оператора (Operator Presets)
Попередні налаштування торуса для перевикористання. Ці налаштування зберігаються як скрипти у відповідному каталозі пресетів.
Основні сегменти (Major Segments)
Number of segments for the main ring of the torus. If you think of a torus as a "spin" operation around an axis, this is how many steps are in the spin.
Малі сегменти (Minor segments)
Кількість сегментів малого кільця торуса. Це число вершин кожного кругового сегмента.

Розміри Торуса (Torus Dimensions)¶

Режим додавання (Add Mode)

Змінює спосіб визначення торуса.

Більший/Менший (Major/Minor), Зовнішній/Внутрішній (Exterior/Interior)

Більший Радіус (Major Radius)
Радіус від початку (origin) до центру поперечного перетину.
Менший Радіус (Minor Radius)
Radius of the torus' cross section.
Зовнішній Радіус (Exterior Radius)
Якщо дивитися вздовж головної осі, це радіус від центра до зовнішнього ребра.
Внутрішній Радіус (Interior Radius)
Якщо дивитися вздовж головної осі, це радіус отвору в центрі.

Сітка (Grid)¶

Регулярна квадратична сітка, яка є поділеною площиною. Приклади об'єктів, які можуть бути створені з сітки, включають ландшафти та органічні поверхні.

X Підрозділи (X Subdivisions)
Кількість проходів по осі X.
Y Підрозділи (Y Subdivisions)
Кількість проходів по осі Y.

Мавпа (Monkey)¶

This is a gift from old NaN to the community and is seen as a programmer's joke or "Easter Egg". It creates a monkey's head once you press the Monkey button. The Monkey's name is "Suzanne" and is Blender's mascot. Suzanne is very useful as a standard test mesh, much like the Utah Teapot or the Stanford Bunny.

Ghi chú

Додатки (Add-ons)

In addition to the basic geometric primitives, Blender has a number of script generated meshes to offer as pre-installed add-ons. These are available when enabled in the User Preferences (select the Category Add Mesh, then check any desired items).
