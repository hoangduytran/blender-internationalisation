Вступ¶

A soft body cloth uncovering a text.

Animation video and blend-file.

A soft body in general, is a simulation of a soft or rigid deformable object. It is useful for everything that tends to bend, deform, in reaction to forces like gravity or wind, or when colliding with other objects.

У Blender'і ця система найкраща для простих тканинних об'єктів та закритих, замкнутих сітей, наприклад, для шкіри або гуми. Існує й інша фізика імітації тканини -- Cloth Simulation, що використовує інший обчислювач та краща власне для тканин.

Імітація м'якого тіла робиться через застосування сил до вершин або керувальних точок об'єкта. Для неї передбачені зовнішні сили, як гравітація або силові поля, та внутрішні сили, що утримують вершини м'якого тіла разом. Таким чином ви можете імітувати форми, які об'єкт може набувати у реальності, ніби він має об'єм, заповнений чимось, та ніби на нього діють реальні сили.

Soft bodies can interact with other objects through Collision. They can interact with themselves through Self-Collision.

The result of the soft body simulation can be converted to a static object. You can also bake edit the simulation, i.e. edit intermediate results and run the simulation from there.

Типові сценарії для використання М'яких Тіл¶

A wind cone. The cone is a soft body, as the suspension.

Animation -- blend-file.

Soft bodies are well suited for:

  • Еластичні об'єкти з або без зіткнення.
  • Стяги, тканина, що реагують на сили.
  • Певні завдання моделювання, як подушка або скатертина на об'єкті.
  • Blender has another simulation system for clothing (see Clothes). But you can sometimes use soft bodies for certain parts of clothing, like wide sleeves.
  • Волосся (поки ви мінімізуєте зіткнення).
  • Анімування гойдальних мотузок, ланцюгів тощо.

Створення М'якого Тіла¶

Soft body simulation works for all objects that have vertices or control points:

  • Сіті -- Meshes.
  • Криві -- Curves.
  • Поверхні -- Surfaces.
  • Решітки -- Lattices.

To activate the soft body simulation for an object:

  • У Редакторі Властивостей -- Properties editor перейдіть у вкладку «Фізика» -- Physics (це весь шлях направо та виглядає як відскікний м'яч).
  • Активуйте кнопку «М'яке Тіло» -- Soft Body.

З'являться багато опцій. Для довідки про всі ці устави дивіться -- this page.

  • You start a soft body simulation with Alt-A.
  • Призупиняєте ви імітацію за допомогою Spacebar та продовжуєте її за допомогою Alt-A.
  • Зупиняєте ви імітацію за допомогою Esc.

Обчислювач М'якого Тіла -- Soft Body Solver¶

Орієнтир -- Reference

Panel:Physics ‣ Soft Body Solver

Устави на панелі Soft Body Solver визначають точність імітації м'якого тіла.

Розмір Кроку -- Step Size¶

Мін Крок -- Min Step
Minimum simulation steps per frame. Increase this value, if the soft body misses fast-moving collision objects.
Макс Крок -- Max Step
Максимум кроків імітації за кадр. Зазвичай, кількість кроків імітації задається динамічно (за допомогою устави Error Limit), але ви можливо маєте вагому причину для зміни цього.
Авто-Крок -- Auto-Step
Використовує Скорості для автоматичних розмірів кроків.

Ліміт Помилки -- Error Limit¶

Правила загальної якості представленого рішення. Стандартно -- 0.1. Цей найбільш важлива устава, що говорить, наскільки точним повинен бути обчислювач при перевірці зіткнень. Починайте зі значення, що є 1/2 усередненої довжини ребер. Якщо присутні видимі помилки, тремтіння або перебільшені реакції, зменште це значення. Обчислювач відстежує, наскільки «погано» він працює, а Error Limit змушує обчислювач робити деякі «адаптивні зміни розміру кроку».

Помічники -- Helpers¶

Душення -- Choke
Затискає у бік зменшення (скорочує скорість виходу) вершину або ребро, коли вони проникають у сіть об'єкта зіткнення.
Розпливчато -- Fuzzy
Імітація є швидшою, але менш точною.

Діагностика -- Diagnostics¶

Вивести Продуктивність на Консоль -- Print Performance to Console
Виводить на консоль відомості, як працює обчислювач.
Матриця Припущення -- Estimate Matrix
Матриця припущення. Розділяється на COM, ROT, SCALE

Кеш -- Cache¶

Орієнтир -- Reference

Panel:Physics ‣ Soft Body Cache

Soft bodies and other physic simulations use a unified system for caching and baking. See Particle Cache for reference.

Результати імітації автоматично кешуються на диск при відтворенні її анімації, так щоб при наступному його запуску воно відтворювалося швидше на основі зчитування цих результатів з жорсткого диска. Після запечення Bake імітації кеш стає захищеним і вас буде запитано кожен раз, коли ви намагатиметься змінити уставу, що вимагає перерахунку імітації.

Mẹo

Зверніть увагу на устави Start та End

The simulation is only calculated for the frames in between the Start and End frames (Bake panel), even if you do not actually bake the simulation! So if you want a simulation longer than the default setting of 250 frames you have the change the End frame.

Кешування -- Caching

  • При відтворенні анімації для кожної системи фізики записується розрахований кожен кадр на жорсткий диск у діапазоні між Start та End. Ці файли зберігаються у теках з префіксом blendcache біля даного blend-файлу.
  • Кеш очищується автоматично при змінах. Але не для усіх змін, а тому може знадобитися вивільнити його вручну, наприклад, якщо ви змінили силове поле. Зауважте, щоб заповнити цей кеш, відтворення імітації слід починати перед або на кадрі, з якого стартує імітація.
  • If you are not allowed to write to the required subdirectory caching will not take place.
  • Кеш може бути вивільнений для кожної системи фізики за допомогою відповідної кнопки на панелі або скорочення Ctrl-B для усіх виділених об'єктів.
  • Ви можете зіткнутися з проблемами, якщо шлях до вашого blend-файлу є надто довгим, а ваша операційна система має ліміт на підтримувану довжину шляхів.

Запікання -- Baking

  • Система захищена від змін після запікання.
  • The Bake result is cleared also with Ctrl-B for all selected objects or click on Free Bake for the current soft body system.
  • Якщо сіть змінюється, то імітація не перераховується знову.

For render farms, it is best to bake all the physics systems, and then copy the blendcache to the render farm as well.

Interaction in Real-Time¶

To work with a soft body simulation, you will find it handy to use the Timeline editor. You can change between frames and the simulation will always be shown in the actual state. The option Continue Physics in the Playback menu of the Timeline editor lets you interact in real-time with the simulation, e.g. by moving collision objects or shaking a soft body object.

Mẹo

Continue Physics does not work while playing the animation with Alt-A. It will work only if you start the animation with the Play button of the Timeline editor.

You can then select the soft body object while running the simulation and Apply the modifier in the Modifiers tab of the Properties editor. This makes the deformation permanent.

Поради¶

  • Soft bodies work especially well if the objects have an even vertex distribution. You need enough vertices for good collisions. You change the deformation (the stiffness) if you add more vertices in a certain region (see the animation of Fig. A wind cone. The cone is a soft body, as the suspension.).
  • Розрахунок зіткнень може тривати довго. Якщо щось не видиме, то чому воно розраховується?
  • Для прискорення розрахунку зіткнення часто корисно стикати м'яке тіло з додатковим, простим, невидимим, дещо більшим об'єктом (дивіться приклад на Ілюстрації A soft body cloth uncovering a text.).
  • Use soft bodies only where it makes sense. If you try to cover a body mesh with a tight piece of cloth and animate solely with soft body, you will have no success. Self-collision of soft body hair may be activated, but that is a path that you have to wander alone. We will deal with Collisions in detail later.
  • Try and use a Lattice or a Curve Guide soft body instead of the object itself. This may be magnitudes faster.
