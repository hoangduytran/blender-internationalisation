Зіткнення -- Collisions¶

У більшості випадків, шматок тканини не просто висить десь у 3D просторі, він зіштовхується з іншими об'єктами в навколишньому середовищі. Для гарантування належної імітації існує кілька елементів, що можуть бути укладені та працювати разом:

  • Для об'єкта тканини Cloth повинно бути зазначено, що він бере участь у зіткненнях.
  • Опціонально (але рекомендується) зазначити для тканини зіштовхування самої з собою.
  • Інші об'єкти повинні бути видимі для об'єкта тканини Cloth через спільні шари сцени.
  • Інші об'єкти повинні бути типу «сіть» -- mesh.
  • Інші об'єкти можуть переміщатися або бути самі деформованими іншими об'єктами (такими як арматура або ключ форми).
  • Інші сітьові об'єкти мають бути визначені як такі, що відхиляють об'єкт тканини.
  • Blend-файл з цією імітацією повинен бути збережений у тому ж каталозі, кути зберігаються результати цієї імітації.
  • Ви, далі, запікаєте -- Bake цю імітацію. Імітатор обчислює форму тканини згідно із заданим діапазоном кадрів.
  • Ви можете, далі, редагувати результати імітації або робити корекції для сіті тканини, у певних кадрах.
  • Ви можете робити корекції середовища або деформувальних об'єктів, а потім пере-запускати імітацію тканини з поточного кадру вперед.

Устави зіткнень¶

Reference

Panel:Physics ‣ Cloth Collision

Панель «Зіткнення Тканини» -- Cloth Collision.

Тепер, ви повинні зазначити для об'єкта тканини Cloth, що ви хочете, щоб він брав участь у зіткненнях. Для цього об'єкта тканини розгорніть панель Cloth Collision, показану тут на ілюстрації:

Якість -- Quality
Загальна устава, що визначає наскільки гарною та доброю буде бажана для вас імітація. Чим вище тут значення, тим більше потрібно часу на обчислення імітації, але це гарантує менше розривів та перетинань у тканині.
Відстань -- Distance
Як тільки інший об'єкт наближається ближче за це значення (в Одиницях Blender -- Blender Units), то імітація почне відштовхувати тканину від нього.
Відштовх -- Repel
Сила відштовхування для застосування, коли тканина близька до зіткнення.
Відстань Відштовху -- Repel Distance
Максимум відстані для застосування сили відштовхування. Повинна бути більшою, ніж мінімальна відстань для зіткнення.
Тертя -- Friction
Коефіцієнт того, наскільки слизькою буде тканина, коли стикається з іншим сітьовим об'єктом. Наприклад, шовк має менше значення цього коефіцієнта тертя, ніж бавовна.

Само-зіткнення¶

Реальна тканина не може проникати сама в себе, а тому зазвичай вам необхідно, що тканина мала властивість само-зіткнення.

Вмикання само-зіткнень
Вмикання стягу «Само Зіткнення» -- Self Collision визначає, що об'єкт тканини не повинен пронизувати сам себе. Це збільшує час обчислення імітації, але забезпечує більш реалістичні результати. Для прапора, який переглядається з відстані, це не сильно потребує вмикання цієї опції, але крупний план капюшона або блузки на персонажів вимагає, щоб ця опція була увімкнена.
Якість -- Quality
For higher self-collision quality just increase the Quality and more self-collision layers can be solved. Just keep in mind that you need to have at least the same Collision Quality value as the Quality value.
Відстань -- Distance
If you encounter problems, you could also change the Min Distance value for the self-collisions. The best value is 0.75; for fast things you better take 1.0. The value 0.5 is quite risky (most likely many penetrations) but also gives some speed-up.

Blend-file регресії: Cloth selfcollisions.

Сумісні шари¶

Suppose you have two objects: a pair of pants on layers 2 and 3, and your character mesh on layers 1 and 2. You have enabled the pants as cloth as described above. You must now make the character "visible" to the Cloth object, so that as your character bends its leg, it will push the cloth. This principle is the same for all simulations; simulations only interact with objects on a shared layer. In this example, both objects share layer 2.

Для перегляду/зміни шарів об'єкта клацніть RMB у Режимі Об'єкта -- Object Mode у 3D Огляді -- 3D View на об'єкті для його виділення. Натисніть M для виклику спливного меню «Перемістити на Шар» -- Move to Layer, яке покаже усі шари, на яких поточно знаходиться об'єкт. Для поставлення об'єкта на одиничний шар клацніть LMB на кнопці потрібного шару. Для поставлення об'єкта на кількох одночасно шарах клацайте Shift-LMB на кнопках потрібних шарів. Для вилучення знаходження об'єкта на певному шарі просто клацніть Shift-LMB на його кнопці для зняття об'єкта з цього шару.

Об'єкти зіткнень повинні бути сітями¶

Якщо ви хочете зіткнути об'єкт тканини з не сітьовим об'єктом, яке поверхня НОРБС -- NURBS surface або об'єкт тексту, то ви повинні конвертувати останній у сітьовий -- mesh об'єкт. Для цього виділіть цей об'єкт у Режимі Об'єкта -- Object Mode та у 3D Огляді -- 3D View через заголовок виберіть «Об'єкт > Конвертувати у» -- Object ‣ Convert to чи натисніть Alt-C та у підменю або спливному меню виберіть «Сіть із Крива/Мета/Сплайн/Текст» -- Mesh from Curve/Meta/Surf/Text.

Зіткнення «Об'єкт - Тканина»¶

Устави зіткнень відхильного об'єкта.

The cloth object needs to be deflected by some other object. To deflect a cloth, the object must be enabled as an object that collides with the cloth object. To enable Cloth -- Object collisions, you have to enable deflections on the collision object (not on the cloth object).

Для цього виділяється саме цей інший об'єкт та у Редакторі Властивостей -- Properties editor, у вкладці «Фізика» -- Physics tab, вмикається імітація «Зіткнення» -- Collision. Також важливо відзначити, що ця панель використовується для зіткнень з усіма імітаціями, які може мати цей об'єкт на спільних шарах (з частинками, м'якими тілами та тканиною).

Cảnh báo

There are three different Collision panels, all found in the Physics tab. The first (by default), a tab beside the Fields panel, is the one needed here. The second panel, a tab in the Soft Body group, concern soft bodies (and so has nothing to do with cloth). And we have already seen the last one, by default a tab beside the Cloth panel.

Вплив стеку модифікаторів сітьового об'єкта¶

Стек модифікаторів, враховуваний зіткненнями.

Форма об'єкта деформує тканину, а тому імітація тканини повинна знати «справжню» форму іншого сітьового об'єкта у певному кадрі. Ця справжня форма є базовою формою, що визначається модифікацією ключів форм або арматур. Звідси, модифікатор «Зіткнення» -- Collision повинен знаходитися після всіх цих модифікаторів у стеку (у самому низу). Ілюстрація тут показує панель «Модифікатори» -- Modifiers для сітьового об'єкта Персонажа (а не для об'єкта тканини).

Кеш Тканини -- Cloth Cache¶

Устави кешу для тканини такі ж, як і для інших динамічних систем. Дивіться детальніше тут -- Particle Cache.

Запікання зіткнень¶

After baking.

Після того, як ви уклали відхильну сіть для діапазону кадрів, потрібного для імітації (включаючи анімування сіті через арматури), ви можете тепер визначити для імітації тканини обчислення (та уникання) зіткнень. Виділіть об'єкт тканини, на панелі «Кеш Тканини» -- Cloth Cache у вкладці «Фізика» -- Physics задайте устави «Старт» -- Start та «Кінець» -- End відповідно до діапазону імітації та клацніть на кнопці «Запікання» -- Bake.

Ви далі не зможете змінити ці устави Start чи End без зчищення запікання імітації. Після завершення запікання ви помітите, що стали доступні опції для звільнення, редагування або повторного запікання імітації.

Є кілька речей, які ви мабуть, помітите відразу ж. Перше, це те, що запікання відбувається відчутно повільніше, ніж до цього, і це можливо дещо дасть не досить гарний результат.

Редагування кешованої імітації¶

Кеш містить форму сіті у кожному кадрі. Ви можете редагувати кешовану анімацію після її запікання, натиснувши кнопку Bake Editing. Просто перейдіть у бажаний кадр та натисніть Tab для переходу у Режим Правки -- Edit Mode. Тут ви можете рухати ваші вершини, використовуючи усі засоби формування сіті, доступні у Blender'і. Коли ви вийдете з Режиму Правки, то форма сіті буде записана у цьому кадрі анімації. Якщо ви хочете, що Blender відновив імітацію вже з цієї нової форми, клацніть LMB на кнопці Rebake from next Frame та запустіть відтворення анімації. Blender візьме цю форму та відновить імітацію.

Відредагуйте сіть для виправлення незначних розривів та місць, де об'єкт зіткнення протикає тканину.

If you add, delete, or extrude vertices in the mesh, Blender will take the new mesh as the starting shape of the mesh back to the first frame of the animation, replacing the original shape you started with, up to the frame you were on when you edited the mesh. Therefore, if you change the content of a mesh, when you press Tab out of Edit Mode, you should unprotect and clear the cache so that Blender will make a consistent simulation.

Вирішення проблем¶

Якщо ви стикнулися з деякими проблемами при визначенні зіткнень, існує два способи виправлення цього:

  • Найшвидшим рішенням є збільшити значення устави Distance на панелі Cloth Collision. Це буде найшвидший спосіб виправлення відсікання; однак, це буде менш точним та не буде виглядати так само добре. Використання цього методу, зазвичай, робить вигляд тканини, ніби це позиція її спокою на повітрі, та дає дуже її заокруглений вигляд.
  • Другий метод -- це збільшити значення устави Quality (на панелі Cloth). Це дає менші кроки для імітатора та звідси вищу ймовірність отримання швидко рухомих зіткнень. Ви можете також збільшити уставу Quality на панелі Collision для здійснення більше повторів для обчислення зіткнень.
  • Якщо жоден з цих методів не допомагає, то ви можете легко підредагувати кешований/запечений результат у Режимі Правки -- Edit Mode після всього.
  • Для тканини, що розривається деформувальною сіттю -- ніби «велетнем»: збільште для тканини структурну тугість (устава Structure Stiffness, панель Cloth) дуже сильно, біля 1000.

Ghi chú

Модифікатор «Підподіл Поверхні» -- Subdivision Surface

Запікання/кеш робиться для кожного рівня підподілу, а тому, будь ласка, використовуйте однаковий рівень підподілу для рендера та передогляду.
