Зображ, Рендер -- Render¶

Орієнтир -- Reference

Panel:Particle System ‣ Render

Панель «Зображ, Рендер» -- Render керує тим, як виглядатимуть частинки, коли вони будуть відзображені, рендерені.

Індекс Матеріалу -- Material Index
Індекс матеріалу або гніздо (slot) матеріалу об'єкта задає те, який матеріал буде використовуватися для відтінення, забарвлення частинок.
Приріднення -- Parent
Використовує координати іншого об'єкта для визначення народження частинок.
Емітер -- Emitter
При вимкненні, сам емітер не буде рендеритися, зображуватися. При увімкненні кнопки Emitter рендеритися, зображуватися буде і сіть самого емітера.
Предки -- Parents
Render also parent particles if child particles are used. Children have a lot of different deformation options, so the straight parents would stand between their curly children. So by default Parents are not rendered if you activate Children. See Children.
Ненароджене -- Unborn
Вмикає рендеринг частинок до того, як вони народяться.
Вмерле -- Died
Вмикає зображування частинок після того, як вони померли. Це дуже корисно, якщо частинки вмирають при зіткненні Die on hit, так що ви можете покривати об'єкти частинками.

Тип¶

Нема -- None¶

При увімкненні кнопки None частинки взагалі не будуть рендеритися, зображуватися. Це корисно, якщо ви використовуєте частинки для дублювання об'єктів.

Ореол -- Halo¶

Частинки типу Halo рендеряться як матеріали типу «ореол» -- Halo Type Materials.

Розрахунок Сліду -- Trail Count
Задає кількість частинок сліду. При значенні більше за 1 з'являються додаткові опції.
Довжина у Кадрах -- Length in Frames
Часування шляху в абсолютних кадрах.
Довжина -- Length
Час кінця рисування шляху.
Випадково -- Random
Задає випадкову варіацію для довжин шляхів.

Лінія -- Line¶

Цей тип візуалізації Line створює (більш або менш тонкі) полігональні лінії за допомогою зображувача, рендерера пасма у напрямку скоростей частинок. Товщина ліній задається параметром Start чи Root для типу відтінювача Strands (вкладка Material, панель Links and Pipeline чи Strand).

Хвіст -- Tail
Задає довжину хвоста частинки.
Голова -- Head
Задає довжину голови частинки.
Швидкість -- Speed
Вмикає множення довжини лінії на швидкість частинок. Чим швидше, тим довше лінія.
Розрахунок Сліду -- Trail Count
Дивіться опис для типу Halo.

Шлях -- Path¶

Панель устав для типу візуалізації Path.

Візуалізація типу Path працює для типу емітера «Волосся» -- Hair та для фізики «Ключована» -- Keyed типу «Емітер».

Рендер, зображ Пасма -- Strand Render
[Keypointstrands] Використовує примітив пасма для рендерингу, зображування. Дуже швидкий та ефективний зображувач, рендерер.
Адаптивний Зображ, Рендер -- Adaptive Render

Намагається вилучити небажану геометрію зі шляхів перед рендерингом частинкових пасм, щоб зробити зображування, рендеринг швидшим та легшим для пам'яті.

Кут -- Angle
Скільки градусів шлях має відносно кривої для продукування іншого сегменту зображу (прямолінійні частини шляхів потребують менше сегментів).
Піксель -- Pixel
Скільки пікселів шлях має мати для покриття для продукування іншого сегменту рендеру (дуже коротка волосинка або довга волосинка, переглядувані з далекої відстані, потребують менше частин). (тільки для Адаптивного Рендера).
Б-Сплайн -- B-Spline
Інтерполює волосинки, використовуючи Б-Сплайни. Це може бути прийнятним варіантом, якщо ви хочете використовувати низькі значення Render. Ви втрачаєте трохи контролю, але отримуєте плавніші шляхи.
Кроки -- Steps
Задає кількість підподілів рендерених шляхів (це значення підноситься до ступеня 2). Ви повинні задавати це значення обережно, оскільки збільшення вами значення рендера лише удвічі вимагатиме у чотири рази більше пам'яті для рендерингу. Також, рендеринг є швидшим, якщо ви вживаєте низькі значення рендеру (інколи різко). Але наскільки низько ви можете піти з цим значенням, залежить від хвилястості волосся (значення підноситься до ступеня 2). Це означає, що 0 кроків дає 1 підподіл, 1 дає 2 підподіли 2 --> 4, 3 --> 8, 4 --> 16, ... n --> n2.

Часування -- Timing¶

Абсолютний Час Шляху -- Absolute Path Time
Часування шляху в абсолютних кадрах.
Старт -- Start
Час старту для рисування шляху.
Кінець -- End
Час кінця рисування шляху.
Випадково -- Random
Задає випадкову варіацію для довжин шляхів.

Please see also the manual page about Strands for an in-depth description.

Об'єкт -- Object¶

Об'єкт для Дублів -- Dupli Object
Цей визначений об'єкт дублюється у місці кожної частинки.
Глобально -- Global
Вживає глобальні координати об'єкта для дублювання.
Обертання -- Rotation
Використовує обертання об'єкта.
Масштаб -- Scale
Вживає масштаб об'єкта.

Група -- Group¶

Група для Дублів -- Dupli Group
Об'єкти, що належать цій групі, дублюються послідовно у місцях частинок.
Група Цілком -- Whole Group
Використовує усю групу за раз, замість кожного з її елементів, ця група буде показуватися повністю у місці кожної частинки.
Обирати Випадково -- Pick Random
Об'єкти у групі вибираються у випадковому порядку і тільки один об'єкт показується у місці частинки. Будь ласка, зауважте, що цей механізм повністю заміню стару систему частинок Blender'а, використовуючи походження та дубль-вершини -- DupliVerts для заміни частинок фактичною геометрією. Цей метод є повністю застарілим та більше не працює.
Вжити Розрахунок -- Use Count
Use objects multiple times in the same groups. Specify the order and number of times to repeat each object with the list view that appears. You can duplicate an object in the list with the Plus button, or remove a duplicate with the Minus button.
Глобально -- Global
Вживає глобальні координати об'єкта для дублювання.
Обертання -- Rotation
Використовує обертання об'єктів.
Масштаб -- Scale
Вживає масштаб об'єктів.

Плакат -- Billboard¶

Устави візуалізації Billboard для частинок.

Плакати -- Billboards -- це вирівняні квадратні площини. Вони стандартно вирівнюються у напрямку камери, але ви можете обрати інший об'єкт, по якому вони будуть вирівнюватися.

If you move a billboard around its target, it always faces the center of its target. The size of a billboard is set with the parameter Size of the particle (in Blender Units). You can use them e.g. for Sprites, or to replace Halo visualization. Everything that can be done with a halo can also be done with a billboard. But billboards are real objects, they are seen by ray tracing, they appear behind transparent objects, they may have an arbitrary form and receive light and shadows. They are a bit more difficult to set up and take more render time and resources.

Texturing billboards (including animated textures with alpha) is done by using UV coordinates that are generated automatically for them so they can take an arbitrary shape. This works well for animations, because the alignment of the billboards can be dynamic. The textures can be animated in several ways:

  • Залежно від часу життя частинок (відносний час).
  • Залежно від часу стартування частинок.
  • Залежно від кадру (абсолютний час).

Ви можете використовувати різні частини зображеннєвої текстури:

  • Залежно від часу життя плаката.
  • Залежно від часу емісії.
  • Залежно від вирівняння або відхилу.

Оскільки, ви вживаєте звичайні матеріали для плаката, то ви маєте усі свободи у змішуванні текстур на свій смак. Матеріал сам анімується в абсолютному часі.

Головне тут для розуміння -- це те, що якщо об'єкт немає жодних карт UV maps, то вам необхідно створити принаймні одну у вкладці Objects Data і вона буде для будь-якого з них працювати. До того ж, текстура має бути задана по UV координатам на панелі Map Input. Якщо ви хочете подивитися приклади деяких з цих можливостей анімування, то дивіться тут -- Billboard Animation Tutorial.

Цікавою альтернативою плакатам є певні випадки пасм, оскільки ви можете анімувати форму цих пасм. Оскільки, цей тип візуалізації має так багато опцій, то його пояснення детальніше нижче.

Вирівняння -- Align

Ви можете лімітувати переміщення за допомогою цих опцій. Як вісь попередньо вирівнюється під час емісії.

X, Y, Z
Along the global X, Y, Z axis respectively.
Огляд -- View
Без попереднього вирівняння, орієнтація нормалей на ціль.
Скорість -- Velocity
Уздовж вектора швидкості частинки.
Блокування -- Lock
Блокує вісь вирівняння, зберігаючи цю орієнтацію, плакат вирівнюється тільки уздовж однієї з осей його цілі.
Об'єкт для Плаката -- Billboard Object
Цільовий об'єкт, до якого повернені плакати. Стандартно, використовується активна камера.
Відхил Кут -- Tilt Angle
Кут обертання площини плакатів. Відхил на 1 обертає на 180 градусів (перевертає плакат зверху донизу).
Випадково -- Random
Випадкова варіація відхилу.
Зсув по X -- Offset X
Зсув плаката горизонтально відносно центра частинки, це не переміщує текстуру.
Зсув по Y -- Offset Y
Зсув плаката вертикально відносно центра частинки.
Канали UV
Плакати -- це просто квадратні полігони, багатобічники. Для їх текстурування різними способами ми повинні мати спосіб для задання того, які текстури ми хочемо для плакатів, та як ми хочемо їх розкласти на ці квадрати. Це може далі бути задано уставами розкладання текстур для установлення бажаних текстур по різних координатах. Ви можете використовувати три різних карти UV та мати три різних набори координат UV, які можуть далі бути застосовані для різних (або одних і тих же) текстур.
UV для Нормалі Плаката -- Billboard Normal UV
Координати є однаковими для кожного плаката, і це просто розміщує зображення прямо на квадраті.
UV для Індексу Часу Плаката (X-Y) -- Billboard Time-Index (X-Y)
Coordinates actually define single points in the texture plane with the X axis as time and Y axis as the particle index. For example using a horizontal blend texture mapped to color from white to black will give particles that start off as white and gradually change to black during their lifetime. On the other hand a vertical blend texture mapped to color from white to black will make the first particle to be white and the last particle to be black with the particles in between a shade of gray.

Анімація текстур UV є дещо хитромудрою. Текстура UV розділяється на ряди та стовпці (N разів по N). Ця текстура повинна бути квадратною. Ви маєте задати UV Split на каналі UV та заповнити ім'я карти UV. Це генеровані координати UV для цього шару.

Розділити UV-ти -- Split UV's
The amount of rows/columns in the texture to be used. Coordinates are a single part of the UV Split grid, which is an n × n grid over the whole texture. What the part is used for each particle and at what time is determined by the Offset and Animate controls. These can be used to make each billboard unique or to use an "animated" texture for them by having each frame of the animation in a grid in a big image.
UV для Розділу Плакату -- Billboard Split UV
Задає ім'я карти UV map для використання з плакатами (ви можете використати різні по одному для кожного каналу UV Channel). Стандартно, це є активна карта UV (перевірте вкладку Object Data у Редакторі Властивостей).
Анімування -- Animate

Меню вибірника, що вказує те, як розділені UV-ти можуть бути анімовані (змінювання від частинки до частинки з часом):

Нема -- None
Жодна анімація не відбувається на самій частинці, плакат використовує одну секцію текстури під час свого часу життя.
Вік -- Age
Секції текстури проходять послідовно у ході часу життів частинок.
Кут -- Angle
Зміна секції базується на куті обертання навколо осі Align, якщо ж вживається View, то ця зміна базується на величині відхилу.
Кадр -- Frame
Секція змінюється відповідно до кадру.
Зсув -- Offset

Визначає, як обирати першу частину (з усіх частин сітки n×n у текстурі, визначених числом UV Split) для усіх частинок.

Нема -- None
Усі частинки стартують з першої частини.
Лінеарно -- Linear
Перша частинка стартуватиме з першої частини, остання частинка -- з останньої частини, частинки між ними -- отримають призначення частин за прямолінійним порядком від першої до останньої частини.
Випадково -- Random
Задає випадкову стартову частину для кожної частинки.
Розрахунок Сліду -- Trail Count
Дивіться опис для типу Halo.
