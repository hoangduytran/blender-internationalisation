Рій -- Boids¶

Орієнтир -- Reference

Panel:Particle System ‣ Physics
Type:Рій -- Boids

Boid Physics settings.

Системи частинок з фізикою «Рій» керуються обмеженим штучним інтелектом, який може програмуватися відповідно до базових правил та поведінки. Вони є ідеальними для імітації натовпів, роїв, стад та зграй різних видів тварин, комах та риб або імітацій хижаки проти здобичі. Вони можуть реагувати на присутність інших об'єктів та на членів своєї ж системи. «Рій» може обробляти тільки певну кількість інформації, а тому послідовність устав поведінки є дуже важливою. У певних ситуаціях обчислюються тільки перші три параметри.

Щоб бачити панель для цього типу, додайте систему частинок Particle System типу Emitter та погляньте на середину вкладки Particle System.

Опції¶

Головне¶

Рої намагаються уникати об'єктів з активованим відхилянням -- Deflection. Вони намагаються досягти об'єктів з додатними Сферичними полями та відлітати від об'єктів з від'ємними Сферичними полями. Об'єкти для взаємодії з ними мають перебувати на одному загальному шарі для ефекту фізики рою. Не є обов'язковим рендеринг, зображування цього спільного шару, а тому ви можете використовувати невидимий вплив.

Рої можуть мати різну фізику залежно від того, знаходяться вони у повітрі чи на землі (на об'єкті із зіткненням).

Дозволити Політ -- Allow Flight

Дозволити Землю -- Allow Land

Подробиці¶

Як згадувалося раніше. дуже подібно як і частинки з фізикою «Ньютонова» -- Newtonian, «Рій» буде реагувати на навколишні відхильники та поля, відповідно до потреб аніматора.

Відхиляння¶

Boids will try to avoid deflector objects according to the Collision rule's weight. It works best for convex surfaces (some work needed for concave surfaces).

Spherical Fields¶

Для фізики «рій» Сферичні поля визначають спосіб, як об'єкти, що мають таке поле, ідентифікуються іншими. Тому, від'ємне Сферичне поле (на об'єкті чи системі частинок) буде переслідувачем, хижаком для усіх інших систем частинок з фізикою «рій», а додатне поле буде ціллю, здобиччю, жертвою для усіх інших систем частинок з фізикою «рій».

Ці ефектори можуть бути хижаками (від'ємна Сила -- Strength), яких рої намагатимуться уникати, або цілями (додатна Сила), яких рої намагатимуться досягти відповідно до вагомостей правил Уникання -- Avoid та Ціль -- Goal. Сила Сферичного ефектора множиться на фактичну відповідну вагомість (наприклад, якщо Сила -- Strength або Ціль -- Goal є нульовими, то зграя не буде відстежувати додатне Сферичне поле).

Інтелект Рою -- Boid Brain¶

Панель «Інтелект Рою» керує тим, як ройові частинки будуть реагувати одна на одну.

Правила¶

Поведінка ройових частинок керується списком правил. Тільки певна кількість інформації у цьому списку може бути обчислена. Якщо доступний обсяг пам'яті перевищено, то інші правила ігноруються.

Ці правила стандартно аналізуються зверху донизу у списку (тобто, з явним пріоритетом), і їх порядок може бути модифіковано за допомогою кнопок зі стрілками справа від списку.

Ціль -- Goal

Шукання цілі (об'єкти зі Сферичними полями та додатною Силою).

Передбачення -- Predict
Вмикає передбачення переміщення цілі.
Уникання -- Avoid

Уникання «хижаків» (об'єкти зі Сферичними полями та від'ємною Силою).

Передбачення -- Predict

Фактор Страху -- Fear Factor
Уникання об'єкта, якщо небезпека від нього є вищою за заданий поріг.
Уникання Зіткнень -- Avoid Collision

Рій -- Boids

Оцінювання Правил -- Rule Evaluation¶

Існує три способи керування тим, як правила оцінюються.

Усереднено -- Average
Усі правила усереднюються.
Випадково -- Random
Для кожної ройової частинки правило вибирається випадково.
Розпливчато -- Fuzzy
Використовує розпливчасту, нечітку логіку для оцінювання правил. Правила оцінюються зверху вниз. Оцінюється тільки перше правило, що має вплив вище за поріг розпливчастості -- Rule Fuzziness. Це значення розглядатися так, наскільки сильно ройова частинка буде намагатися дотримуватися цього правила (значення 1.000 означає, що ройова частинка буде точно його дотримуватися, а значення 0.000, що вона не буде це робити ніколи). Якщо ройова частинка відповідає більше ніж одній умові, що суперечать одна одній, одночасно, то вона намагатиметься дотриматися усіх цих правил відповідно до вагомостей кожного.

Будь ласка, зауважте, що задана ройова частинка, намагатиметься, наскільки це можливо, дотриматися кожного заданого правила, але більш ймовірно, що деякі правила матимуть перевагу над іншими в деяких випадках. Наприклад, щоб уникнути переслідувача, ройова частинка може «забути» про правила зіткнення, громадження та центрування, що означає, що «під час паніки» вона зможе входити у перешкоди, наприклад, навіть, якщо це не задано для неї, у більшості випадків.

Наостанок, алгоритм Зіткнення ще не є ідеальним і дослідження тривають, а тому ви можете очікувати неправильну поведінку у деяких випадках. Над ним триває робота.
