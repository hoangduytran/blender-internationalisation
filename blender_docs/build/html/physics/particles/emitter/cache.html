Кеш -- Cache¶

Орієнтир -- Reference

Panel:Particle System ‣ Cache

Particles Cache settings.

In order to increase real-time response and avoid unnecessary recalculation of particles, the particle data can be cached in memory or stored on a drive.

Emitter systems use a unified system for caching and baking (together with soft body and cloth). The results of the simulation are automatically cached to disk when the animation is played, so that the next time it runs, it can play again quickly by reading in the results from the disk. If you Bake the simulation the cache is protected and you will be asked when you are trying to change a setting that will make a recalculating necessary.

Mẹo

Зверніть увагу на устави Start та End

The simulation is only calculated for the positive frames in between the Start and End frames of the Bake panel, whether you bake or not. So if you want a simulation longer than 250 frames you have to change the End frame!

Кешування -- Caching¶

  • При відтворенні анімації для кожної системи фізики записується розрахований кожен кадр на жорсткий диск у діапазоні між Start та End. Ці файли зберігаються у теках з префіксом blendcache біля даного blend-файлу. Зауважте, щоб заповнити цей кеш, відтворення імітації слід починати перед або на кадрі, з якого стартує імітація.
  • Кеш очищується автоматично при змінах. Але не для усіх змін, а тому може знадобитися вивільнити його вручну, наприклад, якщо ви змінили силове поле.
  • Якщо запис неможливий у вказаний підкаталог, то кешування не відбудеться.
  • Кеш може бути вивільнений для кожної системи фізики за допомогою відповідної кнопки на панелі або скорочення Ctrl-B для усіх виділених об'єктів.
  • Якщо шлях файлу для кешу є довшим, ніж це можливо для вашої операційної системи (більше за 250 символів, наприклад), то можуть відбуватися дивні речі.

Запікання -- Baking¶

  • Система захищена від змін після запікання.
  • Результат Bake очищується також за допомогою Ctrl-B для усіх виділених об'єктів або клацком на кнопці Free Bake для одиничної системи частинок.
  • Якщо сіть змінюється, то імітація не перераховується знову.
  • Sorry: no bake editing for particles like for soft bodies and clothes.

Дві примітки наостанок:

  1. For render farms, it is best to bake all the physics systems, and then copy the blendcache to the render farm as well.
  2. Будьте обережні з послідовністю модифікаторів у стеку модифікаторів (як завжди). Ви можете мати різну кількість граней у 3D Огляді -- 3D View та для зображування, рендерингу (наприклад, при використанні підподілення поверхні), а тому, відзображений, рендерений результат може дуже відрізнятися від того, що ви бачите у 3D Огляді.
