Запікання імітацій диму¶

Опції «Кеш Диму» -- Smoke Cache.

Запікання -- Baking використовується для збереження результату імітації, щоб не було потреби її перераховувати.

Устави запікання диму знаходяться в «редактор Властивостей > Фізика > Дим > Кеш Диму» -- Properties ‣ Physics ‣ Smoke ‣ Smoke Cache. На відміну від більшості імітацій фізики фізика диму має деякі устави, що є специфічними саме для диму.

Формат Файлу -- File Format¶

Формат файлу, в якому зберігаються дані кешу.

Кеш Точок -- Point Cache¶

Власний для Blender'а формат кешування.

OpenVDB¶

Просунутий та ефективний метод зберігання, розроблений DreamWorks Animation.

Поля імітації можна знайти у файлах .vdb під наступними іменами:

  • "color" -- колір
  • "density" -- щільність
  • "heat" -- теплота
  • "heat old" (the temperature at the previous frame)
  • "flame" -- полум'я
  • "fuel" -- паливо
  • "react" -- реакція (координати реакції, використовувані для вогню)
  • "velocity" -- скорість
  • "shadow" -- тінь (тіні об'єму, обчислювані для рендерингу, зображування оглядвікна)
  • "texture coordinates" -- координати текстури (використовуються для турбулентності)
Стиснення -- Compression

Метод стиснення даних.

Zip
Ефективний, але повільніший метод стиснення.
Blosc
Багатопотокове стиснення з приблизно такими ж якістю та розмірами як Zip.
Нема -- None
Стиснення даних не застосовується.
Глибина Даних -- Data Depth

Бітова глибина для запису усіх скалярних (включаючи вектори) даних, нижчі значення зменшують розмір файлу кешу.

Дійсночислове (Половина)
Половинні дійсночислові значення (16-ти бітні дані). Дає менше даних з вигодою менших розмірів файлів.
Дійсночислове (Повне)
Повні дійсночислові значення (32-х бітні дані). Дає більше даних за рахунок більших розмірів файлів.

Xem thêm

Про інші опції дивіться детальніше тут -- General Baking.

Ghi chú

Запікання може бути зроблене, тільки після того, як ваш blend-файл з імітацією попередньо збережено. Якщо такий blend-файл не було збережено, то панель Smoke Cache є недоступною.
