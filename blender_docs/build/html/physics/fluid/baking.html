Запікання -- Baking¶

Опції імітації рідини з виділеним Доменом -- Domain.

Кнопка «Запікання» -- Bake¶

Perform the actual fluid simulation. Blender will continue to work normally, except there will be a progress bar in the header of the Info Editor, next to the render selector. Pressing Esc or the "x" next to the status bar will abort the simulation. Afterwards two .bobj.gz (one for the Final quality, one for the Preview quality), plus one .bvel.gz (for the Final quality) will be in the selected output directory for each frame.

Каталог Запікання -- Bake Directory¶

Каталог та префікс файлу для збереження запечених сітей поверхонь.

Це подібно до устав виводу анімації, тільки вибір файлу є дещо спеціальним: коли ви вибираєте будь-яку з попередньо генерованих сітей поверхні (наприклад, test1_fluidsurface_final_0132.bobj.gz), то префікс буде автоматично заданий (test1_ у цьому прикладі). Таким чином, імітація може бути зроблена кілька разів з різними уставами, що дозволяє швидко змінювати різні набори даних поверхні.

Примітки¶

Унікальний домен
Через наявну можливість поширення та пов'язування між сценами в одному blend-файлі може бути присутній тільки один домен.
Виділення запеченого домену
Після того, як домен був запечений, він змінюється на сіть рідини. Для повторного виділення домену з метою можливості повторного його запікання після зроблених змін перейдіть у будь-який кадр та виділіть з RMB цю сіть рідини. Далі, ви можете клацнути на кнопці Bake знову для переобчислення потоків рідини всередині цього домену.
Запікання завжди починається з кадру номер 1
Імітатор рідини ігнорує уставу Start на панелі Animation та завжди починає запікання з кадру 1. Якщо ви бажаєте мати початок імітації пізніше за кадр 1, то ви повинні ключувати об'єкти рідини у вашому домені як неактивні до кадру, в якому ви хочете мати початок імітації.
Запікання завжди закінчується у кадрі, заданому уставою End на панелі Animation Редактора Властивостей
If the frame rate is set to 25 frames per second, and ending time is 4.0 seconds, then you should (if your start time is 0) set your animation to end at frame 4.0 × 25 = 100.
Вивільнення попередньо запечених обчислень
Видалення вмісту теки "Bake" є руйнівним способом досягнення цього. Будьте обережні, якщо більше ніж одна імітація використовує однаковий каталог для запікання (переконайтеся, що вони використовують різні імена файлів, інакше вони будуть перезаписані один поверх іншого)!
Повторне використання запікань
Ручне уведення (або задання через пошук) маски попередньо збережених (запечених) каталогу та імені файлу обчислення перемкне цей потік рідини та деформування сіті так, що вони будуть вживати таке старе запікання. Тому, ви можете повторно використовувати запечені потоки, просто вказавши їх у цьому полі.
Час обробки запікання

Запікання забирає багато обчислювальної потужної (а звідси, і часу). Залежно від сцени може краще буде вмикати запікання на всю ніч.

Якщо сіть має модифікатори, то устави рендерингу будуть вживатися для експортування цієї сіті в обчислювач рідини. Залежно від устав цієї ситуації час обчислення та вживання пам'яті може експоненціально зрости. Наприклад, коли вживається рухома сіть з модифікатором Subdivision Surface як перешкода для зменшення часу імітації може допомогти його вимикання або задання більш низького рівня підподілу. Коли така уклад/оснастка коректна, то ви можете завжди збільшити цю уставу для отримання більш реалістичного результату.
