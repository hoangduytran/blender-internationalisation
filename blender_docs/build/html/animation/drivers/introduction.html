Вступ¶

Драйвери -- це скрипти, головне призначення яких полягає у керуванні властивостями за допомогою інших властивостей. Як приклад, обертання одного об'єкта може скеровуватися локацією, місцем знаходженням іншого об'єкта.

Додання і Вилучення¶

Існує кілька різних способів додання драйверів у Blender'і. Після додання драйверів вони зазвичай модифікуються у Редакторі Графів Graph Editor в його режимі Драйвери Drivers.

Додання Драйвера¶

Орієнтир

Меню:«Контекстне меню > Додати Драйвер» -- Context menu ‣ Add Driver
Скорочення:Ctrl-D

Поширеним способом додання драйвера для властивості є клацок RMB над потрібною властивістю, та вибором відповідної команди додання драйвера у контекстному меню. Драйвери також можуть бути додані шляхом наведення миші над установкою властивості в інтерфейсі та натиснення Ctrl-D. Виділені властивості будуть використані як призначення (вивід) для цього драйвера.

Усе від Цілі -- All from Target (іконка властивостей)
Це додасть драйвери до набору властивостей, що використовується як призначення. Це створює стандартну криву з ключкадрами у (0,0) та (1,1). Наприклад, це додасть драйвери для трьох значень по осям X, Y та Z для властивості Локація -- Location.
Одиничне від Цілі -- Single from Target
Це додасть одиничний драйвер для виділеної властивості, що буде використовуватися як призначення.
Узгодження Індексів -- Match Indices (іконка колірного круга)
Використовує відповідний індекс для скеровування відповідною властивістю подібної за розміром вектора/масиву властивістю. Це корисно для керування ob1.location за допомогою ob2.location або RGB color за допомогою XYZ location.
Пізніше Створити Вручну/(Одиничне) -- Manually Create Later/(Single) (іконка з рукою)
Це додає/установлює драйвер(и), кожен зі своєю одиничною змінною (але не заповнює значення змінних). Піпетка не з'являється.

У випадку задання джерельної/цільової властивості (увід) їх можна вибрати за допомогою Піпетка (наприклад, "Scale Y").

Ghi chú

Оскільки додання драйвера працює через Контекстний інтерфейс користувача Blender'а, то вам необхідно мати відкритими одночасно два примірники, вікна редактора «Властивості» -- Properties (і одне з них має бути пришпилене, що показувалися властивості на невиділеному об'єкті). Це необхідно, оскільки інтерфейс користувача не можна користуватися, поки активний режим піпетки для вказування даних. Тому, вам необхідно бути у змозі бачити обидві джерельну і цільову властивості при використанні піпетки.

Копіювання та Вставлення¶

Орієнтир

Меню:«Контекстне меню > Копіювати/Вставити Драйвер» -- Context menu ‣ Copy/Paste Driver

Драйвери можуть бути скопійовані у буфер та вставлені для іншої відповідної властивості за допомогою контекстного меню. При доданні драйверів з однаковими уставами, це може сприяти економії часу, який витрачається на зміну устав.

Вираз -- Expression¶

Це швидкий спосіб додання драйверів за допомогою скриптованого виразу. Спершу клацніть на полі значення властивості, для якої хочете додати драйвер, далі уведіть символ хешу # та за ним уведіть сам скриптований вираз.

Деякі приклади:

  • #frame
  • #frame / 20.0
  • #sin(frame)
  • #cos(frame)

Вилучення Драйверів¶

Орієнтир

Редактор:Редактор Графів -- Graph editor
Режим:Драйвери -- Drivers
Панель:«Регіон Властивості > Драйвер > Драйвери > Вилучити Драйвер» -- Properties region ‣ Driver ‣ Drivers ‣ Remove Driver
Меню:«Контекстне меню > Видалити (Одиничний) Драйвер(и)» -- Context menu ‣ Delete (Single) Driver(s)
Скорочення:Ctrl-Alt-D

Огляд Графа¶

Приклад драйвера у Редакторі Графів.

Головна область Редактора Графів -- Graph editor у режимі Driverпоказується як Ф-Крива -- F-Curve, що розкладає Значення Драйвера -- Driver Value на цільову властивість. Driver Value -- це вивід скрипта драйвера. Вісь X представляє Значення Драйвера, а вісь Y -- це значення цільової властивості. На зображенні прикладу, якщо Driver Value є 2.0, то властивість буде 0.5.

Стандартна F-крива демонструє ідентичну розкладку значень, тобто співвідношення значень не змінюється. Це може використовуватися для створення коригувальних драйверів.
