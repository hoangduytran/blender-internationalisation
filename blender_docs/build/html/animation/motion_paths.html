Шляхи Руху -- Motion Paths¶

Орієнтир

Редактор:3D Огляд -- 3D View, Редактор Властивостей -- Properties editor
Режим:«Режим Об'єкта» -- Object Mode
Панель:«Полиця Інструментів > Анімація > Анімація > Шляхи Руху: Розрахунок» -- Tool Shelf ‣ Animation ‣ Animation ‣ Motion Paths: Calculate
Панель:«Редактор Властивостей > Об'єкт > Шляхи Руху» -- Properties editor ‣ Object ‣ Motion Paths

Орієнтир

Редактор:3D Огляд -- 3D View, Редактор Властивостей -- Properties editor
Режим:«Режим Пози» -- Pose Mode
Панель:«Полиця Інструментів > Інструменти > Інструменти Поз > Шляхи Руху: Розрахунок» Tool Shelf ‣ Tools ‣ Pose Tools ‣ Motion Paths: Calculate
Панель:«Редактор Властивостей > Арматура > Шляхи Руху» Properties editor ‣ Armature ‣ Motion Paths
Меню:«Поза > Шляхи Руху» Pose ‣ Motion Paths

Анімований куб з показуваним для нього шляхом руху.

Засіб «Шляхи Рухи» дозволяє вам візуалізувати рух точок як шлях через серію кадрів. Цими точками можуть бути початки (origins) об'єктів або суглоби кісток.

Перед тим, як ми глянемо на його опції, давайте спершу подивимося як показувати/ховати ці шляхи. На відміну від «привидів» -- Привиди -- Ghosts, це доводиться робити вам вручну, і спершу ви повинні виділити кістки, для ких ви хочете увімкнути/вимкнути шляхи руху. Далі:

  1. Для показування (або оновлення, при потребі) шляхів клацніть на кнопці «Розрахунок Шляху» -- Calculate Path.
  2. Для ховання цих шляхів клацніть на кнопці «Зчищення Шляхів» -- Clear Paths.

Ghi chú

Пам'ятайте, що лише виділені кістки та їх шляхи задіюються цими діями!

Дані шляхи рисуються світлим відтінком сірого для невиділених точок та дещо синюватим відтінком сірого для виділених точок. Навколо поточного кадру сяйво на шляху вказує напрямок руху: синій вказує на майбутні кадри, а зелений вказує на минулі кадри. Кожен кадр показується на шляху маленькою білою точкою.

Як і привиди, шляхи автоматично оновлюються, коли ви редагуєте ваші пози/ключкадри, а також вони активні у ході відтворення анімації. Відтворення за анімації за скороченням Alt-A корисне при увімкненій опції «Навколо Кадру» Around Current Frame.

Опції¶

Панель Motion Paths у вкладці Armature.

Тип
Навколо Кадру -- Around Frame
Показує шляхи точок у межах фіксованої кількості кадрів навколо поточного кадру. Коли ви вмикаєте цю кнопку, ви отримуєте шляхи для заданої кількості кадрів перед і після поточного кадру (знову, так само як і привиди).
У Діапазоні -- In Range
Показує шляхи точок у межах визначеного діапазону кадрів.
Діапазон Показу -- Display Range
Перед -- Before, Після -- After
Кількість кадрів для показу перед і після поточного кадру (тільки для методу тільки «Навколо Кадру» Around Current Frame).
Старт -- Start, Кінець -- End
Стартовий і Кінцевий кадри діапазону шляхів для показу/розрахунку (тільки не для методу «Навколо Кадру»).
Крок -- Step
Це те ж саме, що й устава «Крок» Step для привидів. Вона дозволяє вам показувати на шляху тільки певну кількість номерів кадрів через кожен n кадр. Здебільшого корисна, коли ви вмикаєте показ номерів кадрів (дивіться нижче), для запобігання безладдю у 3D-Оглядах -- 3D Views.
Кеш -- Cache / Кеш для Кістки -- Cache for Bone
Із -- From, До -- To
Це стартовий і кінцевий кадри діапазону, в якому рисуються шляхи руху. Ви не можете модифікувати цей діапазон без видалення спершу попереднього шляху рух.
Розрахунок -- Calculate / Оновлення Шляхів -- Update Paths

Якщо немає ще жодних розрахованих шляхів, то кнопка «Розрахувати» -- Calculate створить новий шлях руху у кеші на основі опцій визначених у спливному меню або на панелі.

Якщо шлях вже перед цим було розраховано, то кнопка «Оновити Шляхи» -- Update Paths оновить форму шляху для поточної анімації. Для зміни діапазону кадрів розрахованого шляху вам необхідно спочатку видалити наявний шлях та розрахувати його знову.

Старт -- Start, Кінець -- End
Це стартовий і кінцевий кадри діапазону, в якому рисуються шляхи руху. Ви маєте «розрахувати шляхи» Calculate Paths знову, якщо ви модифікували цю уставу, для оновлення шляхів у 3D-Оглядах -- 3D Views. Зауважте, що на відміну від привидів, стартовий кадр є «включним» inclusive (тобто, якщо ви установите значення «Старт» Start як 1, то ви реально бачитимете кадр 1 як стартову точку цього шляху...).
Запікання Локації -- Bake Location
Кістки лише -- Стандартно, ви отримуєте шляхи верхівок (tips, Tails). Змінивши значення цієї опції з «Хвости» Tails, ви можете отримати шляхи коренів, «Голови» Heads (пам'ятайте, що в інтерфейсі користувача Blender'а корені кісток називаються «головами» "heads"...). Ви маєте «розрахувати шляхи» Calculate Paths знову, якщо ви модифікували цю уставу, для оновлення шляхів у 3D-Оглядах -- 3D Views.
Зчищення Шляхів -- Clear Paths X
Зчищає шляхи на усіх об'єкта/кістках або тільки на виділених, коли утримуєте натисненою Shift.

Показ -- Show¶

Номери Кадрів -- Frame Numbers
При увімкненні будуть показуватися маленькі номери біля кожної точки кадру на шляху, які, звичайно ж, відповідають номеру відповідного кадру.
Лінія -- Line

Перемикає те, чи рисуються лінії між цими точками.

Товщина -- Thickness, Колір -- Color (іконка колірного круга)
Налаштовувані товщина та колір для ліній.
Ключкадри -- Keyframes
При увімкненні великі жовті квадратні точки рисуються на шляху руху, показуючи ключкадри кістки (тобто, тільки на шляхах ключованих кісток у заданому діапазоні мають жовту точку у ключовому кадрі).
+ Не-Груповані Ключкадри -- + Non-Grouped Keyframes
Для шляхів руху кісток, при увімкненні здійснює пошук по всій Дії -- Action замість пошуку у групах тільки за відповідними іменами (це повільніше).
Номери Ключкадрів -- Keyframe Numbers
При увімкненні ви будете бачити номери показуваних ключкадрів, а тому очевидно ця опція працює тільки, якщо увімкнена опція Ключкадри -- Keyframes.
