Пов'язані Бібліотеки -- Linked Libraries¶

Ці функції допомагають вам повторно використовувати матеріали, об'єкта та інші блоки даних -- data-blocks, завантажені із зовнішнього джерельного blend-файлу. Ви можете будувати бібліотеки загального вмісту та ділитися ними через численні посилальні файли.

Прилучення та Пов'язання -- Append and Link¶

Орієнтир -- Reference

Editor:Info Editor
Mode:All Modes
Menu:File ‣ Append or Link
Hotkey:Shift-F1 or Ctrl-Alt-O

«Пов'язання» -- Link створює посилання на дані у джерельному файлі, так що зміни зроблені там будуть відображатися у файлі, з якого здійснене посилання, наступний раз при завантаженні останнього.

Тоді як, «Прилучення» -- Append робить повну копію даних із зовнішнього файлу у даний ваш blend-файл. Ви можете робити подальші правки у вашій локальній копії даних, але зміни у зовнішньому джерельному файлі не будуть відображатися у файлі, в яке було здійснене прилучення.

В Оглядачі Файлів -- File Browser перейдіть у зовнішній джерельний blend-файл та виберіть блок даних, котрий ви хочете повторно використати.

Опції¶

Відносний Шлях -- Relative Path
Доступно тільки при пов'язуванні, детальніше дивіться тут -- relative paths.
Виділ -- Select
Робить об'єкт активним -- Active після його завантаження.
Активний Шар -- Active Layer
Об'єкт буде призначено на видимі шари у вашій сцені. Інакше, він буде призначений на ті ж шари, на яких він знаходиться у джерельному файлі.
Примірники Груп -- Instance Groups
Ця опція пов'язує Групу -- Group з об'єктом, додаючи її в активну сцену.
Псевдо Користувач -- Fake User
Sets a Fake User for the appended items.
Локалізувати Усе -- Localize All
Appends/links all appended data, including those indirectly linked from other libraries.

Коли ви вибираєте елемент типу Об'єкт -- Object, то він буде поміщений у сцені у місці курсора. Багато інших типів даних, наприклад, камери, криві та матеріали, повинні бути пов'язані з об'єктом, перед тим як вони стануть видимими.

Newly added Group types are available in Add ‣ Group Instances in 3D View, or for Node Tree groups, the same menu in the Node Editor.

Дивіться у Структураторі -- Outliner із заданим режимом показу blend-file, щоб бачити усі пов'язані та прилучені блоки даних. Клацок Ctrl-LMB на імені файлу дозволяє вам перенаправити пов'язування в інший файл.

Замісники Об'єктів -- Proxy Objects¶

Орієнтир -- Reference

Editor:3D View
Mode:Object Mode
Menu:Object ‣ Make Proxy...
Hotkey:Ctrl-Alt-P

Дозволяє вам робити зміни локально для об'єкта (або групи), пов'язаної із зовнішньої бібліотеки. Деякі типи змін залишаються обмеженими, але інші можуть бути зроблені локально, залежно від типу об'єкта. Ці зміни не відсилаються у зовнішню бібліотеку. Клацок Ctrl-Alt-P робить активний пов'язаний об'єкт локальним замісником, прилучаючи до його імені суфікс "_proxy".

Використовуючись для оснащених моделей, замісники об'єктів дозволяють визначати шари кісток, через пов'язання назад у джерельний файл, а решту об'єкта та його скелет редагувати локально. Установлюйте «Захищені Шари» -- Protected Layers у джерельному файлі на панелі «Скелет» -- Skeleton на вкладці «Арматура» -- Armature. Детальніше дивіться тут -- Armature Layers. Кістки на захищених шарах будуть мати їх позицію, збережену із джерельного файлу, коли файл з пов'язанням перезавантажиться.

Зробити Зв'язки -- Make Links¶

Орієнтир -- Reference

Editor:3D View
Mode:Object Mode
Menu:Object ‣ Make Link...
Hotkey:Ctrl-L

Пов'язує об'єкти між сценами або блоки даних активного об'єкта з усіма виділеними об'єктами. У деяких випадках (наприклад, Дані Об’єкта, Модифікатор) цільові об'єкти повинні бути такого ж типу, що й активний, або здатні отримувати такі дані. Наявний блок даних якого буде відв'язано від них.

Об'єкти до Сцени -- Objects to Scene

Дозволяє вам створювати пов'язання виділених об'єктів до інших сцен. Повинно бути обране ім'я іншої сцени, а не поточної. Панель оператора Link Objects to Scene дозволяє вам вибирати між сценами.

Це робить, що цей же об'єкт існує у двох різних сценах одночасно, включаючи його позицію та анімаційні дані. Початок такого об'єкта змінить свій колір.

Тип -- Type

Тип блоку даних для пов'язання.

Дані Об'єкта -- Object Data, Матеріали -- Materials, Дані Анімації -- Animation Data, Група -- Group, Дубль-Група -- DupliGroup, Модифікатори -- Modifiers, Шрифти -- Fonts

Перенести UV Карти -- Transfer UV Maps
Активна UV карта виділених об'єктів буде заміщена копією активної UV карти активного об'єкта. Якщо виділений об'єкт не має жодних UV карт, то вона буде створена. Об'єкти повинні бути типу «Сіть» -- Mesh та повинні мати таку же кількість граней (відповідність геометрії).

Xem thêm

Зробити Однокористувацьким -- Make Single User про відв'язування блоків даних.

Зробити Локальним -- Make Local¶

Орієнтир -- Reference

Editor:3D View
Mode:Object Mode
Menu:Object ‣ Make Local...
Hotkey:L

Робить виділений або усі пов'язані ззовні об'єкти локальними у поточному blend-файлі. Це робить, наприклад, його позицію редагованою, оскільки його позиція при пов'язанні визначається у джерельному файлі.

Тип -- Type

Optionally unlinks the object's Object Data and Material Data.

Selected Objects, + Object Data, + Materials, All (тобто, включаючи усі сцени)

Ghi chú

Прилучення даних, що вже були пов'язані, додає об'єкти/групи у сцену, але зберігає їх пов'язаними (та не-редагованими).

Це зроблено, щоб наявні відношення між пов'язаними даними залишалися непошкодженими.

Gợi ý

Інший спосіб трансформування об'єкта локально -- це використання дубль-груп -- Dupli-Groups. Замість пов'язування Об'єктів безпосередньо, часто більш корисно пов'язувати їх через Групи, яким можуть бути призначені порожняки та при пересуві будуть зберігатися зв'язки з оригінальним файлом.

Також корисно бути у змозі додавати/вилучати об'єкти з групи без управління пов'язанням одночасно багатьох об'єктів.

Відомі обмеження¶

For the most part linking data will work as expected, however, there are some corner cases which are not supported.

Кругові залежності -- Circular Dependencies¶

Загалом, залежності не повинні йти в обох напрямках.

Спроба пов'язати або прилучити дані, які пов'язані назад у поточний файл, призведе до втрати пов'язань.

Object Rigid Body Constraints¶

При пов'язуванні об'єктів безпосередньо у blend-файл устави «Тверде Тіло» -- Rigid Body **не будуть* пов'язані, оскільки вони асоційовані з їх світом сцени.

Як альтернатива, ви можете пов'язати їх до цілої сцени та установити їх як уставу фону -- Background Set.
