Блоки Даних -- Data-Blocks¶

The base unit for any Blender project is the data-block. Examples of data-blocks include: meshes, objects, materials, textures, node trees, scenes, texts, brushes, and even screens.

Для ясності, кістки -- bones, смужки відеоряду -- sequence strips та групи вершин -- vertex groups не є блоками даних, вони входять у склад таких типів, як арматура -- armature, сцена -- scene та сіть -- mesh, відповідно.

Деякі загальні характеристики:

  • Вони є первісним вмістом blend-файлу.
  • Вони можуть пов'язуватися один з одним для повторного використання та примірникування. (нащадок/предок, об'єкт/дані об’єкта, з модифікаторами та примусами тощо).
  • Їх ім'я є унікальним.
  • Вони можуть бути додані/вилучені/редаговані/дубльовані.
  • Вони можуть бути пов'язані між різними файлами (дозволено тільки для обмеженого набору блоків даних).
  • Вони мають свої власні анімаційні дані.
  • Вони можуть мати Власні Властивості -- Custom Properties.

При роботі з більш складними проектами управління блоками даних стає важливим, особливо при взаємо-пов'язуванні blend-файлів.

Огляд блоків даних.

Користувачі (Колекція Нутрощів) -- Users (Garbage Collection)¶

It is good to be aware of how Blender handles data-blocks lifetime, when they are freed and why.

Blender слідує загальному правилу, де невикористовувані дані зрештою вилучаються.

Оскільки прийнятим є додання та вилучення багатьох даних у ході роботи, то це надає перевагу у тому, що не потрібно вручну управляти кожним невикористаним одиничним блоком даних.

Це працює через пропускання блоків даних з нульовою кількістю користувачів при записуванні blend-файлів.

In some cases, you want to save a data-block even when it is unused (typically for re-usable asset libraries). See Fake User.

Псевдо Користувач -- Fake User¶

Оскільки блоки даних з нульовою кількістю користувачів не зберігаються, є моменти, коли вам бажано змусити ці дані зберігатися, незалежно від наявності для них користувачів.

Якщо ви вибудовуєте blend-файл, який буде слугувати бібліотекою елементів, і ви намірені пов'язувати їх у та з інших файлів, то вам треба переконатися, що вони випадково не будуть видалені з такого файлу бібліотеки.

Do this by giving the data-blocks a Fake User, by pressing the F button next to the name of the data-block. This prevents the user count from ever becoming zero: therefore, the data-block will not be deleted (since Blender does not keep track of how many other files link to this one).

Користувачі (Суміщення) -- Users (Sharing)¶

Many data-blocks can be shared among other data-blocks.

Приклади, де сумісні дані є поширеними:

  • Суміщення текстур матеріалами.
  • Суміщення сітей між об'єктами (примірниками).
  • Суміщення анімованих дій між об'єктами, наприклад, для зроблення спільного згасання усіх освітлювачів.

Ви можете також суміщувати блоки даних між файлами, детальніше дивіться тут:

  • linked libraries.

Зробити Однокористувацьким -- Make Single User¶

Орієнтир -- Reference

Editor:3D View
Mode:Object Mode
Menu:Object ‣ Make Single User
Hotkey:U

Робить виділені або всі блоки даних об'єктів однокористувацькими, тобто, вони перестають бути суміщеними, спільними для цих об'єктів і кожен об'єкт буде мати свій унікальний блок даних.

Тип -- Type

These actions work on the selected objects, or on all the objects of the scene.

Усе -- All, Виділені Об'єкти -- Selected Objects

Блоки Даних -- Data-Blocks

Дозволяє вам, окрім попередньо визначеного вибору у цьому меню, вибрати тип блоку даних індивідуально.

Об'єкт -- Object, Дані Об’єкта -- Object Data, Матеріали -- Materials, Текстури -- Textures, Анімація Об'єкта -- Object Animation

Вилучення блоків даних¶

Як вже згадувалося у параграфі Users (Garbage Collection), блоки даних типово вилучаються, коли вони більше не використовуються нічим.

Проте, існують деякі винятки з цього.

Наступні блоки даних можуть бути вилучені безпосередньо: Сцена -- Scene, Текст -- Text, Група -- Group та Екран -- Screen.

Інші блоки даних, як Групи -- Groups та Дії -- Actions можуть бути «Відв'язані» -- Unlinked через контекстне меню Структуратора -- Outliner.

Mẹo

Деякі дані (особливо зображення) важко відстежувати, особливо тому, що огляди зображень рахуються як користувачі.

For data-blocks that can be unlinked hold Shift, while pressing on the X button. This force clears the user count, so the data-block will be removed on reloading.
