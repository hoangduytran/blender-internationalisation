Підтримувані Формати Графіки -- Supported Graphics Formats¶

Формати зображень -- Image Formats¶

Це список форматів файлів зображень, підтримуваних внутрішньо Blender'ом:

Формат -- Format Глибина Каналу -- Channel Depth Альфа -- Alpha Метадані -- Metadata DPI Розширення -- Extensions
BMP 8bit ✗ ✗ ✓ .bmp
Iris 8, 16bit ✓ ✗ ✗ .sgi .rgb .bw
PNG 8, 16bit ✓ ✓ ✓ .png
JPEG 8bit ✗ ✓ ✓ .jpg .jpeg
JPEG 2000 8, 12, 16bit ✓ ✗ ✗ .jp2 .jp2 .j2c
Targa 8bit ✓ ✗ ✗ .tga
Cineon & DPX 8, 10, 12, 16bit ✓ ✗ ✗ .cin .dpx
OpenEXR дійсночислове 16, 32bit ✓ ✓ ✓ .exr
Radiance HDR float ✓ ✗ ✗ .hdr
TIFF 8, 16bit ✓ ✗ ✓ .tif .tiff

Gợi ý

Якщо вас не цікавлять технічні подробиці, добрим правилом вибору вивідного формату для вашого проекту є:

Використання OpenEXR
if you intend to do compositing or color grading on these images.
Вживання PNG
якщо ви намірені виводити на екран або кодувати в кілька форматів відео.
Використання JPEG
для виводу на екран, де розмір файлу має значення та втрата якості є прийнятною.

Усі ці формати підтримують стискання, яке може бути важливими при рендерингу анімацій.

Gợi ý

Бітові глибини для форматів зображень представляють таку кількість тональних рівнів на канал:

8:256 рівнів
10:1024 рівнів
12:4096 рівнів
16:65536 рівнів

Відкриття зображень¶

Відносний Шлях -- Relative Path
Визначає шлях файлу відносно поточно відкритого blend-файлу.
Визначати Відеоряди -- Detect Sequences
Automatically detects image sequences in the selected images (based on the file name). Disable this when you do want to get single images that are part of a sequence.

Opening an Image Sequence¶

The filename of the images must contain a digit, indicating the frame. The sequence could be opened by the selection of the images and by the confirmation with the Open Image button or Return.

Збереження зображень¶

Формат Файлу -- File Format
Дає на вибір, в якому форматі зберегти зображення.
Режим Кольору -- Color Mode

Дає на вибір формат кольору для збереження у ньому зображення (або відео). Зауважте, що формат RGBA не є доступним для усіх форматів зображень, перевірте список для деталей.

BW, RGB, RGBA

Глибина Кольору -- Color Depth

Image file formats support a varying number of bits per pixel. This affects the color quality and file size. Commonly used depths:

8 біт (256 рівнів) -- 8 bit (256 levels)
Most common for on-screen graphics and video.
10, 12, 16 біт -- 10, 12, 16 bit
Використовується для деяких форматів, призначених для фотографії та цифрових фільмів (таких як DPX та JPEG 2000).
16 біт половинне дійсночислове -- 16 bit half float
Оскільки повна 32 бітна дійсночислова глибина дає часто більш, ніж достатню точність, то половинна дійсночислова глибина може заощадити простір на диску, все ще забезпечуючи високий динамічний діапазон.
32 бітне дійсночислове -- 32 bit float
Глибина кольору, що дає найвищу якість.

Ghi chú

Внутрішньо система зображень Blender'а підтримує одне з цих двох:

  • 8 біт на канал (4 x 8 біт) -- 8 bit per channel (4 x 8 bits).
  • 32 bit float per channel (4 x 32 bits) -- using 4x as much memory.

Зображення з глибиною вище, ніж 8 біт на канал конвертуються у 32 дійсночислові біти при завантаженін у Blender.

Стиснення -- Compression
Використовується для скорочення розміру файлу зображення. Наскільки, то це може залежати від формату файлу та використаних устав.
Якість -- Quality
Подібно до устави Compression, але вживається для базованих на JPEG форматів файлів. Якість вимірюється у відсотках, 0% означає максимальну величину стиснення, а 100% означає взагалі без стиснення.
Зберегти Як Зображ -- Save As Render
Застосовує трансформування кольору -- color transform до збереженого зображення.
Копіювання -- Copy
Стяг «Копіювання» буде визначати, чи цей блок даних посилається на ново створений файл чи посилання буде без змін, збережене в оригінальному файлі.

Подробиці форматів¶

Cineon & DPX¶

Cineon -- це стандарт фірми Kodak для сканування плівки, 10 бітів/канал та логарифмічно. DPX було отримано на основі Cineon як галузевий стандарт ANSI/SMPTE. DPX підтримує 16 біт кольорів/канал, лінеарно та логарифмічно. DPX є поточно широко адаптований стандарт, що використовується у галузях апаратного та програмнго забезпечення фільмовиробництва.

DPX, а також Cineon тільки зберігає та конвертує діапазон значень «видимих» кольорів між 0.0 та 1.0 (як результат зображування або компонування).

OpenEXR¶

ILM's OpenEXR став стандартом галузі програмного забезпечення для файлів зображень HDR, особливо через його гнучку та розширювану структуру.

Файл OpenEXR може зберігати одночасно кілька шарів та проходів. Це означає, що зображення OpenEXR можуть бути завантажені у компонівник зі збереженням шарів та проходів зображу без змін.

Опції виводу -- Output Options¶

Доступними опціями для виводу зображу OpenEXR є:

Глибина Кольору -- Color Depth

Зберігає зображення у власному форматі 16 дійсночислових біт на канал. Це скорочує фактичну «бітову глибину» до 10 біт з 5 бітами значення степеню та 1 біт знаковий.

Дійсночислове (Половина) -- Float (Half), Дійсночислове (Повністю) -- Float (Full)

Кодек -- Codec
PIZ
Безвтратне брижове стиснення. Стискає добре зображення із зерном.
ZIP
Стандартне безвтратне стиснення за допомогою Zlib.
RLE
Виконувано-довжинне закодоване -- run-length encoded, безвтратне стиснення, працює добре, коли сканлінії мають однакові значення.
PXR24
Втратний алгоритм від Pixar, що конвертує 32 дійсночислові біти у 24 дійсночислові біти.
Z-Буфер -- Z-Buffer
Зберігає інформацію про глибину. У Blender'і це тепер записується також дійсночисловими значеннями, позначаючи точну відстань від камери у значеннях одиниць Blender'а.
Передогляд -- Preview
При зображуванні, рендерингу анімацій (або одиничних кадрів через командний рядок) Blender зберігає ще таке саме зображення як JPEG для швидкого передогляду або завантаження.

Radiance HDR¶

Radiance -- це набір засобів для імітації освітлення. Оскільки Radiance мав перший (і впродовж тривалого часу єдиний) формат зображень HDR, тому цей формат підтримується багатьма іншими програмними пакетами.

Radiance .hdr files store colors still in 8 bits per component, but with an additional (shared) 8 bits exponent value, making it 32 bits per pixel.
