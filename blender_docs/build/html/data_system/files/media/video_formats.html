Підтримувані Формати Відео -- Supported Video Formats¶

Формати Відео -- Video Formats¶

Ці формати, перш за все, використовуються для стиснення рендерених, зображених відеорядів у відтворюваний фільм (вони можуть також використовуватися для зроблення звичайних аудіо файлів).

Кодек -- це невелика процедура, що стискає відео так, щоб воно помістилося на DVD, або могло поточно передаватися через Інтернет. через кабель, або просто мати розумний розмір файлу. Кодеки стискають канали відео для збереження простору та можливості продовжуваного відтворення. Втратні -- lossy -- кодеки роблять файли меншими за рахунок якості зображення. Деякі з кодеків, як H.264, чудово підхожі для стискання великих зображень. Кодеки використовуються для закодовування та розкодовування фільму, а тому повинні бути представлені на обох -- закодовуваній машині (Blender) та цільовій машині. Результати закодування зберігаються у файлі-контейнері.

There are dozens, if not hundreds, of codecs, including Xvid, H.264, DivX, Microsoft, and so on. Each has advantages and disadvantages and compatibility with different players on different operating systems.

Ghi chú

Більшість кодеків можуть стискати тільки RGB або YUV колірний простір -- color space, а деякі також підтримують канал прозорості Альфа. Кодеки, що підтримують RGBA:

  • Quicktime
  • PNG TIFF Pixlet is not lossless, and may be only available on macOS.
  • Lagarith Lossless Video Codec
AVI Codec
AVI codec compression. Which codecs are available depends on the operating system. When an AVI codec is initially chosen, the codec dialog is automatically launched. The codec can be changed directly using the Set Codec button which appears (AVI Codec settings.).
AVI Jpeg
AVI, але зі стисненням Jpeg. Втратний, менші файли, але не такі малі, як ви можете отримати з алгоритмом стиснення Кодека. Стиснення Jpeg також одне з тих, що використовується у форматі DV, що вживається цифровими відеокамерами.
AVI Raw
Звуко-відео черговані нестиснені кадри -- Audio-Video Interlaced (AVI).
Frameserver
Blender puts out frames upon request as part of a render farm. The port number is specified in the User Preferences.
H.264
Закодовує фільми кодеком H.264.
MPEG
Закодовує фільми кодеком MPEG.
Ogg Theora

Файл Apple's Quicktime .mov. Діалог кодека Quicktime доступний, коли цей кодек інстальовано на macOS. Дивіться Quicktime в Video Containers.

Контейнери Відео -- Video Containers¶

MPEG-1: .mpg, .mpeg
Стандарт для втратного стиснення відео та аудіо. Він призначений для стикання сирого цифрового відео з якістю VHS та аудіо CD з частотою бітів до 1.5 Mbit/s.
MPEG-2: .dvd, .vob, .mpg, .mpeg
Стандарт для «загального кодування фільмів та асоційованої аудіо інформації». Він описує комбінацію втратного стиснення відео та втратного стиснення аудіо даних методами, які дозволяють зберігати та передавати фільми за допомогою поточно доступних медіа сховищ та смуги передавання.
MPEG-4(DivX): .mp4, .mpg, .mpeg
Поєднує багато функцій MPEG-1 і MPEG-2, а також інших суміжних стандартів, та додає нові функції.
AVI: .avi
Похідна від формату файлу обміну ресурсів -- Resource Interchange File Format (RIFF), який ділить дані файлу на блоки або «скиби» -- "chunks".
Quicktime: .mov
Багато-стежковий формат. Формати контейнерів QuickTime та MP4 можуть використовувати однакові формати MPEG-4; вони є здебільшого взаємозамінними тільки у середовищі QuickTime. MP4, як міжнародний стандарт, має більше підтримки.
DV: .dv
An intra-frame video compression scheme, which uses the discrete cosine transform (DCT) to compress video on a frame-by-frame basis. Audio is stored uncompressed.
H.264: .avi наразі.
Стандарт для стиснення відео та поточно один з найбільш поширених використовуваних форматів для запису, стиснення та розповсюдження відео з високою роздільністю.
Xvid: .avi наразі
A video codec library following the MPEG-4 standard. It uses ASP features such as b-frames, global and quarter pixel motion compensation, lumi masking, Trellis quantization, and H.263, MPEG and custom quantization matrices. Xvid is a primary competitor of the DivX Pro Codec.
Ogg: .ogg, .ogv
Вільний втратний формат стиснення відео. Він розроблений фундацією Xiph.Org Foundation та розповсюджується без ліцензійних платежів.
Matroska: .mkv
A free open-standard container format, a file format that can hold an unlimited number of video, audio, picture or subtitle tracks in one file.
Flash: .flv
Формат файлу контейнера, що використовується для доставки відео через Інтернет за допомогою Adobe Flash Player.
Wav: .wav
Нестиснений (або легко стиснений) формат файлів аудіо Microsoft and IBM.
Mp3: .mp3
Високо-стиснений, запатентований формат кодування цифрового аудіо, що використовує форму втратного стиснення даних. Це поширений формат аудіо для споживчого зберігання аудіо, а також де факто стандарт стиснення цифрового аудіо для передачі та відтворення музики на цифрових аудіо програвачах.

Контейнери Аудіо -- Audio Containers¶

MP2
Втратний формат стиснення аудіо, визначений ISO/IEC 11172-3.
MP3
Дивіться MP3 у Video Formats (вище).
AC3
Audio Codec 3, технологія стиснення аудіо, розроблена Dolby Laboratories.
AAC
Advanced Audio Codec, стандартизована втратна схема стиснення та закодування для цифрового аудіо. -- AAC загалом забезпечує кращу якість звуку, ніж MP3, з такими ж частотами бітів -- бітрейтами.
Vorbis
Відкрито-стандартний високо-стискуваний формат, порівняний з MP3 або AAC. -- Vorbis загалом забезпечує кращу якість звуку, ніж MP3, з такими ж частотами бітів.
FLAC
Вільний безвтратний кодек аудіо -- Free Lossless Audio Codec (FLAC). Цифрове аудіо, стиснене алгоритмом FLAC, може типово бути меншим у 50-60% від його оригінального розміру, та розпаковуватися в ідентичну копію оригінальних даних аудіо.
PCM
Pulse Code Modulation, метод, використовуваний для цифрового представлення відібраних аналогових сигналів. Він є стандартною формою для цифрового аудіо в комп'ютерах та різних форматах Blu-ray, Compact Disc та DVD, а також для інших використань як цифрові телефонні системи.

Відомі обмеження¶

Розмір Виводу Відео -- Video Output Size¶

Деякі кодеки накладають обмеження на розмір виводу, H.264, наприклад, вимагає, щоб висота і ширина кадру були числом, що ділиться на 2.
