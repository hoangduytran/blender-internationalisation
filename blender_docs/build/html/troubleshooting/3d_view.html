3D-Огляд -- 3D View¶

Прорис¶

Перебої буфера глибини¶

Інколи, коли задається більший діапазон відсікання -- clipping range, це дозволяє вам бачити як глибокі, так і далекі об'єкти, але зменшує точність глибини, що виражається в артефактах.

Модель без жодних артефактів відсікання.

Модель з артефактами відсікання.

Сіть з артефактами в Edit Mode.

Для запобігання цьому:

  • Збільште ближчу межу відсікання при роботі з великими сценами.
  • Зменште дальню межу відсікання, коли об'єкти не показуються на відстані.

When perspective is disabled only the far Clip End is used, very high values can still give artifacts.

This is not specific to Blender, all OpenGL/DirectX graphics applications have these same limitations.

Об'єкти невидимі в огляді камери¶

Якщо ви маєте велику сцену, огляд її через огляд камери Camera View може не показувати всі об'єкти у сцені. Однією з можливих причин може бути те, що відстань відсікання -- clipping distance для камери є надто низькою. Камера буде показувати тільки ті об'єкти, що попадають у межі діапазону відсікання.

Продуктивність -- Performance¶

Повільний прорис¶

Існує купа причин, чому ви можете мати повільне оглядвікно.

Старе устаткування
Інколи ваше устаткування, головним чином ваша відеокарта, може бути надто повільною, щоб реально показувати модель.
Оновлення драйвера відеокарти
У деяких випадках, повільне виділення вирішується за допомогою оновлення драйверів.

Повільне виділення¶

Blender використовує рисування OpenGL для виділення, деякі драйвери відеокарт є повільними для здійснення цієї операції.

Це стає особливо проблематичним при щільній геометрії.

Можливі рішення:

OpenGL Occlusion Queries (User Preferences)

See User Preferences ‣ System ‣ Selection.

Опція «Метод Виділення» -- Selection Method стандартно задана як «Автоматично» -- Automatic, спробуйте задати «Запити на Загороду з OpenGL» -- OpenGL Occlusion Queries, оскільки існує відчутна різниця в продуктивності під деякі конфігурації.

Оновлення драйвера відеокарти
У деяких випадках, повільне виділення вирішується за допомогою оновлення драйверів. Загалом добре використовувати найостанніші драйвери при використанні програми 3D.
Виділення центрів (обхід)
In Object Mode, holding Ctrl while selecting uses the object center point. While this can be useful on its own, its has the side effect of not relying on OpenGL selection.
Зміна режимів прорису (обхід)
Використання режимів прорису «Каркас» -- Wireframe або навіть «Габаритна Коробка» -- Bounding Box може бути вжито для більш швидкого виділення різних об'єктів.

Ghi chú

Очевидно, обходи, перелічені тут, не є довготривалими рішеннями, але зручно знати, якщо ви застрягли, про використання системи з поганою підтримкою OpenGL.

Зрештою, якщо жоден з цих варіантів не працює, то можливо варто оновити ваше устаткування.

Навігація¶

Загублення у просторі¶

При навігації по сцені ви можете випадково зайти далеко від її центру та виявити, що дивитися на порожнє оглядвікно. Існує два шляхи виправити це:

  1. Виділити потрібний об'єкт у Структураторі -- Outliner та далі зумувати його у вигляд за допомогою «Огляд > Показати Активне» -- View ‣ Show Active або NumpadPeriod.
  2. Скористатися Home для вписування усіх об'єктів в 3D Огляд -- 3D View.

Невидимий ліміт присування¶

Іноді при навігації ви пробуєте присувати огляд, але здається, що ви досягли дальньої можливої межі зумування. Це через те, що Blender використовує центральну точку для обльоту сцени.

На практиці це добре для моделювання об'єкта, який ви багато обертаєте, щоб бачити усі його боки (уявіть гончара, який використовує для цього круг). Однак, це робить незручним дослідження сцени чи моделі об'єкта «ізсередини», наприклад.

Рішення¶

  • Use View Dolly.
  • Використати режими «Ходьба» або «Політ» -- Walk/Fly modes.
  • Вжити Авто Глибину -- Auto Depth та Зумування До Позиції Миші -- Zoom to Mouse Position. Ці засоби гарантуватимуть, що відстань -- це завжди значення під вказівником миші.
  • Use Border Zoom as it also resets the center point when zooming.
  • Центрувати огляд навколо вказівника миші з Alt-F. Це візьме позицію під вказівником миші та зробить її центром вашого оглядвікна.
  • Центрувати огляд навколо 3D курсора з Alt-Home.
  • Use an NDOF, also known as a 3D mouse, see configuring peripherals for more information.

Інструменти -- Tools¶

Хибне виділення¶

Є випадки, коли виділення не вдається під певними конфігураціями, часто це помітно у режимі правки Edit Mode для сітей, виділення вершин/ребер/граней з яким виділяються випадкові елементи.

Внутрішньо Blender вживає OpenGL для виділення, а тому драйвер відеокарти опирається на поданих на нього коректних результатах.

Можливі рішення:

Disable Anti-Aliasing FSAA, Multisampling

Це, безумовно, найбільш поширена причина проблем з виділенням.

There are known problems with some graphics cards when using FSAA/multisampling.

Ви можете вимкнути цю опцію:

  • Turning FSAA/multisampling off in your graphics card driver options.
  • Turning Multi-Sampling off in the System Preferences.
Зміна устав вибірки загладжування

Залежно від вашої конфігурації OpenGL, деякі специфічні устави вибірки можуть працювати, а деякі -- ні.

На жаль, знаходження робочої конфігурації передбачає пробування та перевіряння на помилки.

Оновлення драйвера відеокарти

As with any OpenGL-related issues, using recent drivers can resolve problems.

Однак, слід зазначити, що це є досить поширеною проблемою та залишається невирішуваним для багатьох драйверів.
