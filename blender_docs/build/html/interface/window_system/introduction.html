Вступ¶

Після стартування Blender'а та закриття Екрану Заставки -- Splash Screen вікно вашого Blender'а повинно виглядати дещо подібно, як на зображенні нижче. Інтерфейс користувача Blender'а є одноманітним на всіх платформах.

Стандартне вікно при запуску Blender'а.

Елементи інтерфейсу¶

Window ‣ Screen ‣ Areas ‣ Editors ‣ Regions ‣ (Tabs) ‣ Panels ‣ Controls

Цей інтерфейс може бути налаштовано відповідно до конкретних завдань за допомогою Розстав Екрану -- Screen Layouts, які можуть бути іменовані та збережені для подальшого використання. Стандартний екран описується нижче.

Екран організується з однієї чи кількох Областей -- Areas, кожна з яких містить Редактор -- Editor.

Стандартний екран¶

Стандартно Blender запускається з показом стандартного екрану, який розділений на п'ять областей, що мість Редактори -- Editors, перелічені нижче:

  • Редактор Інфо -- Info Editor вгорі.
  • Великий 3D-Огляд -- 3D View.
  • Часолінія -- Timeline внизу.
  • Структуратор -- Outliner у верхньому правому куті.
  • Редактор Властивостей -- Properties Editor у нижньому правому куті.

Blender's default Screen Layout with five editors.

Info (blue), 3D View (yellow), Outliner (red), Properties (green) and Timeline (purple).

Компоненти редактора¶

У загальному редактор забезпечує спосіб огляд та модифікації вашої роботи через певну частину Blender'а. Редактори поділяються на Регіони -- Regions. Регіони можуть мати менші структурні елементи, як вкладки й панелі -- tabs and panels з кнопками, контролерами та віджетами, розміщеними в них.

Редактор 3D Огляд -- 3D View.

Жовтий: Головний Регіон -- Main Region, зелений: Заголовок -- Header, синій: Полиця Інструментів -- Tool Shelf, пурпуровий: Панель Оператора -- Operator Panel, червоний: Регіон Властивостей -- Properties Region.

Принципи інтерфейсу користувача¶

Non-Overlapping
Цей інтерфейс користувача спроектовано так, щоб дозволити вам бачити усі відповідні опції та інструменти з першого погляду без зайвих штовхання та перетягування редакторів.
Non-Blocking
Інструменти та опції інтерфейсу не блокують користувача від будь-яких інших частин Blender'а. Blender типово не використовує спливні панелі (що вимагають від користувачів заповнення даних перед виконанням операції).
Non-Modal Tools
Інструменти можуть бути ефективно доступні без витрати часу на вибір між різними інструментами. Багато інструментів використовують узгоджені та передбачувані дії миші і клавіатури для взаємодії.

Налаштування¶

Blender також передбачає використання клавіатурних скорочень для прискорення роботи. Вони можуть також налаштовуватися у Редакторі Розкладки Клавіш -- Keymap Editor.

Теми кольорів

Blender дозволяє змінювати більшість колірних устав інтерфейсу відповідно до потреб користувача. Якщо ви виявите, що кольори, які ви бачите на екрані не відповідають тим, що згадуються у цьому Посібнику, то можливо у вас змінено стандартну колірну тему Blender'а. Створення нової або вибір/зміна наявної теми може здійснюватися на вкладці «Теми» -- Themes редактора Уподобання Користувача -- User Preferences.
